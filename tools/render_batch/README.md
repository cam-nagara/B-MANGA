# B-Name-Render 連続実行アプリ

複数のPC・複数のファイル・複数のプリセットを、好きな順番で連続レンダリングするための
デスクトップアプリです。各レンダリングの開始・完了・所要時間を記録し、過去の記録から
完了予測時刻を表示します。

## できること

- **実行待ちリスト**に、どのファイルのどのプリセットを実行するか、好きな順で並べる
- 共有フォルダ（Dropbox）を介して、**複数のPCで仕事を分担**して連続実行（1台につき1本ずつ）
- プリセットごと・レンダリング工程ごとの**開始/完了/所要時間を記録**し、「記録」タブで閲覧
- 過去の記録から**完了予測時刻**を各行に表示

## 仕組み（全体像）

```
[アプリ画面]  ──  [共有フォルダ(Dropbox)]  ──  [各PCのアプリ]
 リスト編集          実行待ち / 記録               空いた仕事を取り、
 記録閲覧                                          裏でBlenderを起動して実行
 完了予測
```

各PCで同じアプリを起動し、共有フォルダの同じ実行待ちリストを見ます。アプリが裏で
Blenderを画面なしで起動し、指定ファイルの指定プリセットだけを実行します。

## 必要なもの

- 各PCに **Blender 5.1**（バージョンは全PCで統一）
- 各PCに **B-Name-Render** が導入済み（バージョン統一）
- 全PCから**同じパスで見える共有フォルダ**（Dropbox。ドライブレターまで統一）
- Python 3（Windows標準のPythonで可。追加インストール不要 / Tkinter使用）

## 起動

```
python tools/render_batch/run_app.py
```

## 初回設定（「設定」タブ）

- **共有フォルダ**: Dropbox上のフォルダを指定（全PCで同じパス）
- **自PC名**: 空ならコンピュータ名を使用（記録に残ります）
- **Blender実行ファイル**: 既定は `C:\Program Files\Blender Foundation\Blender 5.1\blender.exe`
- **同期猶予(秒)**: Dropboxの同期を待つ時間。`3`前後を推奨
- **監視間隔(秒)**: 実行待ちを見に行く間隔

## 使い方

1. 「実行待ち」タブで **ジョブ追加** → `.blend` を選ぶ → プリセット一覧から選んで追加
2. 並び順を **▲上へ / ▼下へ** で調整
3. レンダリングさせたいPCで **このPCで実行開始** を押す
4. 各PCは実行待ちの先頭から、自分が担当できる仕事を取って順に実行します
5. 終わると「記録」タブに開始/完了/所要時間が残り、完了予測の精度が上がっていきます

## 完了予測のしくみ

過去の記録から、同じ条件（プリセット・ファイル・解像度・サンプル数・PC）の平均所要時間を
予測値にします。完全一致が無ければ、条件を段階的に緩めて推定します（同解像度→同ファイル→
同プリセット）。記録が貯まるほど精度が上がります。

## 注意（Dropbox利用時）

- Dropboxは同期に時間差があります。2台が同時に同じ仕事を取りに行った場合は、PC名の順で
  自動的に1台に決まります（**同期猶予**の秒数だけ待ってから判定）。
- レンダリングは重いため、**1台につき1本ずつ**実行します。
- 実行中のファイルはそのPCがBlenderで開きます。同じファイルを別作業で同時編集しないでください。

## 出力ファイルごとの時間について

B-Name-Renderは1回のレンダリングで複数の画像を同時に書き出すため、厳密な
「画像1枚ごとの時間」には分けられません。本アプリは**レンダリング工程ごとの時間**と、
その工程で生成された**画像ファイルの一覧**を記録します（「記録」で確認できます）。

## ファイル構成

```
tools/render_batch/
  run_app.py          起動エントリ
  runner.py           Blenderヘッドレス実行スクリプト（アプリが内部で使用）
  config.example.json 設定の例
  app/
    model.py          ジョブ・記録のデータ
    jobstore.py       共有フォルダのジョブ管理（取得の排他含む）
    predictor.py      完了予測（条件別平均）
    worker.py         Blender起動・実行・記録
    blender_locator.py
    config.py         設定の保存/読込
    gui.py            画面（Tkinter）
```
