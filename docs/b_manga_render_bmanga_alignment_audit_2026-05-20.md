## 1. 現行 B-MANGA 側で優先する仕様

- B-MANGA Render は単体運用を主目的にしない。魚眼モード、魚眼FOV、ページ画像のスケール、出力解像度の正は B-MANGA のコマ用blendファイル側。
- コマ用blendファイルには、ページ一覧ファイル用の B-MANGA コレクション全体を持ち込まない。
- コマ用blendファイルでは、ページ一覧側のコマ枠、ラスター、グリースペンシル、テキスト、フキダシ、効果線などを「ページ画像」としてカメラ下絵にする。
- 「ページ画像」は、編集中コマの背景だけを透明にし、ページ一覧側の線や各要素は残す。
- コマ用blendファイルの出力解像度は、ページ一覧ファイルの用紙サイズと dpi から決まる。
- ページ一覧上のコマ枠線は B-MANGA の実オブジェクト / 書き出し処理が正であり、B-MANGA Render が旧来の固定コレクションや固定画像名で合成するものではない。

## 2. 旧参照元アドオンの前提

`カメラ操作＆出力プリセット.py` は、B-MANGA の現在仕様ではなく、固定の出力用シーンを前提にしている。

- カメラ背景画像に `コマ枠.png`、`ネーム画像.png`、`コマNN` 画像群がある前提。
- `コマ枠` コレクションを魚眼出力時に全ビューレイヤーから除外する前提。
- `コマ`、`全コマ統合`、`ページ` などのコンポジター上のノードを切り替えてページ合成する前提。
- 魚眼設定は eeVR 側へ `domeMethodEnum = "1"` を固定で渡す前提。
- 魚眼中に外部スクリプト `下絵スケールに出力解像度調整.py`、`出力解像度を原稿サイズ化.py`、`出力解像度を魚眼レイアウトサイズ化.py` を実行する前提。

この前提は、現行 B-MANGA の「1コマ = 1 cNN.blend」「ページ画像は B-MANGA が生成したカメラ下絵」「コマ枠はページ一覧側の実体 / 書き出し処理が正」という設計と衝突する。

## 3. 現行実装で残っている不整合

### 3.1 旧ページ合成ノード前提が残っている

B-MANGA Render の初期プリセット `ページ` と `すべて` は、`コマ`、`全コマ統合`、`ページ` という旧コンポジター要素を切り替える。

現行 B-MANGA ではページ出力は B-MANGA 本体のページ出力が担当し、B-MANGA Render 側のコマ用blendファイル出力は 3D レンダーパスが中心になる。したがって、この `ページ` 合成は B-MANGA 仕様上の正本ではない。

実機確認:

```powershell
& 'C:\Program Files\Blender Foundation\Blender 5.1\blender.exe' --background --python test\blender_b_manga_render_c00_audit.py
```

結果: `c00.blend` 監査も、現時点では `ページ` と `全コマ統合` の要求名不一致で停止した。これは c00 互換としても崩れ始めており、B-MANGA 仕様への移行を優先する根拠になる。

### 3.2 `コマ枠` コレクション前提が残っている

Pencil+4 系プリセットは、魚眼 / 通常出力時に `コマ枠` コレクションを除外 / 表示する。

現行 B-MANGA のコマ用blendファイルでは、ページ一覧側の枠線は「ページ画像」またはページ一覧側の実体として扱う。コマ用blendファイル内に固定名 `コマ枠` コレクションがあるとは限らない。

修正方針:

- B-MANGA 標準プリセットから `コマ枠` コレクション操作を外す。
- 旧 c00 互換を残す場合は、明示的な「旧出力シーン互換」プリセットまたは任意カードとして分離する。
- コマ枠をレンダーから外す必要がある場合は、B-MANGA 側のページ画像 / 下絵表示と、コマ用blendファイル側のレンダー対象を分けて扱う。

### 3.3 レンダー範囲が旧 `コマ枠.png` 前提

旧参照元は `コマ枠.png` のカメラ背景画像を探してレンダー範囲を決めていた。

現行 B-MANGA は、管理対象のページ画像に `BManga_コマ..._page.png` 系の名前とカスタム情報を持たせ、`full_page_mask` や左右ページ情報で下絵を配置する。`コマ枠.png` 固定検索では現行の見開き / 現在コマ透明化 / ページ画像スケールに追従できない。

修正方針:

- B-MANGA Render 側に、B-MANGA 管理のページ画像を検出する読み取り専用ヘルパを追加する。
- `コマ枠.png` は旧互換のフォールバックに下げる。
- B-MANGA Render 側で魚眼 / 縮小を反映した後も、B-MANGA 管理の下絵に基づくレンダー範囲を再計算できるようにする。

### 3.4 魚眼投影方式を B-MANGA 側設定から上書きしている

現行 B-MANGA は、カメラの魚眼投影方式をユーザーが `等距離射影` / `等立体角射影` で使い分ける方針。未設定時だけ `等立体角射影` を既定にする。

一方、B-MANGA Render は魚眼設定時にカメラを `等距離射影` へ固定している。また eeVR 側へも旧参照元と同じ方式を固定で渡している。

修正方針:

- B-MANGA Render はカメラ側の魚眼投影方式を上書きしない。
- カメラが `等距離射影` なら eeVR 側も等距離相当へ、`等立体角射影` なら等立体角相当へ対応付ける。
- eeVR 側で一致できない方式の場合は、黙って別方式で出さず、B-MANGA の Cycles 魚眼出力へフォールバックするか警告する。
- 魚眼FOV が 180 度を超える場合、現在の 180 度固定 / clamp をやめ、出力方式に応じた値を渡す。

### 3.5 出力先と画像名が B-MANGA のコマ単位に寄っていない

B-MANGA Render は既定で `//passes/` と任意の画像名を使う。これは cNN.blend 直下の `passes/` としては概ね合うが、B-MANGA の現在コマ情報、ページ番号、プリセット名、既存の `cNN_thumb.png` / `cNN_preview.png` との役割分担までは見ていない。

修正方針:

- B-MANGA 管理下のコマ用blendファイルでは、出力先既定をそのコマの `passes/` に確定する。
- 画像名は、プリセット名や出力用途に基づいて B-MANGA 側のコマプレビュー名と衝突しない名前にする。
- `コマプレビューとして出力` の旧プロパティは、現行 B-MANGA では B-MANGA 本体のページ一覧プレビュー生成と役割が重なるため、削除または明確に無効化する。

### 3.6 B-MANGA Render マニュアルが旧UI説明のまま

現行パネルでは魚眼モードと魚眼FOVは B-MANGA 側で調節し、B-MANGA Render では読み取り表示になっている。しかしマニュアルには、B-MANGA Render 側で魚眼モード、魚眼FOV、ページ画像のスケールを設定する説明が残っている。

修正方針:

- マニュアルを現行 UI に更新する。
- 「ページ画像のスケール」は B-MANGA のコマ編集側で扱う項目として説明する。
- 「魚眼出力欄が編集できない」という旧トラブルシュートを削除し、「B-MANGA 側のコマ編集: カメラで調節する」に置き換える。

### 3.7 監査テストが c00 互換中心

既存の B-MANGA Render 監査は、`D:\TM Dropbox\Share\B-MANGA\c_file\c00.blend` との照合が中心になっている。今回の目的は「現在の B-MANGA 仕様に完全に合わせる」ことなので、c00 互換監査だけでは足りない。

修正方針:

- B-MANGA で新規作品を作成し、コマ用blendファイルを開いて、B-MANGA Render を実行する実機テストを追加する。
- テスト対象:
  - B-MANGA 側の魚眼モード / 魚眼FOV / 縮小率を B-MANGA Render が読む。
  - B-MANGA Render がカメラの魚眼投影方式を変えない。
  - 固定名 `コマ枠` コレクションがなくても標準プリセットが破綻しない。
  - `ページ` / `全コマ統合` の旧ノードがなくても B-MANGA 標準の出力導線が成立する。
  - 出力先が現在コマの `passes/` へ向く。
  - ページ画像 / 下絵の表示、レンダー範囲が B-MANGA 側設定と一致する。
- c00 互換テストは、必要なら「旧出力シーン互換」用の任意テストに降格する。

## 3.8 徹底チェック結果

直前に作成した本計画書を起点に、ユーザー操作と呼び出し経路を確認した。結論として、計画の方向性は妥当。ただし開発セッションでは下記の経路を必ず押さえて修正すること。

### 操作1: コマ用blendファイルでプリセットを実行する

想定操作:

1. ユーザーが B-MANGA からコマ用blendファイルを開く。
2. B-MANGA Render タブでプリセットを選ぶ。
3. 「プリセットを実行」を押す。

現在の呼び出し経路:

`プリセットを実行` → `run_active_preset` → 各カードの実行 → `魚眼/通常レンダー` または `フレーム出力レンダー` → レンダー実行。

確認結果:

- `フレーム出力レンダー` は旧コンポジターの対象名を前提にするため、B-MANGA 標準のコマ用blendファイルでは対象が無い可能性が高い。
- `ページ` / `すべて` プリセットは `ページ`、`全コマ統合` の旧ノード名に依存するため、現行 B-MANGA 標準からは外す必要がある。
- 開発時は「対象名が無いカードを無視する」だけでなく、標準プリセット自体から旧前提を外すこと。

### 操作2: B-MANGA 側で魚眼モードを変えてから B-MANGA Render を使う

想定操作:

1. コマ編集のカメラ欄で魚眼モード / 魚眼FOV / 縮小率を変更する。
2. B-MANGA Render で魚眼系プリセットを実行する。

現在の呼び出し経路:

B-MANGA 側のコマ編集カメラ設定 → 共通設定同期 → B-MANGA Render 側の魚眼判定 → 魚眼設定 → eeVR またはレンダー実行。

確認結果:

- B-MANGA Render は B-MANGA 側の魚眼モード / 魚眼FOV を見るようになっているが、魚眼設定時にカメラの投影方式を固定値へ上書きする経路が残っている。
- B-MANGA 側は、ユーザーがカメラの魚眼投影方式を使い分ける前提。B-MANGA Render は投影方式を保存・上書きせず、読み取って出力へ渡す必要がある。
- 縮小率変更は B-MANGA 側と B-MANGA Render 側の両方に更新経路があるため、修正後は双方向同期の無限更新や、元解像度の上書きが起きないことを実機確認すること。

### 操作3: ページ画像 / 下絵を見ながらレンダー範囲を決める

想定操作:

1. コマ用blendファイルを開く。
2. B-MANGA が生成したページ画像 / 下絵_コマを確認する。
3. B-MANGA Render で出力する。

現在の呼び出し経路:

B-MANGA のコマ編集カメラ整備 → ページ画像生成 → カメラ背景画像へ設定 → レンダー範囲更新。

確認結果:

- 現行 B-MANGA のページ画像は管理情報付きの背景画像で、旧 `コマ枠.png` 固定ではない。
- B-MANGA Render からレンダー範囲を再計算する場合は、B-MANGA 管理下絵を読む必要がある。
- B-MANGA 管理下絵が無い旧シーンでは、旧 `コマ枠.png` 互換へフォールバックしてよい。ただし標準経路にはしない。

### 操作4: 旧 c00.blend を使う

想定操作:

1. 旧 c00.blend またはそれを基にしたコマ用blendファイルを開く。
2. 旧プリセット相当の出力を行う。

確認結果:

- 旧 c00.blend 互換は、現行 B-MANGA 標準とは別物として扱う必要がある。
- 標準プリセットを c00 に合わせ続けると、現行 B-MANGA 仕様への整合が崩れる。
- 旧互換を残す場合は、表示上も「旧出力シーン互換」と分かる分類へ分けること。

### 状態変化の必須確認

開発セッションでは、修正後に次を実機テストで確認する。

- プリセット実行後、透明背景、レンダーエンジン、ビューレイヤー、ノードミュート、カメラ設定が意図せず変わったまま残らない。
- B-MANGA 側の魚眼モード / 魚眼FOV / 縮小率 / 元解像度が、B-MANGA Render 実行後に別値へ戻されない。
- カメラの魚眼投影方式が、B-MANGA Render 実行後もユーザー指定のまま残る。
- `コマ枠` コレクション、`ページ` ノード、`全コマ統合` ノードが無い B-MANGA 標準コマ用blendファイルでも、標準プリセットが破綻しない。
- 旧 c00 互換プリセットを残す場合、旧互換側だけが `コマ枠` / `ページ` / `全コマ統合` に依存する。

## 4. 開発セッションでの推奨作業順

1. B-MANGA Render に B-MANGA コンテキスト読み取りヘルパを追加する。
   - B-MANGA が有効でなくても落ちないよう、Scene 属性とカスタム情報だけを見る。
   - 現在のコマ用blendファイル、現在コマの `passes/`、B-MANGA 管理下絵、魚眼設定を読み取る。
2. 魚眼設定を修正する。
   - B-MANGA 側の魚眼モード / 魚眼FOV / 投影方式を正とする。
   - eeVR へ渡す方式と FOV を B-MANGA 側カメラ設定へ一致させる。
3. 初期プリセットを B-MANGA 標準と旧互換に分ける。
   - B-MANGA 標準から `ページ` / `全コマ統合` / `コマ枠` 固定前提を外す。
   - 旧 c00 互換が必要な場合だけ、別カテゴリまたは別プリセットとして残す。
4. 出力先既定と画像名を B-MANGA のコマ単位へ寄せる。
5. 実機テストを追加 / 更新する。
6. マニュアルを現行 UI と B-MANGA 前提へ更新する。
