# B-MANGA Liner 明示更新方式 計画書 2026-07-07

## 目的

tokyo0004 級の大規模背景で、設定変更・プリセット適用・交差線オンなどの操作中に重い線生成が意図せず走り、保存もできず強制終了が必要になる状況を避ける。設定変更は軽い値変更に留め、線の生成・再生成はユーザーが明示的に更新した時だけ行う。

## このセッションのユーザー発言から抽出した設計意図

1. Pencil+4 が速い理由について、ユーザーは「B-MANGA Liner が外部アプリだから重い」と言ったのではなく、「Pencil+4 は外部または専用処理だから速いのでは」と確認した。
2. tokyo0004 のような大量オブジェクトの街背景で「レンダリング範囲内のみを選択」しても「ラインを適用」が 30 分以上終わらないのは実用不可。Pencil+4 の十秒程度に近づけたい。
3. 交差線をオンにしただけで重い処理が走る仕様は危険。交差線設定がいつ適用されるか明確にする必要がある。
4. 街背景ではオブジェクト位置がほぼ固定なので、一度作った線候補・交差候補は流用し、毎回重い検出をしない方がよい。
5. 固定しても線幅・カラー・中間頂点の線幅調整は変更できる必要がある。
6. カメラからの距離によって線幅が自動調整される必要があるため、線幅自体は固定してはいけない。
7. 交差線だけでなく、アウトライン・稜谷線・選択線も固定/再利用で軽くできるべき。
8. ラインプリセットの適用は、即座にオブジェクトへ線を生成するのではなく、各種設定だけへ反映するべき。
9. 設定変更のリアルタイム反映は基本なしにし、線種ごとの更新ボタンを押すまで反映しない方がよい。
10. 「レンダリング範囲内を選択」や「作成する距離」は有用。ただし範囲内にラインがある部分・ない部分が混在する場合、既存ラインを作り直さず二重作成も避け、足りない範囲だけ追加して以後流用する必要がある。
11. 保存データの肥大化は現時点では考慮しなくてよい。
12. 追加管理やキャッシュ判定が増えすぎて、逆に重くなることを避けたい。

## 設計意図としての妥当性チェック

### 妥当

- 「設定変更」と「線生成」を分離する方針は、大規模背景で固まらないために必須。
- 「線幅は固定しない」は、カメラ距離補正と印刷 mm 指定の既存仕様と整合する。
- 「固定するのは候補・形状・交差ペアなどの重い抽出結果で、見た目は更新可能」は、ユーザー要望と既存の線幅仕様を両立できる。
- 「ある範囲は再利用し、ない範囲だけ作る」は、レンダリング範囲選択・作成距離制限を実用上意味のある機能に保つために必要。
- 「保存データ肥大化は優先しない」は、速度と作業継続性を優先する今回の問題設定と整合する。
- 「管理コストが逆に重くならないようにする」は重要。毎回全検証するのではなく、設定変更時に未更新印を付け、明示更新時だけ差分作成する方針が最適。

### 注意点

- 既存の設計意図には「設定変更は選択中かつ生成済みラインだけを対象にして軽くする」という記載があるが、現在の実シーンでは「軽くする」だけでは足りない。設定変更は原則として線を触らない方が安全。
- 完全な「形状固定キャッシュ」は大きな構造変更になる。まずは即時再生成を止め、線種別更新ボタンと未更新表示を導入する。線候補キャッシュ・交差ペアキャッシュは次段階で拡張する。
- カメラ距離による線幅更新は、線生成とは別に「線幅を更新」として残す。これにより、線の作成結果は流用しつつ線幅だけをカメラ距離へ合わせられる。

## 実装方針

### 今回実装する範囲

1. 設定変更時は、選択中オブジェクトへ設定値だけ伝搬し、生成済みラインへの即時反映を基本停止する。
2. 設定変更後、対象オブジェクトに「未更新」印を付ける。
3. ラインプリセットの「選択中に適用」は、設定値だけを反映し、線生成や重い更新を実行しない。
4. パネルに「アウトラインを更新」「稜谷線を更新」「交差線を更新」「選択線を更新」を追加する。
5. 既存の「ラインを適用」は全線種更新として残す。
6. 線種別更新は、既存ラインがあるものは更新し、未作成で作成範囲内のものだけ追加する。範囲外は削除ではなく非表示/無効化寄りにする。ただし既存コードが削除前提の箇所は今回の最小改修では完全固定化せず、次段階でキャッシュ化する。
7. 「線幅を更新」は残し、カメラ距離による線幅再計算を明示操作として維持する。

### 今回あえて入れない範囲

- 交差ペアキャッシュの永続化。
- 内部線・交差線の抽出結果を独立データとして保存する完全固定モード。
- オブジェクト移動・メッシュ変更の自動変更検知。

これらは設計上は必要だが、今回の主目的である「設定操作中に固まらない」を先に満たす。キャッシュ永続化は次の段階で、測定しながら追加する。

## 実装手順

1. core に未更新印の保存・解除・判定ヘルパーを追加する。
2. core の主要設定コールバックを「設定伝搬 + 未更新印」へ切り替える。
3. 既存のライン適用処理を線種単位でも呼べるようにする。
4. operators に線種別更新オペレーターを追加する。
5. panels に線種別更新ボタンと未更新表示を追加する。
6. presets の「選択中に適用」を設定のみ反映へ変更する。
7. バージョンと CHANGELOG を更新する。
8. Python 構文チェックと tokyo0004 プロファイルで、プリセット適用・交差線オンが重処理を走らせないことを確認する。
