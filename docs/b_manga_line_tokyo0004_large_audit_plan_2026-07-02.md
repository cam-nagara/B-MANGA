## 目的

指定blendで、B-MANGA Lineのすべての画面操作、チェックボックス、スライダー、作成順、オン・オフ組み合わせを棚卸しし、正常性と軽さを確認する。
特に次を重点確認する。

- アウトライン、内部線、交差線のオン・オフ全組み合わせと切替順
- 「ラインを適用」「レンダリング範囲内を選択」「ラインを表示」「ラインを非表示」「ラインのみを表示」「通常表示に戻す」「ラインを削除」
- 交差線の新しい「ライン素材（高速）」方式が、大量オブジェクトで交差相手検出に戻らないこと
- アウトライン、内部線、交差線の中間頂点の線幅調整が目視で分かること
- 線幅、オフセット、色、作成範囲、表示範囲、カメラ補正、線幅均一化などの設定変更が生成済みラインだけを軽く更新すること

## 組み合わせ・順番マトリクス

### ライン種別の8状態

`O=アウトライン`, `I=内部線`, `X=交差線`

- `000`: すべてオフ
- `100`: Oのみ
- `010`: Iのみ
- `001`: Xのみ
- `110`: O+I
- `101`: O+X
- `011`: I+X
- `111`: O+I+X

### 24切替

上記8状態それぞれから、O/I/Xのいずれか1つを反転する。各切替で以下を記録する。

- 所要時間
- 新規生成数
- 削除数
- 既存ライン更新だけで済んだか
- 表示状態とモディファイア数

### 作成順

有効化順:

- O → I → X
- O → X → I
- I → O → X
- I → X → O
- X → O → I
- X → I → O

無効化順:

- O → I → X
- O → X → I
- I → O → X
- I → X → O
- X → O → I
- X → I → O

### 交差線方式切替

- ライン素材（高速） → Boolean（精密）
- ライン素材（高速） → SDF（高速）
- Boolean（精密） → ライン素材（高速）
- Boolean（精密） → SDF（高速）
- SDF（高速） → ライン素材（高速）
- SDF（高速） → Boolean（精密）

大規模全選択では「ライン素材（高速）」だけを全量検証する。Boolean/SDFは重い方式なので、指定blend内の代表10メッシュまでの小規模サンプルで正常性を確認し、全量適用は性能リスクとして実行しない。

## スライダー・チェックボックス代表値

連続値は無限に存在するため、実装分岐と境界値に基づく代表値で検証する。

- 線幅 (mm): 0.1 / 0.5 / 1.5
- オフセット: -1.0 / 0.0 / 1.0
- 色: 黒 / 赤 / 緑 / 青
- 検出角度: 20度 / 60度 / 120度
- 作成する距離: 0.5m / 10m / 20m
- 非表示にする距離: 0.5m / 20m
- 線幅基準距離: 0.5m / 2m / 10m
- 補正の影響度: 0 / 0.5 / 1
- 余白: 0度 / 10度 / 30度
- 中間頂点の線幅調整: -1 / 0 / 1
- 中間頂点の乱れ: 0% / 25% / 50%
- 変化グラフ: 0 / 0.5 / 1

## 性能判定

フェーズごとに以下をJSONへ記録する。

- 対象メッシュ数
- ライン適用済みメッシュ数
- 各ライン種別のモディファイア数
- 操作ごとの所要時間
- 1操作あたりの最大時間
- 交差線「ライン素材（高速）」で交差相手候補列挙が呼ばれたか
- 設定変更で全量再生成へ戻ったか

目標値:

- レンダリング範囲内選択: 20秒以内
- 全量ライン適用（レンダリング範囲内、O+I+X、交差線はライン素材）: 120秒以内
- O/I/Xの単独オン・オフ: 15秒以内
- 線幅・オフセット・色などの生成済みライン更新: 5秒以内
- 交差線ライン素材方式の候補列挙回数: 0回

超過した場合は失敗扱いにし、原因を切り分けて修正または明示的に記録する。

## フェーズ

### Phase 0: 計画書作成・計画書の徹底チェック

- この計画書を作成する
- 機能棚卸しと画面UIの整合性を照合する
- 既存テストと不足テストを対応付ける
- 計画漏れ、実行不能な組み合わせ、無限組み合わせの扱いを明記する
- 計画書コミット

### Phase 1: 指定blend読み込み・対象選択・基礎性能

- 指定blendを読み込む
- B-MANGA Lineを登録する
- カメラとレンダリング範囲内メッシュを検出する
- レンダリング範囲内を選択する
- baselineと選択状態のスクショを撮る
- フェーズ結果をコミット

### Phase 2: 全量ライン適用とO/I/X組み合わせ

- レンダリング範囲内メッシュで8状態を順に作る
- 24切替を実行する
- 6通りの有効化順と6通りの無効化順を実行する
- 各状態の代表スクショを撮る
- フェーズ結果をコミット

### Phase 3: スライダー・チェックボックス更新性能

- 全チェックボックス・スライダーを代表値で変更する
- 設定変更時に不要な再生成へ入らないか確認する
- 線幅、オフセット、色は生成済みラインだけが更新されることを確認する
- フェーズ結果をコミット

### Phase 4: 中間頂点・見た目重点スクショ

- アウトラインの中間頂点の線幅調整を強くした状態を撮影する
- 内部線の中間頂点の線幅調整を強くした状態を撮影する

- フェーズ結果をコミット

### Phase 5: 旧交差線方式・板ポリ・カメラ・プリセット・ラインのみ表示

- Boolean/SDFを代表10メッシュで確認する
- 板ポリ除外を確認する
- カメラ補正、カメラ範囲外非表示、遠距離非表示を確認する
- ウェイトを更新を確認する
- AOを焼き付けを代表メッシュで確認する
- AOVパスを追加を確認する
- リンク素材のラインを補正、リンク素材へ選択設定を上書きは、指定blendにリンク素材がある場合は実行し、ない場合はpoll結果と対象0件を記録する
- プリセット保存・適用・削除を確認する
- ラインのみ表示と通常表示復帰を確認する
- フェーズ結果をコミット

### Phase 6: 既存テスト再実行・CHANGELOG・最終コミット・プッシュ

- B-MANGA Line既存実機テストを再実行する

## 計画書の徹底チェック観点

- UIにある操作・項目が棚卸しに含まれているか
- O/I/Xの8状態と24切替が含まれているか
- 作成順と削除順が含まれているか
