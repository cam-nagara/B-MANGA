# B-MANGA Render UX レビュー

作成日: 2026-05-29
種別: レビュー文書（調査セッション / 実装は別途）
対象: `addons/b_manga_render`（panels / command_ui / operators / core / preset_library / command_runner）

## 0. 結論（先に要点）

現状の B-MANGA Render は「プリセット = コマンドの長い一列」で、**1プリセットに複数の出力処理（退避→各種切替→レンダー→復元）がフラットに詰め込まれている**。そのため:

- **把握**: 各コマンドが何をするか、どこからどこまでが1つの出力単位かが見えない。
- **設定**: 似たコマンド群を手で組み直すしかなく（複製なし）、まとまり単位の操作もできない。
- **構造**: 出力単位・プリセット間の関係（親子）を表現できない。

改善は「①コマンド単体の意味を見せる → ②出力単位のまとまりを見せる → ③プリセット間の構造を作る」の順で効く。①②は別計画書（用語改善・処理説明表示／親プリセット）と重なる。

## 1. 評価観点

- 把握しやすさ（このコマンド／このプリセットが何をするか読めるか）
- 設定しやすさ（作る・直す・並べ替える手数）
- 構造の見えやすさ（まとまり・順序・依存）

## 2. 現状の UI / 操作（要約）

- パネル構成（`panels.py` `draw_main_panel`）: 魚眼出力ボックス → プリセット一覧（カテゴリタブ + リスト） → コマンドリスト → 選択コマンド設定 → 「プリセットを実行」。
- コマンド一覧（`BMANGA_RENDER_UL_commands`）: 各行 = 有効チェック + 自動生成名（`command_ui.display_name` = 「種類ラベル: サマリー」）+ チェック状態アイコンのみ。
- 選択コマンド設定（`_draw_active_command_detail` → `command_ui.draw_command`）: 種類セレクタ + 種類別の prop 群。説明文は無し。
- 操作（`operators.py`）: 追加（種類選択ダイアログ）/ 削除 / 上下移動 / ダブルクリックで編集ダイアログ / 初期設定の登録・復元。**複製は無い**。
- プリセット = `BMangaRenderPreset.commands`（コマンドの並び）。カテゴリ（キャラ/背景等）は `preset_category_of` による**表示フィルタ**のみ。

## 3. UX 上の問題点

### A. 把握しづらさ

- **A-1【高】コマンド一覧で種類が区別できない**: 全行が同じチェックボックスアイコン。種類ごとのアイコン・色・インデントが無く、一覧が均質な文字列の羅列になる。
- **A-2【高】コマンドの処理内容が読めない**: 表示は「種類ラベル: サマリー」のみ。サマリーは「出力_キャラ / 出力」等で、何が起きるか（完全一致か部分一致か、何を有効化するか）が読み取れない。→ 別計画書（処理説明表示）で改善予定だが、現状は選択コマンド設定にも説明が無い。
- **A-3【高】用語が動作と不一致**: 「フレーム出力レンダー」「フレーム名」など。→ 別計画書（用語改善: 「レンダー：ワード検出」「検出ワード」）で対応予定。
- **A-4【高】出力単位の区切りが無い**: `STATE_BEGIN`〜`STATE_END` で挟まれた「1つの出力ブロック」が、一覧ではフラットに並ぶだけ。特に「キャラ_低速」等は数十コマンドが続き、どこが1出力か視覚的に分からない。
- **A-5【中】自動分岐の現在状態が見えない**: 魚眼/通常レンダー系コマンドは、魚眼モード ON で eeVR、OFF で通常レンダーへ分岐するが、コマンド設定欄に「今どちらで実行されるか・分岐先」が出ない（魚眼モード表示は魚眼ボックスに別途あるのみ）。

### B. 設定しづらさ

- **B-1【高】コマンド複製が無い**: 似たコマンド（同種・近い設定）を量産する手段がなく、新規追加 → 全項目再入力になる。出力ブロックの繰り返しが多い構成では特に苦痛。
- **B-2【高】出力ブロック単位の操作が無い**: 「退避〜復元」の1まとまりを、まとめて追加・複製・移動・有効無効できない。1枚ずつ動かすしかない。
- **B-3【中】コマンド追加時のガイド不足**: 種類を選んで追加した後、その種類に必要な項目・意味の案内が無い（説明表示が入れば緩和）。
- **B-4【中】長大プリセットの編集コスト**: 低速系は数十コマンド。`template_list` の表示行数は最大8で、スクロールと個別クリック編集が前提になる。

### C. 構造の欠如

- **C-1【高】プリセット間の関係を作れない**: 連続実行（親プリセット）が無い。→ 別計画書（親プリセット）で対応予定。
- **C-2【中】カテゴリは表示フィルタのみ**: 実行・構造には関与しない。ユーザーが「キャラ／背景＝親プリセット」と誤認する一因。

## 4. 改善提案（優先度マトリクス）

### すぐ効く・低コスト（①コマンド単体の意味を見せる）
- **P1 選択コマンド設定に処理説明の一文**（別計画書 `b_manga_render_card_term_and_help_plan`）。完全一致／部分一致、対象、分岐先を明示。
- **P2 用語改善**（同上）: 種類「レンダー：ワード検出」、項目「検出ワード」。
- **P3 コマンド一覧に種類別アイコン**: 種類→アイコンの対応表を作り、`BMANGA_RENDER_UL_commands.draw_item` のアイコンを種類で変える。
- **P4 無効コマンドの視覚的区別**: `enabled=False` の行をグレー表示（`row.active = command.enabled` 等）。

### 中コスト（②出力単位のまとまりを見せる）
- **P5 出力ブロックの視覚的区切り**: `STATE_BEGIN` の行の前にセパレータ/見出し（「── 出力: ○○ ──」）を挿入、ブロック内はインデント表示。一覧の `draw_item` で前後のコマンド種類を見て区切りを描く。
- **P6 自動分岐の現状表示**: 魚眼/通常レンダー系コマンドの設定欄に「現在: 魚眼モードON → eeVRで実行 / OFF → ワード検出レンダー」を出す（P1の説明表示と統合可）。一覧側にも分岐アイコンを付けると更に良い。
- **P7 コマンド複製ボタン**: `command_add` と並べて「複製」オペレーターを追加（選択コマンドを複製して直下に挿入）。

### 大コスト（③プリセット間の構造を作る）
- **P8 親プリセット**（別計画書 `b_manga_render_parent_preset_plan`）: 「プリセット実行」コマンドで子プリセットを並べる。構造化とまとめ実行。
- **P9 出力ブロックのテンプレート挿入**: 「1出力ブロック（退避〜レンダー〜復元）」をまとめて追加するクイック操作。手組みの主要因（B-1/B-2）を根本緩和。
- **P10 ブロック単位の折りたたみ表示**: プリセットを「出力ブロックのリスト」として2階層で見せ、ブロックを畳める。長大プリセットの俯瞰性を上げる。

## 5. 推奨ロードマップ

1. **第1段（理解の改善）**: P1・P2（別計画書）＋ P3・P4。コマンド単体が読めるようにする。低リスク。
2. **第2段（まとまりの可視化）**: P5・P6。出力単位と分岐が見える。一覧描画ロジックの拡張。
3. **第3段（操作性）**: P7・P9。複製とブロック挿入で手組みコストを下げる。
4. **第4段（構造）**: P8（親プリセット）・P10。プリセットの組み立てを構造化する。

## 7. 備考

- いずれの改善も、実行ロジック（`command_runner.py`）の意味は変えず、**見せ方・操作・構造の追加**が中心。既存プリセットの動作互換を保てる。
- 描画系の追加（P3〜P6, P10）は、`docs/B-MANGA_設計意図.md` の「描画中に不正なデータ書き込みでパネル描画を中断させない」原則を厳守（`getattr` 安全読み、ID書き込み禁止）。
- バージョンは PATCH +0.0.1 を積み上げる。
