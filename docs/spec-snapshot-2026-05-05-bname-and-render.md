## 2. B-Name 本体の実態

### 2.1 アドオン範囲

- B-Name 本体のバージョンは `0.3.1`
- Blender Extension ID は `b_name`
- B-Name 本体は、登録時に旧「書き出し」パネルと書き出しオペレータを外す処理を持つ
- 旧書き出し系のコードファイル自体は、現時点では残っている
- サムネイルや高品質プレビュー生成のため、内部的には画像出力系の処理を参照する箇所が残っている

### 2.2 ページ一覧ファイルの作画 UI

- ツールセクションには、オブジェクト、グリースペンシル、ラスター、線編集、レイヤー移動、フキダシ、テキスト、効果線のボタンが並ぶ
- グリースペンシルとラスターは、アクティブレイヤー種別によるグレーアウトではなく、常に操作ボタンとして表示される実装になっている
- ビューセクションには「選択ページ」数値入力がある
- 「ページ一覧ビュー」設定は、コマ編集モード以外の条件付きでまだ表示される
- 旧「ラスター描画」「GP描画」の独立セクションは、現在の Outliner レイヤーパネルにはない

### 2.3 レイヤー、アウトライナー、階層移動

- B-Name は Outliner 中心のレイヤー管理へ移行済み
- 新規レイヤー作成は、GP レイヤー、ラスターレイヤー、効果線、フキダシ再生成、画像再登録、テキスト再登録の入口を持つ
- Alt+ドラッグ、Alt+クリック、Alt+Shift+クリックによるページ / コマへの階層移動オペレータが存在する
- Alt+ドラッグではドロップ位置の表示、選択レイヤー数のプレビュー、確定 / エラー表示のオーバーレイを使う
- Ctrl+Alt+ドラッグはブラシサイズ変更用として Alt+ドラッグ階層移動から除外される

### 2.4 用紙、ラスター、コママスク

- 用紙背景はページごとの実メッシュとして作られ、対応するページコレクション直下へ置かれる
- 用紙色は用紙設定に追従するマテリアルで表現される
- ラスター表示との重なり対策として、用紙背景を不透明、ラスターを上に重ねる構成を採っている
- コマ用のマスク / 背景平面は独立ファイル群として実装されている
- コマ形状変更とコママスク同期の実機テストが存在する

### 2.5 コマ用blendファイル

- プリファレンスには「コマ用blendファイル (共通)」の設定がある
- 作品情報側のコマ用blendファイルが空の場合、共通設定を使う設計になっている
- コマファイル側では、下絵表示とカメラ背景画像を扱う関連処理がある

## 3. B-Name-Render の実態

### 3.1 アドオン構成

- B-Name-Render は `addons/b_name_render/` 配下の独立アドオンとして存在する
- Blender Extension ID は `b_name_render`
- 表示名は `B-Name-Render`
- バージョンは `0.1.1`
- ライセンスは `GPL-3.0-or-later`
- B-Name 本体とは別 manifest を持つ

### 3.2 データモデル

B-Name-Render は、シーンにプリセット状態を持つ。

- プリセット
- カード型コマンド
- アクティブプリセット
- アクティブカード
- 前回クリックカードと時刻

カードの種類は次の範囲を持つ。

- 出力状態を退避して初期化
- 出力状態を復元
- ビューレイヤー
- コレクション除外
- ノードミュート
- ファイル出力切替
- AOV入力
- 出力画像名
- 出力フォルダ
- 画像ノード再読み込み
- レンダー
- フレーム出力レンダー
- eeVR設定
- eeVR魚眼レンダー
- eeVR方向画像レンダー
- eeVRパノラマ合成
- Blenderオペレータ

### 3.3 UI

- 3D ビューのサイドバーに B-Name-Render パネルがある
- プリセット一覧を表示する
- プリセット追加、削除、初期プリセット読み込み、プリセット実行ができる
- カード一覧は、番号、カード名、有効 / 無効、カード種別を表示する
- カード一覧右に、追加、削除、上移動、下移動のボタンがある
- カードはクリックで選択される
- 同じカードを短時間に 2 回クリックすると設定ダイアログを開く

初期プリセットは 37 件存在する。

「すべて」「効果」「ページ」は、実行可能なカード列として定義済み。

### 3.5 実行エンジン

- プリセットは上から順番にカードを実行する
- 出力状態退避では、透明背景、レンダーエンジン、ビューレイヤーの使用状態、ファイル出力ノードのミュート状態を退避する
- 終了時は、退避した状態を復元する
- ビューレイヤー、コレクション除外、ノードミュート、ファイル出力ノード、AOV入力、出力名、出力フォルダ、画像再読み込み、レンダー実行を扱う
- Blender 5.1 のコンポジター構造を参照して出力ノードを探す
- Pencil+4 系カードは、魚眼モード時は eeVR、通常モード時は通常レンダーへ分岐する
- eeVR 系カードは、eeVR のプロパティとオペレータが存在する場合に呼び出す橋渡しになっている
