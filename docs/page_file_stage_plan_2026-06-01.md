# ページ用blend分離計画 (2026-06-01)

## 目的

55ページ以上の標準的な作品で、ページ一覧を開くだけで全ページ分のフキダシ・テキスト・効果線・ラスターが読み込まれ、操作全体が重くなる問題を解消する。

## 方針

- `work.blend` はページ一覧専用にする。
  - ページの並び、用紙設定、ページプレビュー画像を持つ。
  - コマ面、コマ枠線、コマ枠編集用の実体オブジェクトは持たない。
  - フキダシ、テキスト、効果線、ラスター、画像、描画レイヤーの実体は持たない。
- 各ページの編集内容は `pNNNN/page.blend` に置く。
  - ページ用blendには、そのページの中身だけを生成・保存する。
  - 表示上は、他ページを25%サイズの軽いページプレビュー画像として表示する。
  - ページプレビューはビューセクションの「ページ一覧表示」でオン/オフできる。
  - ページ用blendからコマ用blendを開いた場合は、戻り先も同じページ用blendにする。
- 過去ファイルとの互換は扱わない。
  - 新規作品と今後保存される作品構造を正とする。

## 実装対象

- パスと保存読み込み:
  - `pNNNN/page.blend` のパス生成、保存、オープン、存在確認を追加。
- モード状態:
  - ページ一覧表示とページ編集表示を区別する。
  - ページ編集表示では現在のページを保持し、ページの座標は原点扱いにする。
- ページプレビュー:
  - ページ用blendでは現在ページ周辺の25%プレビュー画像を生成し、編集対象外ページの実体は持たない。
  - プレビュー対象は「前後ページ数」で調整できる。
  - プレビュー画像の解像度は「画像解像度%」で調整でき、初期値は25%にする。
  - プレビュー画像上の別ページをダブルクリックすると、そのページ用blendへ移動する。
- 同期:
  - ページ一覧ファイルでは中身レイヤーを生成しない。
  - ページ用blendでは選択ページの中身だけを生成する。
  - 保存前後の同期も現在のファイル種別に従う。
- UI:
  - ページリスト右側に「ページを開く」ボタンを追加。
  - ページ用blendからページ一覧に戻るボタンを追加。
- コマ用blend遷移:
  - ページ用blendからコマ用blendを開いた場合、戻る先をページ用blendにする。

## 検証

- 新規作品で `work.blend` と `p0001/page.blend` が分離される。
- ページ一覧ファイルに中身レイヤーの実体が保存されない。
- ページ用blendには対象ページの中身だけが保存される。
- ページ用blendからコマ用blendへ入り、戻ると同じページ用blendに戻る。
- 55ページ作品のページ一覧を開いたとき、フキダシ・効果線等の全ページ実体が読み込まれない。

## 実装結果

- v0.6.230 で実装済み。
- ページ一覧のページリストからページ用blendを開く。
- ページ用blendが未作成の場合は、作品情報から対象ページだけの編集ファイルを自動作成する。
- ページ一覧ではページの並びとプレビューを扱い、コマ枠を含む編集実体はページ用blendに保存する。
- v0.6.231 で、ページ用blend内の他ページ表示を25%サイズの軽いページプレビュー画像へ変更した。
- ページ用blendでは対象ページだけを実体として保持し、他ページの実体は持たない。
- コマ枠の選択・編集に使う実体は、現在編集中のページ用blendだけに存在させる。作品ファイルと非編集中ページはページプレビュー画像のみで、コマ枠実体を持たない。
- ビューセクションの「ページ一覧表示」でプレビュー画像をオン/オフできる。
- 別ページのプレビュー画像をダブルクリックして、そのページ編集へ移動できる。
- v0.6.232 で、ページプレビューの前後ページ数と画像解像度をビューセクションから変更できるようにした。
- コマ用blendから戻る時は、ページ用blendが存在する場合は同じページへ戻る。
