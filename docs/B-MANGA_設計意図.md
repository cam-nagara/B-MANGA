# B-MANGA（Blender ネーム作画アドオン） 設計意図

> 本ファイルは「仕様定点観測」用の設計意図ファイル。ユーザーの指示・希望を粒度を丸めずに記録する。実態コードとの照合はユーザー指示時のみ実施する。

## 0.11 2026-06-11 時点の追加設計意図（競合時はこの節を最優先）

> 本節は 2026-05-25〜2026-06-11 の ClaudeCode / Codex 全セッションのユーザー発言を機械抽出（444 件）して記録したもの。以前の節（0.10 以下）と矛盾する場合は本節を優先する。

### 0.11.1 フキダシの線は「面としての線」に統一する

- フキダシ・コマ枠を含むすべての線の生成は「面としての線」（塗りつぶした帯メッシュ）方式に統一する。パスに沿った線描画や輪郭オフセット方式は、線同士の角度や形状の複雑さで太さが変わるため不可。
- 線幅は UI で指定した値（mm）どおり、見た目上一定でなければならない。数値的な完全一致よりも「突然曲がったりせず滑らか」であることを優先する。
- 谷部分などの鋭角は、本体カーブから拡張（オフセット）する方式では原理的に破綻する。拡張後の形状そのものを独立に算出して生成する。
- 制御点ごとの線幅編集は必要（楕円に限らない）。
- 自己交差問題は主線・フチ・多重線のいずれでも解決必須。
- 半透明（アルファ）時に、線の交差部分が濃く見えてはいけない。多重線の交差部分を透明にして消すのも不可。
- 雲形状で線幅を太くした時、谷にトゲ状の突起が発生してはいけない。
- 多重線の谷の鋭角部分は鋭角に保つ。

### 0.11.2 フキダシ構成要素の重なり順

- 一番下に塗り、その上に外側フチ・内側フチ、その上に主線・多重線の順で重ねる。
- 効果線の白抜き線は主線の下に重ねる。
- フキダシ同士を重ねた時、すべてのフキダシが同じ高さに存在して線だけ隠れない状態は不可。下のフキダシの線が上のフキダシの塗り領域に透けて表示されるのも不可。

### 0.11.3 主線・フチ・多重線の山/谷線幅と長さ変化

- 「主線・山の線幅」「主線・谷の線幅」「（多重線の）山の線幅」「谷の線幅」は %指定（0〜100%、ただの線幅が 100%）。0% 指定では線がフェードアウトして消えるくらいの細さになる（100% に戻ってはいけない）。
- 線幅 % はその頂点を含む辺全体を影響範囲とし、谷から山にかけて滑らかに変化させる（山 0%・谷 100% なら谷→山で 100%→0%）。
- 山か谷どちらかだけを細くした時、太いままの側の太さは元の太さと完全に同値を保つ。山から谷へ向かう直線のアウトラインを途中で曲げない。
- 山の線幅を変えた時に谷の尖りが消える / 谷の線幅を変えた時に山の尖りが弱くなる、はいずれも不可。それぞれの尖りは保つ。
- 「長さ変化」は谷の頂点を基準とし、多重線は常に谷から伸び、短くなる場合は山側の終点が谷へ近づく形にする。主線に一番近い線ほど変化が小さく、遠い線ほど大きい。一番近い線と一番遠い線の長さを別々に % 指定できる。「角を尖らせる」で伸びた分にも対応する。
- 低い山でも高い山と同じ比率で多重線が細く・短くなる（低い山の頂上で線が繋がって途切れないのは不可）。
- 「長さ変化」「谷の線幅」「山の線幅」「間隔変化」は、トゲ（直線）だけでなく雲・もやもや・トゲ（曲線）にも適用する。
- 雲はカーブ上に頂点が無くても山の先に頂点があるものとして扱う。トゲ（曲線）は谷の底と小山の先にも頂点があると仮定する。
- 多重線は「線の本数」どおりの本数を生成する（1 本欠けは不可）。主線と一番近い多重線の間隔は多重線間隔と等しくする。
- 多重線の外側は、主線の線幅の外側のアウトラインを基準にする（主線を太くしても外側多重線が隠れない）。
- 多重線は小山にも適用する。
- フチは主線の線幅変化で変わったフキダシのアウトラインに沿って付ける（フチ自体の太さは主線に連動して細くしない）。

### 0.11.4 「角を尖らせる」と形状ニュアンス

- 「谷を尖らせる」は「角を尖らせる」へ拡張し、山も谷も、雲以外の形状にも適用する。尖りは直線の尖りではなく曲線と曲線の交点としての角。線幅に関わらず先端は尖り、平らにならない。
- 「角を尖らせる」は形状パラメータセクションの一番下に配置する。詳細設定ダイアログから消えてはいけない。
- もやもやの本体形状はトゲ飛沫状が正で、参考画像（ユーザー提供）を正とする。細かい形状ニュアンスは言葉ではなく参考画像との一致で判定する。
- もやもやで小山・乱れが機能しなくなるのは不可。

### 0.11.5 フキダシの線種・角の種類

- 破線・点線は実線と区別がつく描画にする。破線は線分と間隔、点線は間隔を設定できる。点線の点は真円にする。
- フキダシも角を「直角・丸角・面取り」から選択可能にする。
- 角丸の角半径は mm に加えて % 指定もできるようにする。
- フキダシの線種に「ウニフラ」「白抜き線」を追加する（形状ではなく線種）。設定項目は効果線の集中線と過不足なく同じにする。初期値は入り抜き両方 0%、中間線幅 100%。ベース形状はトゲではなく楕円（効果線の集中線と同じ方式）。
- フキダシのウニフラは黒線・白線・下地の 3 層とし、黒線と下地の間に白線を挟む。白線は黒線と同じ設定項目を持ち、黒線を 100% とした % で指定する。
- ウニフラ・白抜き線では効果線と同様に中心点を表示する。
- ウニフラ・白抜き線は処理が重くならないよう軽快・快速にする。

### 0.11.6 効果線の白抜き線

- 白抜き線は「抜きのある白線が何本も放射状に並んだ」線。集中線等と同じ共通の始点形状・終点形状・中心点に従う。
- 白線の本数、間隔、入りと抜き、入り抜きそれぞれの長さ（% または長さ指定）を設定できる。
- 黒線はフキダシの多重線と同じだけの設定項目を持つ。
- 白抜き線もリンク複製可能にする。
- 重なり順は主線の下。

### 0.11.7 フキダシのしっぽ・結合

- しっぽはフキダシに結合した一体の表示にする（しっぽとフキダシの境界線を残さない）。
- しっぽは外向きだけでなく内向き（フキダシをえぐる向き）にもできる。えぐった領域やしっぽとフキダシの間に下地色がはみ出てはいけない。
- フキダシ同士の結合もしっぽ結合と同様の方式にする。結合中のフキダシを移動しても、下地の残影が元の位置に残ってはいけない。
- 結合・しっぽの操作は軽快・快速にする。

### 0.11.8 自由変形・拡大縮小回転・リンク複製

- フキダシ・テキスト・効果線はハンドルの頂点を Ctrl+ドラッグで自由変形できる（ラティス変形相当）。すべての形状・設定パターンで破綻しないこと。
- 右クリックメニューから自由変形をリセットできる。
- 右クリックメニューから拡大・縮小・回転ができる（自由変形の拡大縮小回転版）。拡大しても形状は一切変わらず、そのままの形状で拡大縮小される。
- その際「線幅を維持」オプションを選べる。オンならどれだけ拡大縮小しても線幅は元の太さを維持、オフなら拡大で太く・縮小で細くなる。
- 複製・リンク複製した時、複製先レイヤーの位置をズラさない。
- リンク複製では位置・角度・中心点・形状線・自由変形を複製元と共有し、それ以外の設定は別々に変更できる。リンク複製に限らず、複数レイヤーを後からリンクさせた場合も同様に共有する。
- リンク状態はレイヤーリストに表示する。
- リンク複製が重いのは不可。

### 0.11.9 テキスト

- テキスト入力中、入力テキストに合わせてハンドル形状がリアルタイムに変化する。テキスト側がハンドル形状に合わされて勝手に改行されるのは不可。
- テキストのハンドルサイズは内包するテキストに合わせた状態で固定し、ハンドルドラッグでは変えられない。
- テキストとハンドルの間には視認できる隙間を空ける。
- Shift+Enter で改行した時、テキストの位置が動いてはいけない。
- テキスト入力中はキャレットを表示する。ドラッグでテキストを選択でき、選択ハイライトが表示される。細かくアンドゥ・リドゥできる。
- 太字・斜体・フォントサイズ・色の変更で文字列がダブって重なったり、その後消えたりしてはいけない。
- オブジェクトツールでテキストをダブルクリックするとテキスト編集状態になる。
- テキストレイヤーの移動は軽量・快速であること（画像方式で軽量化済みの状態を維持）。
- テキストツールはルビ・改行・選択・移動・D&D・カーニング等の一般的機能を含め、直感的かつ快速であることを UX 基準とする。

### 0.11.10 作品ファイル / ページファイル / コマファイルの三段構成

- 55p〜80p 級の作品で重くならないことが必須要件（55p は新連載第一話の標準的なページ数であり極端ではない）。
- そのため作品ファイル（ページ一覧）とコマファイルの間に「ページファイル」段階を設ける。過去ファイルとの互換性は不要。
- フキダシ・効果線・テキスト・ラスター等のページ内容の編集はページファイルで行う。コマ枠の実体は編集中のページファイルにだけ存在し、作品ファイルや他ページでは選択も編集も不可（実体そのものを持たない）。
- ページファイルでは、編集中ページの上下左右に、ページ一覧と同じ配置・同じ大きさで他ページのプレビュー画像を表示する（ページ一覧の中の該当ページをズームして編集している状態に見えること）。
- 他ページのプレビューをダブルクリックすると、そのページのページファイルが開く。
- 作品ファイルの全ページ一覧表示でも、ページをダブルクリックでそのページファイルが開く。ページリストの各ページカードに「開く」ボタンを置く。
- プレビューは統合画像で表示し、全オブジェクトを直接表示しない（軽さ優先）。プレビューには用紙も含める。プレビューしない設定のページに作品情報だけ表示されるのは不可。
- プレビュー範囲（現在ページの前後何ページ分か）を設定できる。プレビュー解像度はプリファレンスで設定でき、初期値は 25%。
- ビューセクションでページ一覧プレビューのオン/オフを切り替えられる。ビューセクションの設定はページごとに変えず、別のページを開いても引き継ぐ。
- ページファイルを開いた時は「ページに合わせる」状態、B-MANGA パネルが開いた状態、レンダーモードにする。
- コマファイルの変更・ページファイルの変更は、戻った時にプレビュー画像へ反映される。
- ページファイルのパネルでは作品セクション・用紙セクション・ページリストを表示しない（ページリストは作品ファイルのみ）。
- 「ページ一覧に戻る」「ページに戻る」ボタンは、一番上の「ファイル遷移」セクションに置く。
- 枠線カットツール等のコマ割り操作はページファイルでのみ使え、作品ファイル（ページ一覧）では使えない。
- ページの追加・並べ替えで他ページの内容（フキダシの位置等）に影響を与えない。ページ追加が遅いのも不可。
- ページ数・コマ数の変更（入れ替え・再採番・枠線カット）時は、実データのフォルダ名・ファイル名も追従して整理される（コマファイルを開く際に実データ名の整理を自動実行）。

### 0.11.11 ページリストとレイヤーリスト

- レイヤーリストとは別に「ページリスト」を置く（作品ファイルのパネルのみ）。右側のボタンでページの並べ替え・追加・複製・削除ができる。
- ページリストで選択したページ内のレイヤーがレイヤーリストに表示される。
- ページのレイヤー名は常にページ数どおりに自動リネームされる。レイヤーリストにはページ行を表示しない。
- レイヤー間に空行を作らない。操作ボタン群はリストの下ではなく右側に置く。
- ページリスト・レイヤーリストともアイコンとレイヤー名は左寄せにし、表示/非表示アイコンとレイヤー名を離しすぎない。
- レイヤーリストの高さは内容より小さく縮められる（伸縮ハンドルを失わない）。
- レイヤーリスト内の D&D 移動機能とそのハンドルは置かない（D&D は成立しなかったため撤去）。
- ページとコマのレイヤー名は編集不可。それ以外のレイヤーはレイヤー名をダブルクリックでインライン編集できる。
- レイヤーリストでコマの番号数値を直接修正できる。数値を変えても重なり順やリスト上の並び順は変えない。
- レイヤーをクリックしたら選択ハンドルは完全に切り替わる（直前の選択のハンドルが残らない）。
- 3D ビューポート上でもレイヤーリストでも、Ctrl / Shift で複数レイヤーを選択できる。
- フキダシ結合のレイヤーフォルダは非表示にでき、折りたたみできる。
- 枠線カットでコマが分かれた時、読む方向で前のコマが小さい番号になる（読む方向が左なら右上→左下、右なら左上→右下）。左下のコマほどレイヤーリストで下に並ぶ。

### 0.11.12 コマの詳細設定ダイアログ

- 「表示名」欄と「表示」チェックボックスは置かない（コマのレイヤー名は変更できなくてよい）。
- 「空の時は作品/プリファレンスの設定が使われる」等の説明文、「形状：」「頂点数：」のラベル、「（Enter=確定/ESC＝キャンセル/緑線＝スナップ）」「（負値は作品共通ルールを継承）」の文言は表示しない（ツールチップに置く）。
- 「用紙」「用紙色」は「背景」「背景色」と呼ぶ。
- 「上下（個別）」「左右（個別）」「自動くり抜き」は使わないため置かない。
- 「白フチ」は「フチ」と呼び、フキダシのフチのように内側・外側・両側を選べる。枠線のフチも外側・内側で別々に色を設定できる。
- 線種・線幅・線色などは二列・三列レイアウトでまとめる。
- 枠線プリセットは改名・追加・複製・削除・並べ替えができ、ドロップダウンからの切り替えが常に機能する。プリセットのボタン群は右寄せ。
- コマ枠の白フチ（フチ）の幅は実際に描画される幅が設定値どおりであること（枠線の中心線基準で細くなるのは不可）。
- 丸角はカクカクして見えない解像度で描画する。
- 山の高さの最大値は 100。
- コマファイルを編集してページ側に戻った時、コマ枠のプリセットやその他の設定がリセットされてはいけない。

### 0.11.13 パネル整理（ページ一覧 / ページ / コマファイル）

- 「オーバーレイ表示」切り替えはビューセクションの一番上に置く。
- 「B-MANGA 階層を修復」とマスク関連の項目は「メンテナンス」セクションにまとめる。メンテナンスの各ボタンは、それが意味を持つファイル種別のパネルにだけ表示する（例: コマ ID 再採番はコマを割れるページファイルのみ）。
- コマ ID の再採番はページ上の読む順番どおりにする。
- コマファイルのパネルに「B-MANGA 操作」チェックボックスを置かない（コマファイルでは B-MANGA 固有ショートカット不要、Blender 標準ショートカット W / Shift+A / S / G 等を妨げない）。
- コマファイルを開いた時、アクティブなビューレイヤーはコピー元ファイルのまま維持する（勝手に「コマ枠」ビューレイヤーへ切り替えない）。
- コマファイルのパネルから出力解像度とそのリストは削除する（用紙設定から算出されるため不要）。
- コマファイルの「コマ編集：カメラ」は「カメラ設定」に改名する。魚眼モード・魚眼FOVは B-MANGA のコマファイル側へ戻し、「カメラ設定」に置く。
- ページ一覧表示・前後ページ数・列数・横間隔・縦間隔・ページ一覧不透明度・ページ画像のスケール・コマ内レイヤー不透明度・グレースケール表示・背景を透過・ワールド背景色・ソリッド背景色・サブディビジョンサーフェス・コマを後ろにする・ハッチング間隔を表示・ハッチング回転・ビューを更新は、コマファイルの「ビュー」セクションにまとめる。
- 縮小モード・Pencil+4 線幅保存は B-MANGA Render 側に置く。B-MANGA Render 側には魚眼モード・魚眼FOV・ページ画像のスケールの編集欄を置かない。

### 0.12.3 表示制御の階層

表示制御は以下の階層で判定される（上位がOFFなら下位は評価しない）:

1. **全体トグル**: `bmanga_overlay_enabled` — OFFで全オーバーレイ非表示
2. **カテゴリトグル**:
   - 用紙ガイド: 作品ファイルでは `paper.show_guides`、ページ/コマファイルでは `bmanga_page_guides_visible`
   - 作品情報: 作品ファイルでは `work_info.display_visible`、ページ/コマファイルでは `bmanga_page_work_info_visible`
   - ページプレビュー: `bmanga_page_preview_enabled`（ページファイルのみUIに表示）
   - コマ内レイヤー: `bmanga_coma_content_visible`（コマファイルのみUIに表示）
3. **個別要素トグル**:
   - ガイド線: `paper.show_canvas_frame`, `paper.show_bleed_frame`, `paper.show_finish_frame`, `paper.show_inner_frame`, `paper.show_safe_line`, `paper.show_trim_marks`
   - セーフ外塗り: `safe_area_overlay.enabled` + 不透明度
   - 断ち切り外塗り: `safe_area_overlay.bleed_outer_enabled` + 不透明度
   - 作品情報各項目: 各`item.enabled`
4. **ページ/エントリ単位**: `page_range.page_visible_in_work`, LayerCollection表示状態, エントリの`visible`

### 0.12.4 GPUオーバーレイ描画の技術仕様

- 用紙白背景は `depth_mask=True`, `depth_test=LESS_EQUAL` で描画（ラスターMeshの手前に出ないため）
- ガイド線・セーフ外塗り・断ち切り外塗りは `depth_test=NONE` で描画（全ページでdepth bufferに依存せず確実に表示）
- ページプレビュー画像は `depth_test=NONE` で描画
- page_preview Meshオブジェクトは常に `hide_viewport=True`（GPUオーバーレイ移行済みのため実体不要）
- ガイド線の太さはビュー一定（ズームに追従して画面上の太さが一定に見える）
- 作品情報テキスト・ページ番号はPOST_PIXEL（blf）で描画。depth testの影響を受けない

### 0.12.4a コマ背景・枠線・白フチの描画責任

**コマの背景色・枠線・白フチは実体Meshオブジェクト（coma_plane / coma_border / coma_white_margin）で描画する。オーバーレイ側では描画しない。**

理由:
- オーバーレイで `depth_test=NONE` のポリゴン塗りを行うと、先に描画されたページプレビュー画像を上書きしてプレビューが見えなくなる
- 描画順序（プレビュー → コマ塗り）で後から描くオーバーレイが前面に出てしまい、ページ一覧プレビューが表示されない
- 実体オブジェクトは Blender の depth buffer で正しく前後関係を管理でき、マテリアルも per-coma で制御できる

オーバーレイ側のコマ関連描画は以下に限定する:
- **選択中コマの補助線**: アクティブコマの枠を `SELECTION_STRONG` 色でハイライト表示（`_draw_comas` 内）
- **コマ辺選択ハイライト**: コマ辺の選択・編集UI（`overlay_coma_selection.py`）

## 0.13 2026-06-26 ページファイル/コマファイルのページ一覧プレビュー（正本）

> 0.12 節と矛盾する場合は、本節を優先する。ページファイルとコマファイルでは、ページ一覧プレビュー自体に「開いた時点で確認できるページ状態」を含める。

### 0.13.1 ページファイルを開いた直後の表示

- ページファイルを開いた直後、編集中ページのコマ枠線はレイヤーリストに存在するだけでなく、3D ビューポート上にも必ず表示される。

- 用紙ガイドや作品情報を表示しない設定にした時は、ページ一覧プレビュー画像もその設定に追従する。

### 0.13.2 コマファイルを開いた直後の表示

- コマファイルを開いた時のページ一覧プレビューは、カメラ下絵として表示する場合でも、ページ一覧と同じ配置・同じ大きさを保つ。見開きページが下へずれたり、現在ページとの上下位置が崩れたりしてはいけない。
- コマファイルのページ一覧プレビューにも、ページ上部のページ番号、作品情報、用紙ガイド、セーフライン外の塗り、断ち落とし枠外の塗りを含める。
- コマファイルを開いた時、カメラのビューポート表示の「外枠」はオフにする。外枠の濃さも 0 とし、開くたびに不要な暗い外枠が戻ってはいけない。

### 0.13.3 プレビュー画像の使い分け

- 作品ファイルのページ一覧で使う軽量プレビューと、ページファイル/コマファイルで使う確認用プレビューは用途が異なる。

- ページファイルまたはコマファイルへ移動した時は、ページ一覧プレビュー画像を必要に応じて再生成し、保存済みの古い画像だけに依存しない。

## 0.10 2026-05-25 時点の追加設計意図（競合時はこの節を最優先）

> 本節は 2026-05-25 の Codex セッションでのユーザー指示・希望を記録したもの。以前の節（0.9 以下）と矛盾する場合は本節を優先する。実装計画は [`docs/coma_content_opacity_mask_plan_2026-05-25.md`](coma_content_opacity_mask_plan_2026-05-25.md) に置く。

### 0.10.1 コマ内要素は切断せず、不透明度マスクで見切る

- コマ内のフキダシ、効果線、グリースペンシル、ラスター、画像、テキストなどは、コマ境界で形状を壊して切断しない。
- コマ外へはみ出す部分は、コマごとの不透明度マスクで透明化する。
- 輪郭ぼかしのあるコマでは、フキダシ、効果線、グリースペンシル、ラスターなども同じぼかし境界でフェードする。
- カーブやメッシュの制御点、グリースペンシルの描線、ラスター画像の元データを、マスク適用のために破壊しない。

### 0.10.2 マスクはコマ単位で、ページとコマを識別できる名前にする

- コマ内容マスクはコマごとに 1 つを正本にする。
- マスク名には、コマ ID だけでなくページ ID と現在のページ番号も含める。
- ページ入れ替えで表示ページ番号が変わっても内部参照が壊れないよう、内部ではページ ID とコマ ID を正本にする。
- ページ入れ替え後は、表示名や保存名に含まれるページ番号を現在のページ番号へ揃える。

### 0.10.3 所属が変わったらマスクも変わる

- フキダシ、効果線、グリースペンシル、ラスター、画像、テキストなどを別コマへ移動したら、新しいコマのマスクを使う。
- 別ページのコマへ移動したら、新しいページとコマのマスクを使う。
- ページ直下やページ外へ移動したら、コマ内容マスクを解除する。
- 所属は座標だけで推測せず、レイヤーリストやアウトライナー上の親子関係を正とする。

### 0.10.4 PSD はコマフォルダのレイヤーマスクを正とする

- PSD 出力では、コマごとのレイヤーフォルダにコマ内容マスクをレイヤーマスクとして付ける。
- フキダシ、効果線、グリースペンシル、ラスターなどの個別レイヤーには、同じコマ内容マスクを重複して付けない。
- ページ直下やページ外の要素はコマフォルダ外に出し、コマ内容マスクを付けない。

### 0.10.5 コマ枠線、用紙ガイド線、セーフライン外塗りは対象外

- コマ枠線はコマ内要素より前面に表示し、コマ内容マスクで透明化しない。
- 用紙ガイド線とセーフライン外塗りは、コマ内容マスクの対象にしない。
- 別コマの輪郭ぼかしやマスクが、他コマの効果線やフキダシの色、不透明度、線幅を変えてはいけない。

## 0.9 2026-05-23 時点の追加設計意図（競合時はこの節を最優先）

> 本節は 2026-05-23 の Codex セッションでのユーザー指示・希望を記録したもの。以前の節（0.8 以下）と矛盾する場合は本節を優先する。実装計画は [`docs/balloon_curve_source_plan_2026-05-23.md`](balloon_curve_source_plan_2026-05-23.md) に置く。

### 0.9.1 フキダシの正本は編集可能な実体にする

- フキダシは、B-MANGA が画面上で描くだけのものや、B-MANGA が重い処理で毎回全面生成するものではなく、Blender 上で編集できる実体を正本にする。
- B-MANGA が無効な状態でも、保存済みファイルだけでフキダシの輪郭、塗り、不透明度、グラデーションがレンダリングできる必要がある。
- フキダシの外形は、制御点を編集できるカーブを基本にする。
- 1 つのフキダシは、ユーザーから見て 1 つのレイヤー、1 つの要素として扱えることを優先する。

### 0.9.2 B-MANGA は作成と操作、Blender 実体が表示とレンダリングを担う

- B-MANGA は、フキダシの新規作成、詳細設定、変換、再生成、整合性確認を行う。
- 確定済みのフキダシ本体は、B-MANGA のオーバーレイ描画だけに依存しない。
- 雲、もやもや、トゲなどの形状は、B-MANGA が作成時や明示的な再生成時に制御点列として作る。
- フキダシ用のジオメトリノードを使う場合でも、塗り、線、グラデーション、不透明度などの軽い補助処理に限定する。
- 雲、もやもや、トゲなどの外形生成を、フキダシ用の巨大なジオメトリノードで常時行わない。
- ページ直下のフキダシはページ全体の作画要素として扱い、ページマスクを不要にする。コマ内のフキダシだけ、コマの範囲に従う。

### 0.9.3 Blender 標準編集後の扱い

- Blender 標準機能でフキダシの制御点を直接編集した場合、次に B-MANGA で編集する時は、その編集後の形を正本として扱う。
- 制御点を直接編集していない限り、雲やトゲなどは B-MANGA の詳細設定から形状パラメータを変更できる。
- 制御点を直接編集した後は、B-MANGA が黙って元の雲やトゲ形状へ戻してはいけない。
- 直接編集後に雲やトゲとして再生成する場合、制御点数、順序、開閉状態が変わっていない時だけ、手編集の差分を再適用する。
- 制御点数や順序が変わっている場合は、黙って上書きせず、自由形状として扱うか、手編集を破棄する明示操作を要求する。

### 0.9.4 パス、グリースペンシル、自由形状

- Blender 標準のパスやグリースペンシルで作成した形状を、フキダシとして登録できるようにする。
- 登録された形状は、まず自由形状として扱う。
- カーブとして登録した自由形状は、登録後のカーブを正本として扱い、B-MANGA が元の楕円や雲へ黙って戻さない。
- 自由形状でも、線、塗り、不透明度、グラデーション、しっぽなどの共通設定は編集できる。
- 自由形状から雲やトゲなどへ戻す場合は、ユーザーの明示操作で再生成する。

### 0.9.5 効果線との関係

- 効果線は大量の線を扱うため、フキダシとは別に軽量な専用方式を使う。
- ただし、効果線の始点形状と終点形状については、フキダシと同じく編集可能な形状を正本にする方針を共有する。
- 効果線も、すべての種類と設定を 1 つの巨大な仕組みに詰め込むのではなく、種類ごとに必要な処理だけを持つ軽い構成にする。

### 0.9.6 軽量化の原則

- フキダシや効果線の移動中は、基本的にオブジェクト変形だけで済ませる。
- 新規作成のドラッグ中は、確定前の範囲表示だけを B-MANGA の補助表示で行い、ドロップ時に実体を作る。
- 使っていない形状の設定項目を、常に処理対象や表示対象にしない。
- しっぽは必要数だけ追加し、追加済みのものだけを設定対象にする。
- 常時監視で全ページや全オブジェクトを走査し続ける設計にしない。

- 魚眼FOV、ページ画像のスケール、魚眼モードは **B-MANGA のコマファイル側を正**とする。B-MANGA Render 側では編集欄を置かず、出力時に B-MANGA 側の値を読む。
- 縮小モード、縮小率、元解像度 X/Y は B-MANGA Render 側でも扱う。ファイルを開いたとき・アクティブシーンを切り替えたとき、保存値が食い違っていたら **B-MANGA 側を正**として揃える。
- 同期は B-MANGA 側だけで完結させ、B-MANGA Render 側の独立性を損なわない。

### 0.8.5 B-MANGA Render は単体運用しない／魚眼設定の主担当は B-MANGA
- B-MANGA Render は単体で使われることをあまり想定しない
- 魚眼モード・魚眼FOV は B-MANGA のコマファイル側で調節する。B-MANGA Render では**むしろ調節できない方がよい**（B-MANGA Render 側は読み取り専用表示）
- ページ画像のスケールは B-MANGA Render の出力では使っていないため、B-MANGA Render の「魚眼出力」枠からは外す（同期プロパティ自体は残す）
- 縮小モード・縮小率は B-MANGA Render 側でも引き続き調節できてよい

- コマごとのプレビューは、毎フレームのオーバーレイ描画ではなく、**コマのメッシュに画像テクスチャとして貼り付ける方式**にする（オーバーレイ方式は重く、かつ枠線の上に重なって枠線が細く見える原因になっていた）

- そのため B-MANGA の標準ビューポート（本体・ページ一覧ビューの両方）は Solid のカラーをテクスチャ表示にする。プレビューの無いコマは背景色で表示する

### 0.7.2 B-MANGA と B-MANGA Render の役割分担
- **B-MANGA 本体の機能範囲は、ページ一覧ファイルでの作画と、コマ用blendファイルでの 3D ビューポート上モデル配置まで** とする
- カメラ操作、魚眼レンダリング、出力プリセットなどのレンダー出力系機能は B-MANGA 本体から分離し、半角英数名の別アドオン **B-MANGA Render** とする
- ただし B-MANGA パネル最下部のページ出力系は B-MANGA 本体にも必要な機能として残す。対象は「現在のページを書き出し」「全ページを一括書き出し」「PDF 結合書き出し」
- B-MANGA Render は B-MANGA リポジトリ内で開発できるようにするが、B-MANGA 本体とは別アドオンとして扱う
- `カメラ操作＆出力プリセット.py` の「キャラ」「背景」などの出力プリセットは、B-MANGA Render で自由に組み立てられる出力コマンドとして再現する
- 出力プリセットは Photoshop のアクションのように、上から順番に実行するカード型リストで扱う
- カードはクリックで選択、ダブルクリックで設定ダイアログを開いて変更、リスト右の「＋」で追加できる
- `eeVR-master_ミウラ修正 (19).zip` の魚眼レンダリングは、Blender の魚眼レンダリングにぴったり合う関係性を保つ
- `eeVR-master_ミウラ修正 (19).zip` と `カメラ操作＆出力プリセット.py` の関係性は、B-MANGA Render へ移しても崩してはいけない
- B-MANGA から移植した内容が元アドオン群より不足している場合は、元アドオン群を参照して補う
- B-MANGA Render は、コマ用blendファイル `D:\TM Dropbox\Share\B-MANGA\c_file\c00.blend` と完全に連動する必要がある
- B-MANGA Render 側だけでなく、`c00.blend` 側に不足がないかも確認対象にする

### 0.7.3 ページ一覧ファイルの表示階層と作画対象
- 枠線、テキスト、フキダシ、効果線、ラスター、グリースペンシル、画像など、作品の内容そのものは Blender の実オブジェクトとして残し、アドオンを無効にしても消えたように見えてはいけない
- オーバーレイは、選択枠、ハンドル、カーソル、作成中プレビュー、IME 入力中プレビューなどの編集補助に限定する
- 用紙要素線群は例外として常に最前面に表示し、コマやテキストで隠れてはいけない
- 通常の前後関係は、上から順に **セーフライン外、テキスト、フキダシ、効果線、コマ** を基準にする
- セーフライン外の塗りつぶしは用紙セクションの設定とし、すべてのコマより上に置く
- セーフライン外の塗りつぶしは、色と不透明度を指定できる。不透明度初期値は 30% とする
- 用紙ガイド線とセーフライン外の塗りつぶしは、Blender の「最前面」設定に依存せず、B-MANGA の表示順として常に作品内容より前面に出す
- フキダシ、効果線、テキストは、ドラッグ開始地点にあるコマまたはページの中に作成する
- コマ外かつページ内でドラッグ作成したフキダシ、効果線、テキストは、ページ内ですべてのコマより前面に作成する
- フキダシ、効果線、テキストは、ページ外にも作成できる
- フキダシ、効果線の作成中は、ドラッグ開始からドロップまで B-MANGA のオーバーレイで作成範囲のハンドルだけを表示し、実体はドロップ時に初めて作成する
- ドラッグしなかった場合、テキスト、フキダシ、効果線はいずれも作成しない
- フキダシと効果線は、ページ内またはコマ内にある時、そのページまたはコマに属し、かつコマより前面に見える
- フキダシをコマに重ねた時、線が消えたように見えたり、コマと合体したように見えたりしてはいけない
- B-MANGA コレクションの一番上に「テキスト」コレクションを置き、すべてのテキストレイヤーはそこにのみ作成する
- テキストレイヤーは重なり順でも最上位に扱う
- ページ一覧ファイルのビューセクションには、選択ページを切り替える数値入力欄を置く
- コマ外かつページ内をクリックした場合、そのページを選択できる
- 作品ファイルと非編集中ページはページプレビュー画像のみを表示し、コマ枠の選択・編集に使う実体オブジェクトは持たない
- コマ枠の選択・編集に使う実体オブジェクトは、現在編集中のページ用blendにだけ存在させる

### 0.7.4 レイヤーリスト、アウトライナー、階層移動
- レイヤーリストは、選択中ページだけを対象にし、そのページ上のレイヤーだけを表示する
- レイヤーリスト上部のページ選択を変更したら、アクティブページとアウトライナー上のページ選択も切り替える
- 各ページのレイヤーは、種別に関係なく上下ボタンで重なり順を入れ替えられる
- 種別ごとの階層は初期階層にすぎず、最終的な重なり順の編集はページ単位で行える必要がある
- レイヤーリストでドラッグ中は、半透明のカードプレビューとドロップインジケーターを表示する
- レイヤー移動はクリック開始ではなくドラッグで開始する
- 子階層は一文字分のインデントで視覚的に区別する
- レイヤー名の表示領域を広く取り、右側ボタンとの無駄な隙間を作らない
- フキダシ、テキストを含む全レイヤーに表示・非表示切り替えを用意する
- ビューポート上でもレイヤーリストでも、Ctrl と Shift による複数選択をすべての種類のレイヤーで可能にする
- Alt+ドラッグでページやコマにドロップインジケーターを表示し、ドロップ先のページ内またはコマ内へ入れる
- ページ外へドロップした場合は、ページまたはコマの外へ出す
- Alt+クリックで位置を動かさずクリック地点の下階層へ移動し、Alt+Shift+クリックで位置を動かさず上階層へ移動する
- 上記の Alt 操作中もドロップインジケーターを表示する
- ページ、コマ、各種レイヤーはページ外にも出せる必要がある
- アウトライナーでページやコマを選択した時、ビューポート上でも B-MANGA として選択状態にする
- ビューポート上でページ、コマ、レイヤーを選択した時、アウトライナー上でも対応要素を選択する

### 0.7.5 ツール、右クリック、詳細設定
- B-MANGA パネルのツールセクションでは、グリースペンシルとラスターを常に選択可能にする
- グリースペンシルとラスターは、ツールをクリックした時に描画開始し、他のツールに切り替えた時に直前の描画を終了する
- 「ラスター描画」セクションと「GP描画」セクションは不要
- ラスター描画中も Ctrl+Alt+ドラッグでブラシサイズを調節できるようにし、グリースペンシルと同じ挙動にする
- ビューポート右上ナビゲーションや B-MANGA パネル操作は、どのツール中でも妨げてはいけない
- ビューポート上のオブジェクトをクリックしただけで詳細設定ダイアログを開く挙動は廃止する
- 詳細設定は右クリックメニューから開く
- 詳細設定ダイアログは前回開いた位置を記憶し、次回も同じ位置に表示する
- 右クリックメニューには、詳細設定、複製、リンク複製、削除を用意する
- 削除時はシンプルな確認ダイアログを出す
- すべてのツールと右クリック位置の組み合わせで、空の右クリックメニューを出してはいけない
- 各レイヤーの詳細設定にあった項目は消滅させず、全項目が実際に機能しているか確認対象にする

### 0.7.6 テキスト
- テキスト本文の入力は、ダイアログ本文入力ではなくインライン入力を基準にする
- 半角/全角キーで日本語入力へ切り替えられなければならない
- インライン入力中も縦書きを維持する
- インライン入力内でドラッグしてテキストを選択できる
- 選択中テキストに対して、色、太字、斜体、フォントサイズ、フォントを設定するポップアップを出す
- 選択ハイライトとカーソル位置は、表示文字とずれてはいけない
- テキストはアウトライナー上の階層編集用の実体を持ち、表示と書き出しでは正しくレンダリングされる必要がある

### 0.7.7 フキダシ、効果線、枠線
- フキダシの形状設定は Meldex のボード機能のカード形状と同等にし、形状の再現方式も合わせる
- 効果線の描画位置は「内側」固定とし、「描画位置」セクションは不要
- 効果線では「内側」を「終点」、逆側を「始点」と呼ぶ
- 効果線がコマ内にある場合、「始点をコマ枠に設定」をオンにすると、始点はコマ枠から効果線の太さ分だけ枠外へはみ出す位置に揃える
- 「始点をコマ枠に設定」では、線端を決める基準線はコマ枠に揃え、表示される線は効果線の線幅分だけ外側まで届かせる。距離指定の密度計算は常に実際のコマ枠を基準にする
- 距離指定では終点形状が極端な横長/縦長の場合も密度補正できるようにする
- 効果線の始点形状と終点形状は、フキダシ形状と同じ設定・編集ができる
- 「始点乱れ」「終点乱れ」は、各線一本ごとの実際の始点から終点までの長さを基準に、指定％まで線端を短くする。乱れ値は0%と100%も取り得るため、100%指定では線がゼロ長まで短くなり得る。線ごとの乱れは隣接線が波のように連続しない疑似ランダムとし、同じ効果線では安定して再表示される
- 効果線は、作成範囲の中心ではなく保存された中心点を放射方向の基準にする。終点形状は作成範囲のハンドル位置と縦横比に一致し、選択時の形状ガイドと実際の表示結果がずれてはいけない
- 雲、もやもや、トゲ系の形状計算は、フキダシ本体、効果線の始点形状、効果線の終点形状で共通化し、同じ設定なら同じ見た目になるようにする
- フキダシ本体は、制御点を編集できる Blender カーブ実体を正本にする。ジオメトリノードを使う場合でも、塗り、線、切り抜き、グラデーション、不透明度などの軽い補助に限定する
- フキダシの雲、もやもや、トゲなどの外形は、B-MANGA が作成時または明示再生成時にカーブ制御点として作り、確定後の見える本体は Blender の実体として残す
- B-MANGA の画面描画は選択枠、ハンドル、作成中範囲、形状ガイドなどの編集補助に限定する
- B-MANGA が行ってよい処理は、最初の実体作成、UI 値の同期、素材・表示状態の同期、選択・階層・マスクなどの管理に限定する。通常移動やサイズ変更中にページ全体を常時監視したり、重い再生成を繰り返したりしてはいけない
- ジオメトリノードモディファイア上に出す設定項目は、実際の表示へ効くものだけにする。表示に効かない項目を見せる場合は不具合として扱う
- フキダシの雲・もやもや・トゲ系や未追加のしっぽのように、常時は使わない重い設定はジオメトリノードモディファイアの設定欄に常時出さない。追加済みのしっぽは B-MANGA の詳細設定側で編集する
- 効果線の線の長さ設定は廃止し、始点形状から終点形状までで決まる
- 始点の形状ラインは紫、終点の形状ラインは水色にする
- 効果線の形状線はベジェにする
- 効果線には通常とは別に「白抜き線」と「流線」を追加する
- 白抜き線は、束の数、束の幅(mm)、太さ乱れ、長さ乱れ、白線割合、白線本数、白線間隔(mm)、白線太さ(mm)、白線減衰、白線の入り/抜き、入り/抜き範囲(％指定または長さ指定)、黒線本数、黒線方向、黒線太さ(mm)、黒線間隔(mm)、黒線幅変化、黒線長さ変化、黒線減衰、角度を設定できる。白線側は通常の塗り色に引きずられず白として表示する
- 白抜き線の初期値は、本数 5 本、間隔 0.2mm、太さ 10mm、白線割合 30%、白線太さ 0.3mm、黒線太さ 0.3mm を基準にする
- 集中線やウニフラッシュの線には、主線とは別に「白抜き線」を追加できる。初期値はオフとする
- 集中線やウニフラッシュに追加する白抜き線は、効果線の線幅を100%として、-300%〜300%の範囲で幅を指定する。初期値は150%とする
- 集中線やウニフラッシュに追加する白抜き線は、値がプラスの時は線の中心線の右側だけ、マイナスの時は左側だけを太くする
- 効果線には「下地線ズラし」や「下地線の終点を揃える」を置かず、白抜き線のオン・オフと幅だけで制御する
- 流線では、始点線と終点線を閉じた形状にせず、開いたベジェ曲線にする
- リンク効果線は、位置、中心点、形状線を連動し、それ以外の設定は別々に変更できる
- 枠線カットツールでは、ドラッグ中に赤線だけでなくカット後プレビューを表示する
- コマ枠の辺・頂点選択とドラッグ編集は、独立した枠線選択ツールではなくオブジェクトツールに統合する
- 枠線カットツールでも、オブジェクトツールと同じ辺・頂点選択とドラッグ編集ができる
- コマ枠の辺の選択ハイライトは、選択した辺だけに出す
- 三角ハンドルは連続して操作でき、カーソルオーバーでハイライトし、横に広げて押しやすくする
- 複数コマ選択時は詳細設定にコマ結合ボタンを出し、凹凸のある多角形になる結合も許容する

### 0.7.8 ラスター、用紙、コママスク
- ラスターレイヤーは 150dpi と 300dpi を選択できる
- ラスターレイヤーはページまたはコマにぴったり合う平面として扱い、透明部分は透過して表示する
- ページ外背景まで用紙色や白で塗ってはいけない
- 用紙色は用紙セクションで変更でき、3D ビューポートにも反映する
- 用紙は白または設定した用紙色として見え、単なるビューポート背景が見えている状態にしてはいけない
- ラスター描画時に、存在しない右ページが白くなったり黒くなったりしてはいけない
- ラスター描画で Blender が強制終了してはいけない
- ラスター描画はノイズ状にちらつかず、ズームで蠢くように見えてはいけない
- コマ形状変更、コマ移動、三角ハンドル、オブジェクトツール、レイヤー移動ツールでの編集に、コママスクが追従する
- コマごとの背景色は初期値を用紙色とする
- コマのマスクは、コマ内の各種レイヤーに一貫して適用される必要がある
- レイヤーを追加する時、選択中のコマがある場合はそのコマ内に作成する
- Z 方向の重なり順はページごとにリセットされるもので、ユーザーが見る重なり順として扱う

### 0.7.9 コマ用blendファイル
- UI 表記は「コマ3Dテンプレート」ではなく「コマ用blendファイル」にする
- コマ用blendファイルは、すべての作品で使い回せるようプリファレンスで設定可能にする
- コマファイル内には、ページ一覧ファイル用の B-MANGA コレクション全体を持ち込まない
- コマファイルを開いた時は、「ページ画像」として、ページ一覧ファイルのコマ枠、ラスター、グリースペンシル、テキスト、フキダシ、効果線など全要素を参照表示する
- 「ページ画像」では、編集中のコマの背景だけを透明にし、ページ一覧ファイル側の線・テキスト・フキダシ・効果線などは残す
- コマファイルの出力解像度は、ページ一覧ファイルで設定した用紙サイズと dpi から算出したピクセル数に一致させる
- コマファイルで 3D オブジェクトなどを配置してページ一覧ファイルへ戻った時、該当コマにはコマファイルのスクショが表示される
- コマ用blendファイルの仕組みは、B-MANGA Render と `c00.blend` の連動確認対象にも含める

### 0.7.10 実機確認と報告
- 「徹底チェック」や「微細挙動チェック」では、B-MANGA パネルのチェックボックス一つ一つ、ビューポート上のハンドル、起こり得る操作を実機操作と目視で確認する
- 修正後は、ユーザーへ渡す前に実機操作と目視で解決を確認する
- ユーザー向け説明は、コード名ではなく B-MANGA と Blender の画面に出る UI 表記で行う
- ユーザー操作に影響する修正では、バージョンは 0.0.1 ずつ上げる

## 0.6 2026-04-26 時点の追加設計意図（競合時はこの節を優先）

### 0.6.1 仕様定点観測の扱い
- B-MANGA でも `仕様定点観測` を採用する
- ただしグローバル共通ルールには専用規定が無いため、`Meldex-public` 側の運用を参照し、B-MANGA では `docs/仕様定点観測_設計意図.md` を基準に扱う
- 今後は **設計意図**（ユーザーが望む仕様）と **計画書**（現在の仕様整理・未達差分）と **実装実態スナップショット** を分離して残す

### 0.6.2 コマ編集モードの意図仕様
- **1コマ = 1 `cNN.blend`** の独立ファイル方式は維持する
- ただし、コマ編集モードに入っても **作品共通の用紙設定は崩れてはいけない**
  - 開始ページの位置
  - 読む方向
  - 用紙色
- コマ編集モードでは **そのコマだけが完全に孤立して見える状態は不可**
- 少なくとも以下は参照表示できる必要がある
  - 同じページの他コマ
  - 他のページ
  - 紙面上の位置関係
- 参照表示は読取専用でよいが、**構図合わせの基準として常時見える** ことを優先する

### 0.6.3 コマ編集結果の紙面反映
- コマ編集モードを閉じてページ一覧・紙面編集へ戻った時、**そのコマの編集内容は即座に紙面側へ見える** 必要がある
- 反映手段はサムネイルでもプレビューでもよいが、「保存したのに紙面では古いまま」は不可
- 自動更新の既定は軽量な方式でよいが、ページ表示・書き出し・後続編集の3箇所で同じ最新内容を参照できる必要がある

### 0.6.4 枠線ツールの最新意図
- 枠線カットツール / オブジェクトツールのモーダル中でも、B-MANGA パネルやビューポート右上ナビゲーションは操作できる
- オブジェクトツールでは以下のコマ枠要素を直接編集できる
  - 枠線全体
  - 1辺だけ
  - 1頂点だけ（接続2辺へ適用）
- `コマ:枠線` セクションの対象は、アクティブコマではなく **オブジェクトツールで選択した枠線** を優先する
- 枠線の `▲` 拡張は、隣接候補へぴったり重なる方向の操作で **コマ形状を壊してはならない**
- 縦横均等分割は枠線ツール文脈で使う。対象は選択中コマで、矩形だけでなく変形コマにも対応する

### 0.6.5 用紙設定の最新意図
- 用紙パネルから **用紙色を直接編集可能** にする
- 開始ページの初期値は **左から**
- セーフライン外側の塗りは、色と不透明度を指定可能にし、不透明度初期値は **30%** とする
- 用紙セクションでコマ枠線幅を指定できる。新規コマはこの値で作成し、既存コマにも同じ用紙設定として反映する
- 用紙ガイド線とセーフライン外側の塗りは、Blender の「最前面」設定をオンにせず、B-MANGA の表示順で常に前面に見せる

### 0.6.6 書き出しの最新意図
- 実用優先の主目標は **PSD 書き出し強化**
- PSD では、コマは **コマ形状のレイヤーマスクが掛かったレイヤーフォルダ** として出ることを優先する
- Grease Pencil / フキダシ / 効果線 / テキスト / 画像レイヤーも、可能な限り個別レイヤーとして残す
- `.clip` 直書きは現時点では見送る
  - コマフォルダ化やラスターレイヤー化の可能性は調査済み
  - ただし工数と破損リスクを考え、今は計画書だけ残して着手しない

### 0.6.7 B-MANGA と B-MANGA Render の分離
- **B-MANGA 本体の機能範囲は、ページ一覧ファイルでの作画と、コマ用blendファイルでの 3D ビューポート上モデル配置まで** とする
- 出力プリセット、魚眼レンダリング、カメラ操作由来のレンダー出力は、別アドオン **B-MANGA Render** に分離する
- 一方で、ページ一覧ファイルから直接使うページ出力系は B-MANGA 本体の B-MANGA パネル最下部にも残す
- B-MANGA Render は B-MANGA リポジトリ内で開発するが、B-MANGA 本体とは独立したアドオンとして登録する
- 既存の `カメラ操作＆出力プリセット.py` と `eeVR-master_ミウラ修正 (19).zip` 由来の出力系機能は、B-MANGA Render 側へ移す
- B-MANGA Render では「キャラ」「背景」などの出力プリセットを、Photoshop のアクションのように上から順番に実行するカード型コマンドリストとして扱う
- 上記に伴い、0.6.6 の「PSD 書き出し強化」のうち、出力プリセットやレンダー工程と組み合わせる高度な出力構築は B-MANGA Render の担当に移管する。ページ一覧からの基本的なページ出力系は B-MANGA 本体にも残す

## 0. プロジェクト全体の位置付け

### 0.1 B-MANGA とは
- Blender上で動作する、漫画ネーム作画専用のアドオン
- CLIP STUDIO PAINTの3D機能と同等以上の体験を、Blenderの3Dエンジン上で実現する
- 3D操作と2D作画（コマ/フキダシ/ペイント）をシームレスに統合する

### 0.2 Meldexとの関係（独立性）
- **独立したアドオンとして開発する**
- 連携はMeldex→B-MANGA一方向のみ
  - Meldex → B-MANGA: 「クリスタへ送信」と同様の「B-MANGAへ送信」機能でシナリオ（台本テキスト＋ページ構成）を送信
  - B-MANGA → Meldex: 連携なし
- Meldexのソースコードには一切依存しない
- Meldex側には「B-MANGAへ送信」ボタンの追加のみ行う（別計画書で管理）

### 0.3 なぜBlenderアドオンなのか
- Blenderビューポートをブラウザ（Meldex）に埋め込む公式手段は存在しない
- 3D操作と2D作画のシームレス統合にはBlender描画スタックへの直接アクセスが必須
- Grease Pencilオブジェクトは3D空間内で2D描画を行う既存機能で、ネーム作画の基盤として最適

### 0.4 Pythonの速度・UI懸念への方針
- 描画処理本体はBlenderのC++エンジンが担うため致命傷にならない
- UIは Blender標準UI（Panel / Operator / Popup / Modal）で必要十分、不足時は `gpu` モジュールでオーバーレイ描画
- 重いPython処理はC拡張化を後段で検討（MVPでは不要）

### 0.5 開発ディレクトリ
`D:\Develop\Blender\B-MANGA\`

## 1. 階層構造

### 1.1 基本階層
- 作品 ＞ ページ ＞ コマ の**3層で確定**
- 「プロジェクト」という呼称は廃止、「作品」に統一する
- 長期連載作品共通の設定（作品より上の階層）は**一旦不要**

### 1.2 実体
- 作品: `MyWork.bmanga/` ディレクトリ（拡張子風フォルダ名）
- ページ: `pNNNN/` 配下の1ディレクトリ
- コマ: `cNN.blend`（独立ファイル）

### 1.3 メタデータファイル
- `work.json`: 作品全体メタデータ
- `pages.json`: ページ一覧・順序管理（ページ並び替えはこのJSON順序変更のみで完結）
- `page.json`: 個別ページメタデータ
- `cNN.json`: 個別コマメタデータ
- スクショサムネイル画像（`cNN_thumb.png`）／手動高品質プレビュー画像（`cNN_preview.png`）はバックアップ対象外でよい

## 2. ファイル方式

### 2.1 独立コマファイル
- **1コマ=1.blendファイル**として独立させる
- 理由: メモリ効率／コマの移動・複製が容易／バージョン管理との相性／破損の局所化
- コマファイルは3Dモデル本体を持たず、アセットライブラリからのリンク参照のみで構成

### 2.2 ファイル命名規則
- ページディレクトリ: `pNNNN/`（`p` + 4桁ゼロパディング）
- コマファイル: `cNN.blend` / `cNN.json` / `cNN_thumb.png` / `cNN_preview.png`（`c` + 2桁ゼロパディング、最大 `c99`）
- 内部ファイル名は常に連番固定（リネームしない）
- ユーザー向け表示名は `cNN.json` に別途保持（表示名変更で参照が壊れないようにする）

### 2.3 コマの移動・複製
- コマを他のページへ移動・複製できる
- ファイル単位で完結する（ファイル移動/コピー）
- 移動先でファイル名衝突したら自動リネーム（空き番号へ）
- 作画レイヤー（Grease Pencil・フキダシ）もコマと同時に移動・複製される

## 2.5 基本作画機能

### 2.5.1 レイヤー機能（2種類のレイヤーを扱う）
- **ベクターレイヤー**（Grease Pencil v3ベース）— 線画・フキダシ・塗り
- **画像レイヤー**（ビットマップ）— スキャンラフ取り込み・写真参照・実写背景

共通機能:
- レイヤー追加・削除・複製・並び替え・結合
- 表示／非表示・ロック・不透明度・ブレンドモード
- レイヤーフォルダ（Grease Pencil v3 の `GreasePencilLayerGroup` を活用）

### 2.5.2 画像レイヤー固有機能
- 画像ファイル読み込み（PNG / JPG / TIFF / PSD）
- 位置・拡大縮小・回転・反転
- 不透明度・ブレンドモード
- 簡易レベル補正（明度／コントラスト／2値化）— 下書き取り込み用途
- 描画方式は `draw_handler_add` + `gpu` オーバーレイで統一（平面メッシュは使わない、Grease Pencil v3 画像リファレンスも不採用）

### 2.5.3 ペイント機能
- ペン／鉛筆／マーカー／消しゴム／塗りつぶし
- ブラシサイズ・不透明度・**筆圧**
- カラーピッカー（Blender標準 ColorRamp を活用）
- 定規・直線・図形ツール

### 2.5.4 ブラシ方針（段階実装）
- **Grease Pencil v3 標準ブラシ**で漫画的線質（入り抜き・筆圧・テーパー）をまず構築
- 不足する質感（かすれ・アナログ感）は Phase 3 で**独自ブラシプリセット**として追加
- 独自ブラシエンジンは Phase 6 以降に後送り

### 2.5.5 コマ割り機能（基本）
- コマ枠の自動生成: **縦N分割／横N分割／指定レイアウト**
- コマ枠のドラッグ編集
- コマごとにローカル座標系を持ち、その中で3Dモデル配置・作画が可能
- **コマ境界でのクリッピング**（コマ領域外の線は自動で隠す）

## 3. 3D連携

### 3.1 2層構造（紙面ワールド / 3Dワールド）
- 紙面シーン: 原稿用紙・コマ枠・Grease Pencilレイヤー・画像レイヤー・フキダシを保持
- コマごとの3Dシーン: 各コマが独自の3D空間を持つ
- **同一ページ内の異なるコマで、まったく別の場所・別の登場人物を扱える**（長期連載の話数間も共有/非共有で自由）

### 3.2 カメラ構成
- **カメラリグ方式**: 親Empty + 子カメラ
- パン・ズーム → 親Empty
- 画面回転（ロール） → 子カメラ（方法A確定）
- B-MANGAの紙面編集モード中およびコマ編集モード中は、ビューポートを常にカメラビューに固定
- 画面回転ではコンテンツは動かず、カメラが傾くことで結果的に画面上で全要素が一緒に回って見える

### 3.3 モード切り替え
- **紙面編集モード**: 紙面シーンで作画。コマ内の3Dはプレビュー表示
- **コマ編集モード**: コマのダブルクリックまたは右クリックで入る。アクティブシーンがそのコマの3Dシーンに切り替わる
- コマ編集モードへ入っても、**用紙色／開始ページ／読む方向など作品共通の用紙・ページ配置設定は紙面編集モードと一致したまま**でなければならない
- コマ編集モードで編集対象として直接操作できるのはそのコマ自身だけに限定するが、**参照表示として同一ページの他コマ・前後ページ・見開き相手ページを読み取り専用で見られる**ようにする
- 「そのコマ以外が一切見えない孤立した空シーン」は採用しない。構図合わせ・連続性確認・視線誘導の確認ができることを優先する

### 3.4 コマ枠内の3D表示（スクショサムネイル方式）
- **「コマを閉じるたびにレンダリング」は避けたい**という方針
- レンダリングは書き出し時のみ
- コマ編集モード終了時: **ビューポートのスクリーンショット**を `cNN_thumb.png` に保存（数十ms、レンダリングなし）
- 紙面編集モードへ戻った時は、**直前に編集したコマの内容がページ一覧・紙面上に即反映される**こと。直近スクショがそのままページ側の可視状態の基準になる
- 過去スクショがあればそれを表示
- **新規コマでスクショ未生成時: コマ枠の背景色（白）のみ**（プレースホルダ画像は生成しない）
  - 理由: 新規コマ作成直後はどのBlenderファイルを読み込むかも決まっていないため、スクショ取得は原理的に不可能
- スクショ解像度はコマ枠の表示サイズに合わせた最小限
- ユーザー手動「高品質プレビュー生成」: コマ右クリックから、そのコマだけ低解像度レンダリングし `cNN_preview.png` に保存
- `cNN_preview.png` と `cNN_thumb.png` の両方がある場合は、更新日時が新しい方を使う。古い高品質プレビューが編集後の自動サムネイルを隠してはいけない

### 3.5 3Dアセット管理
- **Blender標準のアセットブラウザを活用する**（B-MANGA独自のアセット管理UIは作らない）
- アセットライブラリは**2層構造**:
  1. グローバルアセットライブラリ（全作品共通）
  2. 作品固有アセット（`MyWork.bmanga/assets/`）
- 作品単位の中間層は作らない（長期連載共通設定と同じく不要）
- 3D素材は作品に関係なく全作品で共有したい

### 3.6 モデル本体の編集
- **3Dモデル本体の編集はBlender本体で直接 `.blend` を開いて行う前提**
- B-MANGA上ではリンク参照したモデルのアセンブル（配置・ポーズ）のみ
- カスタム右クリックメニュー「リンク元ファイルを開く」で、**新しいBlenderインスタンスを subprocess で起動**（現在の編集作業を失わない）
- 設定で「現在のBlenderで開く」にも切替可能

### 3.7 3D編集機能
- **3Dモデルのポーズ編集**はコマ単位で独立（他コマに影響しない）
- **3Dモデルの線画化**（Freestyle / Grease Pencil変換）
- **3D→2D投影して作画レイヤーへ落とし込み**

### 3.8 非採用機能
- **パース定規機能は実装対象外**（3Dカメラがパース情報を直接持つため不要）
- **トーンは不要**

## 3.9 Meldex連携（受信）

- Meldex側の「B-MANGAへ送信」ボタンでシナリオをPOST送信
- 受信方法: ローカルHTTPサーバー（詳細は 11.6）

### 3.9.2 受信データ構造（JSON）

- 各ページのコマ数目安

## 6. コマ枠機能

### 6.1 形状
- 長方形／多角形／曲線（ベジェ）／フリーフォーム
- **コマ枠同士の重なりを許容**（飛び出しコマ／挿入コマ／アイキャッチ等）

### 6.2 枠線スタイル
- 線種: 実線／破線／点線／二重線／カスタム
- 線幅（mm）、線色
- 角処理: 直角／丸角／面取り
- **辺ごとの個別オーバーライド**（上辺だけ太線、右辺だけ破線、等）
- **枠線全体の非表示**（コマ機能は維持したまま）

### 6.3 フチ
- ON/OFF、幅（mm）、**既定 0.5mm**
- 色（既定: 白）
- 枠線に対して **外側／内側／両側** を選べる

### 6.4 作成・編集ツール
- 矩形コマ作成
- **縦横均等分割**は独立した「分割テンプレート」セクションではなく、**枠線ツール内**に置く
- 縦横均等分割の対象は「現在アクティブなページ全体」ではなく、**オブジェクトツールで選択中のコマ**とする
- 縦横均等分割は**矩形コマだけでなく変形コマ（polygon）にも対応**する
- **枠線カットツール**（CLIP STUDIO PAINTの同名ツール相当、ドラッグで切断、切断後に作品共通のコマ間隔が自動適用）
- 水平カット／垂直カットの個別ボタンは不要
- **オブジェクトツール**で、辺単位／頂点単位／枠線全体単位の選択と編集を行える
- オブジェクトツールで選択した辺の両端頂点は、選択ハイライトとして表示する
- オブジェクトツール・枠線カットツールの使用中でも、**B-MANGA パネル操作とビューポート右上ナビゲーション（虫眼鏡・手のひら等）は使える**ようにする
- オブジェクトツール内のコマ枠ハンドルクリック1回、ドラッグ確定1回を**それぞれ独立してアンドゥ／リドゥ**できるようにする
- 枠線の色・太さは、コマ全体だけでなく**1 辺だけ／1 頂点に接続する 2 辺だけ**にも設定できる
- 頂点編集、結合・再分割、フリーフォーム

### 6.5 スナップ
- **仕上がり枠・基本枠・セーフライン・トンボにスナップする**
- **他のコマ枠にもスナップする**（作品共通のコマ間隔を保って）
- Shiftで一時無効化

### 6.6 コマ間隔ルール（作品内共通）
- **上下の間隔・左右の間隔を別々に設定可能**
- mm単位
- **既定: 上下 7.3mm / 左右 2.1mm**
- **作品内共通ルールとして共有**（`work.json` に保存、作品内全ページ・全コマで共有）

### 6.7 Z順序・重なり
- 各コマはZ順序値を持つ
- **重なり順を変えられる**（最前面へ／最背面へ／1つ前面へ／1つ背面へ、コマ一覧ドラッグ）
- ぶち抜きコマ（仕上がり枠超え・裁ち切りまで）も同じ仕組み

## 7. フキダシ・テキスト

### 7.1 フキダシ形状（本体）
**Meldexボードのカード形状と完全一致で実装する**:
- 矩形
- 楕円
- 雲
- トゲ（曲線）
- トゲ（直線）

**5形状で確定。**（角丸・付箋型しっぽは形状プリセットではない）

### 7.1.1 フキダシの線
- ラインスタイルは、線なし、実線、破線、点線、多重線を持つ
- 多重線は、線の本数、線幅、間隔、線幅変化、重ねる方向を設定できる
- 多重線の初期値は、線の本数 3、線幅 0.3mm、間隔 0.4mm とする
- 重ねる方向は、外側、内側、両方向から選べる
- 線幅変化は、主線から離れるごとに線幅へ掛ける割合として扱う。たとえば 80% の場合、主線から離れるたびに 0.8 倍の線幅にする
- トゲ（直線）の多重線では、谷の頂点側と山の頂点側の線幅を別々に設定できる
- トゲ（直線）の多重線では、谷の頂点側を始点、山の頂点側を終点とし、主線から離れるごとに長さを指定割合で変化させられる
- 多重線の追加線は表示用の軽い補助線として扱い、ユーザーから見える管理単位は 1 フキダシ 1 要素に保つ

### 7.2 角丸オプション
- フキダシ形状ではなく**全形状共通のオプション**
- ON/OFF、角半径（mm）
- 楕円のように既に丸い形状では無効

### 7.3 カスタムフキダシ形状
- **パスツールで作成した任意の形状をフキダシとして登録できる**
- 名前を付けて保存、縮尺方式（絶対／相対）を登録時選択
- 登録後の頂点再編集・削除・リネーム可
- 保存場所: 作品ローカル `MyWork.bmanga/assets/balloons/` ／グローバル

### 7.4 尻尾（しっぽ）
- **全形状共通で追加できるオプション**
- 3種類:
  - 直線尻尾（三角形）
  - 曲線尻尾（ベジェで膨らませる）
  - 付箋型しっぽ（Meldex注釈ツールの付箋のような矩形タブ）
- 各尻尾のパラメータ: 方向／長さ／根元幅／先端幅、曲線尻尾ではカーブ曲がり具合
- 1フキダシに複数尻尾対応（話者共有セリフ・ハモり等）
- フキダシなしテキストにも追加可（擬音から線を引き出す等）

### 7.5 フキダシなしテキスト
- **フキダシ本体を持たないテキスト単体オブジェクトも作成可能**
- 用途: 擬音／手書き風の台詞／ナレーション／心の声／場面転換の文字等
- テキスト属性はフキダシ内テキストと共通

### 7.6 テキスト属性
- フォント、フォントサイズ、行間、字間、縦/横書き切替
- ルビ、縦中横
- 文字色
- **白フチ（テキストストローク）**: 指定幅（mm）でテキスト外周を縁取り
  - 幅既定 0.2mm、色既定 白
  - ON/OFF
  - フキダシ内テキスト・フキダシなしテキスト両方で利用可能

### 7.7 縦書き日本語組版
- **縦書きを実現する**
- **ルビ**（親文字＋ルビ文字、モノルビ／グループルビ／熟語ルビ）
- **縦中横**（半角数字・英字の縦書き内横向き表示）
- **行間（行送り）**設定可能
- **字間（字送り）**設定可能
- 禁則処理（行頭禁則／行末禁則／ぶら下げ）
- 字形選択（縦書き用グリフへの自動切替）

### 7.8 セリフ種別→形状マッピング
Meldexから受信したセリフ種別を既定形状にマッピング（作品単位で変更可）:
- 通常セリフ → 矩形（角丸ON）
- 思考 → 雲
- 叫び → トゲ（直線）
- ナレーション → 矩形（角丸OFF）

## 8. 効果線ツール

### 8.1 対応タイプ
**クリスタのような、ベタフラ・ウニフラ・集中線も作成できるようにする**:
- 集中線
- ウニフラ（ウニフラッシュ）
- ベタフラ（ベタフラッシュ）
- 流線（**直線または曲線パスに沿って**平行な線を並べる動き・速度表現、ベジェ沿い対応）

### 8.2 基準図形
- 長方形／楕円／多角形（頂点数指定）
- 放射線定規を中心に用いる

### 8.3 線パラメータ（クリスタのツールプロパティ準拠）
- 線幅（既定 0.30mm）＋乱れ
- 線の間隔（角度指定／距離指定、距離既定 1.00mm）＋乱れ
- 始点をコマ枠にした時の距離指定は、密度計算も常に実際のコマ枠を基準にする
- 距離指定では終点形状の縦横比による密度差を常に自動補正する（独立した密度補正設定は置かない）
- まとまり（数の既定 5、数の乱れ既定 0.5、まとまり間隔既定 5.00mm、間隔乱れ既定 0.5）
- 長さ（既定 10.0mm）＋乱れ、コマ外まで延長
- 基準位置（始点／中点／終点）＋ずれ（既定 2.0）＋ギザギザ化
- 入り抜き（適用モード／入り／抜き、既定 100%）

### 8.4 色・塗り
- 線の色（3スロット）、塗りの色（ベタフラ用）
- 下地を塗りつぶす（ON/OFF、ベタフラ）、塗りの不透明度（既定 100）

### 8.5 描画先・プリセット
- 既定「常に新規効果線レイヤーを作成」
- Grease Pencilベクターレイヤーとして生成（後から頂点編集可）
- プリセット保存（作品ローカル／グローバルの2層）
- 既定プリセット: 集中線標準/細かい/粗い、ウニフラ標準/激、ベタフラ標準、流線標準/スピード

## 9. ビューポート操作ショートカット

**CLIP STUDIO PAINTのショートカットを実装する:**

| 操作 | ショートカット |
|---|---|
| 手のひら（パン） | `Space + ドラッグ` |
| 回転 | `Shift + Space + ドラッグ` / `Shift + マウスホイール` |
| ズームイン | `Ctrl + Space + クリック`（macOS: 先にSpace→Cmd） |
| ズームアウト | `Alt + Space + クリック` |
| ズーム（連続） | `Ctrl + Space + ドラッグ` / `マウスホイール` |
| レイヤー選択 | `Ctrl + Shift + クリック` または `Ctrl + Shift + ドラッグ` |
| スポイト | `右クリック` |

- B-MANGA専用キーマップとして独立定義、アドオン設定でカスタマイズ可
- Blender標準操作（中ボタンドラッグ等）も従来通り併用可能
- B-MANGAモード中のみ既定キーマップを一時オーバーライド（KeyMapItem.active切替）

### 9.1 モード依存の挙動
- **紙面編集モード中**: ショートカットは紙面シーンのビュー用カメラに作用
- **コマ編集モード中**: ショートカットはそのコマの3Dシーンのシーンカメラに作用
- **ペイント中**: すべてのショートカットが優先（ストローク描画より先に判定）
- **オブジェクト選択中**: レイヤー選択は作画レイヤーのみを対象、3Dオブジェクト選択はBlender標準キー（左クリック）を維持
- **テキスト編集中**（フキダシ内文字入力）: ビューポート操作ショートカットは一時無効化

## 10. 作品情報・ノンブル・書き出し

### 10.1 作品情報（原稿上表示）
- **作品情報を原稿上に表示したい**
- 作品名／話数／サブタイトル／作者名
- 表示位置: 上／下 × 左／中央／右 の9通り（項目ごとに個別指定）
- フォント、サイズ、色
- 見開きページでの左右個別配置
- 書き出し時にも原稿画像に焼き込まれる

### 10.2 ノンブル（ページ番号）
- ON/OFF、開始番号（既定1）、フォーマット（プレースホルダ `{page}`）
- フォント既定 `I-OTFアンチックStd B`、サイズ既定 9.0pt
- 位置、基本枠との間隔（縦5.00mm / 横0.00mm 既定）
- フチをつける（ON/OFF、フチ幅既定 0.30mm）
- 隠しノンブル（ON/OFF、裁ち落とし外へ配置）

### 10.3 単一ページ書き出し
- 現在開いているページ（または選択した1ページ）の書き出し
- フォーマット: PNG / JPEG / TIFF / PSD / PDF
- カラーモード: モノクロ／グレースケール／RGB／CMYK
- 出力範囲: 仕上がり枠／裁ち落とし込み／基本枠／キャンバス全体
- 解像度・倍率指定
- **PSD はレイヤー保持で出力**し、各コマはフォルダ化して扱う

### 10.4 複数ページ一括書き出し
**CLIP STUDIO PAINTの「複数ページ書き出し＞一括書き出し」のように、作品内の全ページもしくは選択した複数ページを連番統合縮小画像出力できるようにする**:
- 対象: 全ページ／選択ページ／範囲指定
- 見開きページは1枚の画像として書き出し（または左右分割も選択可能）
- 出力形式: PNG/JPEG/TIFF/PSD/PDF
- カラーモード: モノクロ／グレースケール／RGB／CMYK
- 連番ファイル名テンプレート（プレースホルダ `{workName}` / `{episode}` / `{page}` / `{pageStart}-{pageEnd}`）、ゼロパディング桁数指定
- 縮小出力: 倍率／最長辺ピクセル／解像度指定
- **補間方法**: バイキュービック／ランチョス／バイリニア／ニアレストネイバー
- 出力範囲: 仕上がり枠／裁ち落とし込み／基本枠／キャンバス全体
- 追加オプション: コマ枠線／白フチ／トンボ／ノンブル／作品情報／用紙色の含める/含めない、レンダーキャッシュ使用／強制再レンダリング
- セーフライン外側オーバーレイは**常に含めない**
- バックグラウンド処理、進捗ダイアログ（「Nページ中Mページ処理中」）、キャンセル可
- エラー発生ページはスキップして続行、最後にエラー一覧表示

#### 10.4.1 PSD 出力のレイヤー方針
- コマごとに**コマフォルダ**を作る
- コマ形状のレイヤーマスクは、コマフォルダ内の**content サブフォルダ**に掛ける
- 白フチと枠線は、マスクで切れないよう content とは分けて保持する
- Grease Pencil、効果線、フキダシ、テキストは**ラスターレイヤー**として出力する
- `.clip` 直接書き出しは当面の主目標にせず、**PSD を優先**する

### 10.5 書き出し出力先・履歴
- **既定出力先**: `MyWork.bmanga/exports/<出力日時>/`
- 書き出し履歴は `work.json` の `lastExport` に記録（タイムスタンプ／使用プリセット／ページ数／出力ディレクトリ）

### 10.6 書き出しプリセット
- よく使う設定を名前付きで保存・呼び出し
- `work.json` の `exportPresets` に格納
- **既定プリセット（同梱）3種**:
  - 「印刷入稿用PNG」（600dpi / 裁ち落とし込み / モノクロ）
  - 「確認用縮小PNG」（長辺1600px / RGB）
  - 「PDF結合（全ページ）」

### 10.7 書き出しパイプライン（オーバーレイ要素の焼き込み）
`draw_handler_add` によるビューポートオーバーレイは `bpy.ops.render.render()` の出力画像には焼き込まれないため、書き出し時は Pillow で再描画して合成する:

1. 各コマの3Dレンダリング（EEVEE-Next / Workbench）
2. 紙面ベース画像の生成（Pillowで用紙サイズの白画像）
3. 画像レイヤー合成（Pillow）
4. コマ内レンダー画像の貼り込み（Z順序・重なりくり抜きも反映）
5. Grease Pencil v3 レイヤーの合成
6. コマ枠線・白フチの描画（Pillowベクター描画）
7. フキダシ・テキストの描画（Pillow + fontTools）
8. 作品情報・ノンブルの描画
9. トンボの描画（オプション）
10. セーフライン外側オーバーレイは含めない
11. カラーモード変換（モノクロ／グレースケール／CMYK は littleCMS でICCプロファイル変換）
12. ファイル保存（PNG/JPEG/TIFF/PSD/PDF）

書き出し時の描画関数とビューポート用 `draw_handler` 関数は同じ計算ロジックを共有するよう設計し、共通層を用意する。

## 11. 技術実装方針（Blender 4.3+ / 5.x基準）

### 11.1 対応バージョン
- 下限: Blender 4.3.0（Grease Pencil v3正式導入版）
- 主ターゲット: 5.x（最新メジャー）

### 11.2 配布形式
- **Extensions Platform対応**（`blender_manifest.toml` を正、`bl_info` は最小）
- `wheels/` フォルダに同梱ライブラリを配置

### 11.3 描画方式
- コマ枠画像・画像レイヤー・オーバーレイ要素（セーフライン外側／コマ枠線／白フチ／作品情報／ノンブル）は `SpaceView3D.draw_handler_add` + `gpu` モジュールで統一
- 書き出し時は Pillow で同じロジックを再描画して画像合成（`draw_handler` の出力はレンダリング画像に焼き込まれないため）

### 11.4 組版エンジン
- 3層構成:
  - ビューポート表示: `blf`
  - 書き出し再描画: `Pillow`
  - グリフ選択・メトリクス: `fontTools`
- Blenderテキストオブジェクトは使わない（縦書き・ルビ非対応のため）

### 11.5 スクショ取得
- 第1候補: `bpy.ops.screen.screenshot_area()` + `temp_override`
- フォールバック: `gpu.types.GPUOffScreen.draw_view3d()`

### 11.6 Meldex受信
- ローカルHTTPサーバー（localhost限定、既定ポート 47817）
- 別スレッドで `http.server.ThreadingHTTPServer` を起動、受信JSONは `queue.Queue` に積む
- `bpy.app.timers.register` でメインスレッドに戻して `bpy.data` を更新
- アドオン無効化時は確実にshutdown + join + timer解除

### 11.7 Grease Pencil
- v3 前提（`bpy.data.grease_pencils_v3` / `GreasePencilLayer` / `GreasePencilLayerGroup` / `GreasePencilDrawing`）
- レイヤーフォルダは v3ネイティブの `GreasePencilLayerGroup` を使用（独自実装しない）

### 11.8 subprocess起動
- `bpy.app.binary_path` で取得した実行ファイルを `subprocess.Popen` で起動
- Windows: `CREATE_NO_WINDOW` でコンソール窓抑制
- macOS: `.app` バンドル内実行ファイルを直接起動

### 11.9 画面回転
- B-MANGAモード中は常にカメラビュー（`region_3d.view_perspective = 'CAMERA'`）に固定

### 11.10 キーマップオーバーライド
- B-MANGAモード中のみ既定 `KeyMapItem.active = False` に切替、モード終了時に復元
- Blender Preset差異を検出して対応マッピングを選択

### 11.11 アセットブラウザ連携
- `bpy.types.FileHandler.poll_drop`（D&D公式フック）
- フォールバック: `bpy.app.handlers.depsgraph_update_post` で新規Object出現を検知

### 11.12 モジュール分割
- 計画書 5.1 の分割に従う
- register順序: preferences → core → io → typography → operators → panels → ui → keymap（unregister逆順）
- 1ファイル1000行制限、1関数50行目安（グローバルCLAUDE.md準拠）

### 11.13 同梱ライブラリ
- `fontTools`（MIT）、`Pillow`（HPND）
- PDF結合: `pypdf`（Phase 6b）
- PSD書き出し: `psd-tools` / `pytoshop`（Phase 6c）
- CMYK: `littleCMS` + ICCプロファイル（Phase 6d）

## 13. 判断の履歴（撤回された案は記録しない、最終判断のみ）

以下は議論途中で一度検討されたが、最終的に**採用された**方針:

- シーン方式ではなく**独立ファイル方式**（1コマ=1.blend）を採用
- アセットライブラリは**2層**（グローバル／作品固有）。3層案（グローバル／作品単位／プロジェクト固有）は廃止
- コマ閉じるたびのレンダリングは廃止し、**スクショサムネイル方式**に変更
