ルビ組版の業界標準 — 調査レポート
A. ルビのフォントサイズ
業界標準: 親文字の 50%（1/2）

これはすべての規格・ソフトウェアで一致しています。

規格/ソフト	ルビサイズ
JIS X 4051	親文字の 1/2（デフォルト）
W3C JLREQ	親文字の 1/2（デフォルト）
CSS Ruby Module	50%（HTML5仕様のデフォルト）
Adobe InDesign	50%（例: 本文10.5pt → ルビ5.25pt）
マンガ・コミック	50% が標準
見出しなど大きな文字の場合は50%では目立ちすぎるため調整されることがあります。

B. ルビと親文字の間の隙間（gap）
業界標準: 0（接触配置）

JLREQ-d: 「親文字の文字の外枠とルビ文字の外枠を接して配置する」「デフォルトは行間ゼロ」

JIS X 4051: gap = 0（親文字フレームとルビフレームが接する）
W3C JLREQ: gap = 0（デフォルト。指定により変更可）
CSS Ruby Module: 明示的な gap プロパティなし（隣接配置が前提）
InDesign: 「親文字からのオフセット」で調整可能。デフォルトは 0
現在のB-MANGAの ruby_gap_mm = 0.3 は業界標準の0より大きい。 0にするのが標準に最も近いですが、ビューポートでの視認性を考えると小さめの値もあり得ます。

C. ルビの字間（letter-spacing）
業界標準: 0（ベタ組 = 字間なし）

JIS X 4051: ルビ文字列が2字以上の場合はベタ組が原則
JLREQ: ルビ文字はベタ組で配置
InDesign: ルビ文字は solid（スペースなし）が初期値
例外: ルビ文字列が親文字列より短い場合に、空きを均等配分する処理があります（下記D参照）。

D. 配置方法（モノルビ / グループルビ / 熟語ルビ）
モノルビ（親文字1字 : ルビ文字列）
親文字とルビ文字列の中心を揃える（中付き）
ルビが親文字より長い場合: はみ出し分を前後に均等配分
「肩付き」（先頭揃え）は出版社による選択
グループルビ（親文字列全体 : ルビ文字列全体）
短い方に空きを入れて長さを揃え、中心を揃える
空きの比率: 字間 2 : 先頭/末尾 1（JIS X 4051の規定）
例: 親文字2字「東京」にルビ4字「とうきょう」→ ルビ全幅=親文字全幅なので空きなし
例: 親文字1字「猫」にルビ2字「ねこ」→ 親文字幅=ルビ幅なので空きなし
行をまたいでの分割は不可
熟語ルビ（各親文字 : 各ルビ の対応を保持）
モノルビの集合だが、熟語として統一配置
行をまたいでの分割を許容
隣接する熟語内の漢字へのはみ出しを許容
E. ルビが親文字からはみ出す場合のルール
隣接する文字の種類	はみ出し許容	具体的ルール
ひらがな・カタカナ	許容	ルビを前後の仮名に掛けてよい
漢字	不可	読み誤り防止。四分アキを挿入
句点・読点	許容（全文字分まで）	約物の後/前の空きにルビを配置
F. 縦書きと横書きの違い
基本ルールは共通 — JLREQ:「縦組と横組とで配置処理法を変えることなく、共通の処理ができる方法とした」

項目	横書き	縦書き
ルビの位置	親文字の上	親文字の右
サイズ	同一（50%）	同一（50%）
間隔	同一（0）	同一（0）
はみ出し方向	左右	上下
G. ルビのある行の行送り
標準: 行送りは固定のまま、ルビは行間スペースに配置

JIS X 4051: 基本版面で設定した行送りを基準に配置。ルビ付き行だけ行送りを変えるのではなく、全行の行送りを一定にするのが原則
JLREQ: ルビが行間に収まらない場合は版面外にはみ出す
InDesign: ルビを含む行送り推奨値 = 本文サイズ × 150〜175%（例: 10pt本文 → 行送り15〜17.5pt）
マンガ組版: ルビ付き行は行送り二分、ルビなし行は行送り四分
B-MANGAの ruby_line_height は本来のデファクトスタンダードでは「ルビのための追加行間を別途設けない」が正解ですが、マンガ制作ツールとしては可変にしておく利便性があります。

H. 小書き仮名（っ、ゃ、ゅ、ょ）の扱い — ご指摘の問題に直結
これが今回の「変な隙間」の根本原因です。

ルビテキスト内での小書き仮名
3つの方針が業界で共存しています（JAGAT）:

小書き仮名を使う — 「きゅう」「しゅう」のように小書きのまま
小書き仮名を使わない — 「きゆう」「しゆう」のように直音表記
データは小書き、表示は直音 — 音声読み上げ等を考慮した方式
縦書きでの小書き仮名の字幅問題
小書き仮名は全角幅のボディを持ちますが、グリフ（字面）が小さいため、ボディ内に余白が生じます。 縦書き用グリフ（OpenType vert フィーチャー）では:

通常仮名: ボディ中央にグリフ配置
小書き仮名: ボディの右上寄りにグリフ配置（縦書き時）
そのため、ルビテキストを等幅（ベタ）で組んだとき、小書き仮名の前後に見た目上の余白が生じます。 これが「変な隙間」の原因です。

業界での対処
DTPソフト（InDesign等）: フォントの縦書きグリフに任せる。小書き仮名の字面の偏りは「仕様通り」として受け入れる
マンガ制作: ルビでは小書き仮名を使わず直音表記にする制作者も多い
フォント側の対応: 一部のフォントは縦書き用小書き仮名のグリフ位置を調整している
現在のB-MANGAへの提言
項目	現在のB-MANGA	業界標準	対応案
ルビサイズ	ruby_size_percent = 50.0	50%	一致。問題なし
ルビgap	ruby_gap_mm = 0.3	0mm（接触）	デフォルトを 0 に変更検討
ルビ字間	ruby_letter_spacing = 0.0	0（ベタ）	一致。問題なし
行間	ruby_line_height = 1.8	固定行送り（ルビは行間に配置）	マンガ用途ではこのままでOK
小書き仮名の隙間	余白が生じる	フォント依存	描画時に小書き仮名のadvance幅を補正、またはフォントのvert情報を正しく使用
