# フキダシ / NURBSフキダシ / 効果線 統合計画

作成日: 2026-06-28  
担当: Codex  
対象バージョン開始: v0.6.398

## 目的

フキダシ、NURBSフキダシ、効果線で重複している設定と作成入口を整理し、同じ意味の設定が別々に増えて不整合になる状態を避ける。

## 方針

- フキダシツールとNURBSフキダシツールは、画面上は「フキダシツール」に統合する。
- NURBSフキダシはフキダシツールのプリセット項目として選べるようにする。
- 既存のNURBSフキダシ操作は互換用として残し、既存ショートカットや内部呼び出しを壊さない。
- 効果線とフキダシのウニフラ / 白抜き線は、共通定義と共通UI部品を使う。
- 効果線専用の基準パス / 画像線は、統合前のフキダシ設定画面へ混ぜない。
- コマやページのマスク、リンク複製、保存復元、詳細設定は段階ごとに実機確認する。

## Phase 0: 安全な共通定義化

状態: 完了

実施内容:
- 画像パス、効果線、フキダシ線で共通する選択肢と保存対象一覧を共通定義へ移す。
- フキダシのウニフラ設定画面では、効果線専用の基準パス / 画像線を表示しない。
- 共通定義の重複とリンク対象漏れを単体テストで確認する。

確認:
- 全管理対象スクリプトの構文確認。
- 画像パス、効果線の基準パス / 画像線、フキダシのウニフラ / 白抜き線の実機確認。

## Phase 1: フキダシツールの入口統合

状態: 完了

実施内容:
- ツールバー上のNURBSフキダシ専用ボタンを廃止し、フキダシツールのプリセットに「なめらか自由形状」を追加する。
- 「なめらか自由形状」を選んでフキダシツールを押すと、従来のNURBSフキダシ作成操作を開始する。
- 既存のNURBSフキダシ操作は互換用として登録を維持する。
- ツール下のプリセット行は、通常フキダシとNURBS作成中のどちらでも同じフキダシプリセットを表示する。

確認:
- フキダシ通常プリセットで従来どおりドラッグ作成できる。
- なめらか自由形状で従来のNURBS作成が開始できる。
- 他ツールへ切り替えた時、作成中ツールが正しく終了する。

## Phase 2: 線設定UIの共通部品化

状態: 完了

実施内容:
- 効果線の白抜き線設定と、フキダシの白抜き線設定で共通している表示部品を一箇所へ集約する。
- フキダシ側と効果線側で保存名が異なる項目は、表示用の対応表で吸収する。
- 効果線専用項目とフキダシ専用項目は、共通部品内の明示オプションで出し分ける。

確認:
- 効果線の白抜き線で、白線 / 黒線 / 入り抜き / 基準パス / 画像線が従来どおり表示される。
- フキダシの白抜き線で、従来の白線 / 黒線 / 入り抜きが表示され、効果線専用項目は出ない。
- 詳細設定パネル、レイヤー詳細、ツールパネルのどの入口でも同じ表示になる。

## リスクと対策

- NURBS作成はモーダル操作なので、画面ボタン統合だけで内部操作名を消すと既存の終了処理が壊れる。対策として、内部操作は互換名のまま残す。
- フキダシのウニフラ / 白抜き線は効果線の設定表示を借りているため、効果線専用項目を無条件に表示するとフキダシ側で存在しない設定を参照する。対策として、表示オプションで分離する。
- 効果線とフキダシは保存名が完全一致していない。対策として、保存名を一気に変えず、表示用の対応表で段階的に共通化する。
