# 白抜き線 設定項目の徹底監査 (2026-07-12)

対象: B-MANGA v0.6.483 (コミット 39f7fb0b)。効果線 (effect_type="white_outline") とフキダシ (line_style="white_outline") の白抜き線に関する全設定項目。
方法: 多エージェント監査 (棚卸し 2 → 配線追跡 11 → 突き合わせ 1 → 反証検証 16、計40エージェント)。棚卸しは効果線 111 項目・フキダシ 80 項目。配線判定 204 件、突き合わせ 24 件、broken/not_wired/partial の指摘 16 件を個別に反証検証し 14 件成立・2 件棄却。数値検証は bpy 非依存の生成コア (operators/effect_line_white_outline.py) を直接実行して確認。

## 実装結果 (B-MANGA v0.6.484)

- A-1〜A-5、A-7、A-8、B-1、B-4〜B-6を修正した。B-2・B-3は「短くした線を元の長さまで伸ばす」設定であることを説明へ明記した。
- 新規の白線割合・黒線割合を50%ずつ、自動本数をオン、乱れの最小値を50%、間隔を0.2mmへ統一した。手動本数は指定領域いっぱいへ配置する仕様とし、使われない間隔を無効表示にした。
- フキダシに白線の太さと黒線の太さ割合、効果線に線色を追加した。黒線の重ねる方向、白線・黒線の減衰、白線の入り抜き、不透明度も生成結果へ反映する。
- 詳細設定を幅560の縦長2列へ統一し、白抜き線で機能しない設定を隠した。画面に無かった内側白帯は、ページ出力側からも白抜き線について削除した。
- 旧作品は、当時の初期値と隠し倍率を使って白線太さ・入り抜き・減衰を一度だけ新しい画面値へ変換する。変換済みマーカーにより二重適用を防止した。
- 残件はA-6の画面に出ない「mm指定」保存項目、追加候補D-2の専用乱れシード、D-4の線全体の入り抜き。今回のユーザー操作には影響しないため、低優先度の整理項目として継続管理する。
- Blender 5.1.2で関連テスト14本が失敗0。代表画像3枚と直近修正7項目のAI目視チェックも合格した。

## A. 完全に死んでいる設定 (UI にあるのに効かない)

### A-1【フキダシ・高】白線の「入り (%)」「抜き (%)」が全く効かない
- utils/balloon_flash_effect_line_mesh.py:329-330 で `white_endpoint × UI値 ÷ 100` と乗算される。white_endpoint は **UI 非表示**の `flash_white_line_valley_width_pct` (core/balloon.py、既定 0、flash 切替時も 0 へ初期化) 由来で常に 0。
- 実測: UI 値 100% と 20% で生成結果が完全一致。効果線側は直渡しで正常。
- 修正案: 乗算を撤廃して効果線と同じ直渡しへ (フキダシの白線入り抜きが初めて機能する。既定値の食い違い A-8 と同時に対処)。

### A-2【フキダシ・高】黒線・白線の「減衰」が実質無効 (単位不一致)
- フキダシ側は 0〜1 の FACTOR (core/balloon.py)、生成コア _attenuated_length は「0〜100 の %」として /100 解釈。UI 最大 1.0 でも短縮 1% 未満で知覚不能。
- 実測: 基準長 10mm で att=1.0 (フキダシ最大) → 9.9mm、att=100 (効果線最大相当) → 0mm。
- 修正案: プロパティを効果線と同じ % レンジへ統一 (保存データ移行つき) か、アダプターで ×100 換算。既存 AGENT_INBOX P2 と同件。

### A-3【フキダシ・中】「線の本数」「線の間隔」の行が白抜き線では死んでいる
- `flash_line_count` / `flash_line_spacing_mm`。UI の表示条件は is_flash_line_style だが、ウニフラは別画面のため**実質白抜き線でだけ表示**される。白抜き線の生成・キャッシュ署名・コアのどこからも参照されず、変更しても再構築すら起きない (実利用は旧データ移行のみ)。
- 機能する類似項目 (本数=flash_white_outline_white_line_count / 間隔=flash_white_outline_spacing_mm) と紛らわしい。
- 修正案: line_style==white_outline のとき 2 項目を非表示。

### A-4【フキダシ・高】「入り・抜き」(line_valley_width_pct) の行が白抜き線では無意味
- 白抜き線では generate_flash_strokes_rect_local が黒線用の端点% を計算後に**破棄**しており (`_black_endpoint_pct` をアンダースコアで捨てる)、見た目に影響しない。署名には入るため無意味な再構築だけ発生する。同じ行の「中間線幅」(line_peak_width_pct) は黒線太さの導出に機能する。
- 修正案: 白抜き線では「入り・抜き」を非表示 (黒線ボックスの入り/抜きと機能が重複)。「中間線幅」は機能するので残す (ラベルは「黒線太さ (%)」等への改名を検討)。

### A-5【フキダシ・中】「外側フチ」「内側フチ」が白抜き線では無効なのに表示される
- ビューポートは flash 系分岐でフチメッシュを無条件削除、ページ出力も not is_flash ゲートでスキップ。v0.6.483 で塗り設定は非表示にしたがフチ行が残っていた。
- 修正案: 塗りと同じ has_body_fill 条件で非表示 (enabled/width/color の 6 プロパティ×2 行)。

### A-6【両側・低】入り抜き範囲の「mm 指定」系 12 プロパティが到達不能
- `white/black_inout_range_mode` と `in/out_range_mm` (効果線・フキダシ両側)。生成コアが `range_mode="percent"` をハードコード (operators/effect_line_white_outline.py:556 白 / :608 黒) しており、mode プロパティは一切 getattr されず mm 値も死値。線幅グラフの書き戻しも常に percent へ強制。**UI に enum が露出していない**ため実害はないが、定義・保存・署名・プリセットには乗り続ける死にデータ。
- 由来: v0.6.479 (fedceb15) の「フキダシと効果線の線設定を統一」でコアが percent 固定になった。
- 修正案: percent 専用で確定するなら両側の schema から段階的に削除、mm 対応を復活するならコア 2 箇所を getattr 化。

### A-7【フキダシ・中】ビューポートの「不透明度」が白線に効かない
- ページ出力はレイヤー全体へ乗算され正しいが、ビューポートの白線マテリアルが固定 (1,1,1,1) で opacity 非乗算 → 下げると黒線だけ薄くなり白線が残る。ビューポートと出力の見た目不一致。
- 修正案: 効果線側と同じく white_mat の alpha へ opacity を乗算。

### A-8【フキダシ・中】隠し機能「内側白帯」が出力にだけ描かれる
- `flash_white_line_enabled` (UI 非表示・flash 切替時に強制 ON)。ページ出力では輪郭沿いの内側白帯を描くのに、ビューポートの白帯メッシュ生成は flash 分岐で削除→return しており描かれない。白抜き線では常に「出力にだけ白帯が出る」不一致。
- 修正案: ビューポートにも白帯を出すか、白抜き線では出力側も無効化するかの二択 (ユーザー判断)。

## B. 部分的にしか機能しない設定 (効果線側)

### B-1【中】黒線の「減衰」が方向=内側で完全 no-op
- 内側黒線は offset が白線半幅以下のため offset_from_center が恒等的に 0 → 減衰係数が常に 1。実測: 方向=内側で減衰 0 と 80 の黒線長が全 12 本一致。方向=両側では外側半分のみ減衰。外側は正常。
- 修正案: 内側黒線の正規化基準を「白線端からの深さ」に変える等、意味論を決めて再実装。

### B-2【低】「減衰」の負値 (延長) が既定状態で無効 (白線・黒線とも)
- 実距離クランプ (min(band_length, 実距離)) のため、長さ(%)=100 の既定では負の減衰 (伸ばす方向) にしても見た目不変。長さ<100% に縮めた場合のみ実距離を上限に再伸長する。
- 修正案: 仕様として「延長は元の長さまで」を明文化 (ツールチップ) するか、負レンジを廃止。

### B-3【低】黒線の「長さ変化 (内側/外側)」の 100% 超が既定状態で無効
- 同じ実距離クランプ。長さ(%)<100 か長さ乱れ ON のときだけ 100〜200% が「縮んだ分を取り戻す」範囲で機能する。
- 修正案: B-2 と同じ扱い (ツールチップで明記 or max=100 に制限)。

### B-4【中・両側】雲系形状の「小山高 0%」が実際は 50% で描画される
- balloon_shapes.py の `(sub_h if sub_h>0 else 50.0)` フォールバック。実測: sub_h=0 と 50 の点列が完全一致、1% とは不一致 (0→1% で不連続ジャンプ)。小山幅側にも同型フォールバックあり。**ただし効果線の既定値 (小山幅30/小山高0) の見た目自体がこのフォールバックに依存**しており、単純修正すると既定の雲形が変わる。白抜き線の外端/内端形状に雲系を選んだ場合に影響。
- 修正案: 挙動を変えずツールチップに「0 は自動 (50%)」と明記するのが低リスク。数値どおりに直すなら既定値と保存データの移行が必要。

### B-5【低】「全体回転」の表示条件が 4 経路中 1 経路だけ食い違う
- サイドバーのレイヤー詳細 (インライン表示) だけ白抜き線で非表示、他 (N パネル・右クリック詳細・レイヤーリストのダイアログ) は表示。生成には正しく配線済み (束の角配置の回転追従も確認済み)。
- 修正案: サイドバー側の表示条件を合わせる。

### B-6【低】パス線の「縦横比」「間隔」がリボン (伸ばす) では無効なのに常時表示
- 画像 1 枚を伸ばすモードでは spacing/aspect 未使用 (仕様上妥当)。表示だけの問題。
- 修正案: 伸ばすモード選択時に 2 項目を enabled=False。

## C. 効果線とフキダシの不一致 (デフォルト・レンジ・UI)

| 項目 | 効果線 | フキダシ | 判定 |
|---|---|---|---|
| 白線割合/黒線割合の既定 | 30% / 70% | 70% / 30% | 要検討。各々「従来の見た目」互換由来だが同名設定で真逆 |
| 本数を自動計算の既定 | ON | OFF | 要検討。フキダシ既定では束の太さ・割合・太さ乱れが見た目に影響しない死にスライダー化 |
| 乱れ最小値の既定 | 50% | 100% | 要検討。フキダシは乱れ ON にしても既定では何も起きない |
| 白線・黒線の太さ | mm 直接指定 | UI なし (主線幅×隠しプロパティから導出) | 黒線=主線幅由来は妥当。**白線の太さを変える手段がフキダシに無い** (コアは受理済みで配線のみで露出可能) |
| 共通「入り抜き」ボックス | なし | あり | 効果線に線全体テーパー手段が無い。追加は可能だがフキダシ側の意味論問題の整理が先 |
| 線色 | UI なし (生成は対応済み) | あり | 効果線の白抜き線ボックスに線色を追加するのが自然 (early return が原因で UI から欠落) |
| 黒線 幅変化の max | 200% | 300% | 効果線を 300 へ広げて統一が低リスク |
| 間隔 (mm) の既定 | 0.2 | 0.25 | 実害小。統一するなら要移行 |
| 重ねる方向 | 黒線ボックス内「方向」 | 共通行のラベル無しドロップダウン | フキダシ側にラベルを付け「重ねる方向」へ用語統一を推奨 |
| 乱れのシード | 「効果線を追加」で引き直し可 | 形状シード共用。矩形/楕円ではUIが出ず振り直し不可 | 白抜き線の乱れ ON 時にシード行を表示するか、線専用シードを新設 |
| 角度のレンジ | 制限なし | soft ±360 | 実害なし。効果線にも soft ±360 を付けると操作感が揃う |
| 角度のリンク共有 | **共有対象外** (白抜き線系で唯一) | — | ただし反証検証で「意図的除外を明示するテストが存在する」ことを確認 (test/blender_effect_line_frame_spacing_check.py:718-741)。現状維持が正 |

### 妥当と判定した不一致 (対応不要)
- フキダシに外端/内端形状・束の配置・位置以外の形状系が無い: フキダシの白抜き線は「本体の外接楕円」を内端とする設計で一貫。
- フキダシの黒線太さが「線幅」由来: 主線と太さを揃える設計として妥当。
- 効果線に塗り・フチが無い: フチはフキダシ本体輪郭の付属物。
- v0.6.483 で追加した束の配置がフキダシ非表示: 内端が楕円固定で角が存在しないため妥当 (保存データに corner が入っても等間隔へ安全にフォールバックすることを確認済み)。

## D. あるべきなのに無い設定 (追加候補)

1. **フキダシ: 白線の太さ** — コアは white_outline_white_brush_mm を受理済み。アダプターと UI の配線のみで露出できる (最優先候補)。
2. **フキダシ: 線の乱れ用シード** — コアは seed 引数を受けるだけなので専用シード新設は容易。
3. **効果線: 線色** — 生成は黒線ロールに line_color 適用済み。UI 追加のみ。
4. **効果線: 共通入り抜き (線全体のテーパー)** — コアのロール別 _apply_inout_profile で技術的に可能。ただしフキダシ側実装の意味論問題 (A-1 ほか) の修正を先行させること。

## E. 修正の推奨順序

- **第1弾 (死んだ UI の修理・低リスク)**: A-3, A-4, A-5 の非表示化 / B-5 の表示条件統一 / B-6 の enabled 制御
- **第2弾 (機能修理・要移行判断)**: A-1 白線入り抜きの直渡し化 / A-2 減衰の単位統一 / A-7 ビューポート不透明度
- **第3弾 (整合性・要ユーザー判断)**: C 表のデフォルト統一 (特に「本数を自動計算」OFF 時の死にスライダー対策) / D の追加候補 / A-8 白帯の扱い / A-6 mm 範囲系の削除か復活
