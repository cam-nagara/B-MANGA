# B-MANGA / B-MANGA Render 仕様定点観測 差分照合 2026-05-05

## 1. 判定基準

- OK: 設計意図に対応する実装または実機確認がある
- 部分対応: 土台はあるが、仕様の全量再現には不足が残る
- 未達: 設計意図に対し、現在の実態で不足が明確
- 要実機目視: コードや自動テストだけでは、最終挙動を保証できない

## 2. B-MANGA と B-MANGA Render の分離

| 設計意図 | 実態 | 判定 |
|---|---|---|
| B-MANGA 本体はページ一覧ファイルでの作画とコマ用blendファイルでの 3D 配置まで | B-MANGA 本体は旧「書き出し」パネルと書き出しオペレータを登録しない | OK |
| 出力系機能は B-MANGA Render へ分離 | B-MANGA Render は独立アドオンとして存在し、登録分離テストも通る | OK |
| B-MANGA リポジトリ内で B-MANGA Render を開発できる | `addons/b_manga_render/` に独立 manifest と実装がある | OK |
| B-MANGA 本体から出力系を完全に移す | 旧「書き出し」パネル / 書き出しオペレータは登録されず、B-MANGA 起動時にも旧出力UIモジュールを import しない。B-MANGA Render 側で出力プリセットを登録する | OK |

メモ:

- 旧出力ファイルそのものは互換 / 参照用に残るが、B-MANGA 本体の起動・登録経路からは外した
- サムネイル / プレビュー生成など、B-MANGA 本体の編集体験に必要な内部画像生成は分離対象外として扱う

## 3. B-MANGA Render のカード型出力プリセット

| 設計意図 | 実態 | 判定 |
|---|---|---|
| 出力プリセットを自由に組み立てられる | プリセットとカードをシーン状態として持ち、カード追加 / 削除 / 上下移動ができ、UIトグルも実機確認済み | OK |
| Photoshop のアクションのように上から順に実行 | 実行エンジンはカードを上から順に実行する | OK |
| カードをクリックで選択 | カード選択オペレータがあり、UI から押せる | OK |
| ダブルクリックで設定ダイアログ | 短時間の同一カード再クリックで設定ダイアログを開く | OK |
| リスト右の「＋」でカード追加 | カード一覧右に追加ボタンがある | OK |
| 「キャラ」「背景」など元プリセットをすべて再現 | 初期プリセット名は 37 件あり、未設定カードは解消済み。全37プリセット / 66レンダーを監査用マーカーで実機レンダーし、AI目視した | OK |

不足:

- 参照元アドオンの全ボタンと完全一致する実レンダー比較は未実施
- c00.blend 実素材そのものの Cycles 本番レンダー比較は、ドライバー警告と eeVR 未登録の整理後に追加確認が必要

## 4. eeVR / 魚眼レンダリング

| 設計意図 | 実態 | 判定 |
|---|---|---|
| eeVR の魚眼レンダリングを B-MANGA Render から扱う | 魚眼モード時は eeVR、通常モード時は通常レンダーへ分岐するカードがある | 部分対応 |
| Blender の魚眼レンダリングにぴったり合わせる | c00.blend のカメラはパノラマ型で魚眼画角を持つ | 部分対応 |
| eeVR とカメラ操作＆出力プリセットの関係性を保つ | 橋渡し関数はあるが、参照元と完全一致する検証は未完了 | 未達 |
| eeVR 本体が使える状態で実行できる | B-MANGA Render 側の魚眼設定は登録されるが、eeVR 本体は外部アドオン登録が必要 | 部分対応 |

重要な差分:

- c00.blend には eeVR の保存済みカスタムプロパティ名はあるが、eeVR 本体を登録しない限り実行先は持てない
- B-MANGA Render は eeVR オペレータを呼ぶ橋渡しであり、eeVR 本体の登録や同梱状態までは保証していない
- 魚眼レンダリングの「ぴったり合う」結果は、実レンダー画像比較が必要

## 5. c00.blend 連動

| 設計意図 | 実態 | 判定 |
|---|---|---|
| B-MANGA Render が c00.blend と完全に連動する | c00.blend を開いて、プリセット要求名との照合テストがある。全プリセットのカード列は実行可能なレンダー状態まで確認した | 部分対応 |
| c00.blend 側にも不足がないか確認する | 監査スクリプトで不足名とプロパティを検出できる | OK |
| 出力プリセットの要求構造が c00.blend に揃っている | 主要コレクション、出力要素、出力フレームの要求名欠落は解消済み | OK |

未達 / 要確認:

- AOV入力対象のソケット数が 0
- eeVR 本体のプロパティは、eeVR アドオン未登録状態では使えない
- カメラ背景画像に、画像名とファイルパスが空の項目がある
- c00.blend を開いた時点で、一部ドライバーの数式エラー / ゼロ除算が出る
- c00.blend の保存状態では一部ビューレイヤーの合成出力が無効で、監査前は「すべて」「ページ」「統合系」が有効ビューレイヤー 0 件のままレンダーへ進む経路があった。B-MANGA Render v0.1.3 でレンダー直前に有効ビューレイヤーを補正するよう修正した

判断:

- c00.blend は B-MANGA Render の要求する大枠をかなり持っている
- ただし「完全連動」と呼ぶには、AOV入力実体、eeVR 本体登録、元アドオンとのレンダー結果比較を確認する必要がある

## 7. 仕様定点観測としての結論

現時点の実態は、B-MANGA 本体と B-MANGA Render の分離開始としては成立している。

優先度高:

1. eeVR 本体が B-MANGA Render から実際に使える状態を作る
2. 元アドオンの各出力プリセットと、B-MANGA Render の実素材 Cycles 本番レンダー結果を比較する
3. AOV入力対象が素材入りのコマ用blendファイルで実際に見つかるか確認する

優先度中:

1. 旧出力ファイルの物理削除 / 履歴退避を行うか、互換参照として残すかをリポジトリ運用上で決める
2. c00.blend のドライバーエラーを、B-MANGA Render 連動上の支障があるものと、外部アセット由来で許容できるものに分類する

残るリスク:

- 現在の Codex セッションでは Computer Use が使えないため、Blender GUI を直接クリック / ドラッグするライブ目視は未実施。B-MANGA 本体の該当項目は Blender 実機 API と診断画像の AI 目視で代替確認済み
- B-MANGA Render の初期プリセットは、未設定カードと c00.blend 側の要求名欠落は解消し、全プリセットの監査用実レンダーは通ったが、元アドオンの全副作用を完全再現しているとはまだ言えない
- eeVR 本体が未登録の状態では、魚眼レンダリングの完全連動は未達
