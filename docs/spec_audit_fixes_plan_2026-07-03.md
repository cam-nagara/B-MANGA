## 実行順序と内容

### Phase 1: 実機確認（修正の要否を確定させる）

照合レポート B 群を Blender 5.1 実機（`C:\Program Files\Blender Foundation\Blender 5.1\blender.exe`）で確認し、各項目を「達成 / 要修正」に振り分ける。確認スクリプトは `_verify/` に「日付_機能名」で置く。

1. 枠線カット: テストページで2回カットし、(a) レイヤーリストの並びが操作前の並びを維持するか（`sync_layer_stack_after_data_change` の `align_coma_order=True` の実挙動）、(b) 親子階層維持、(c) 右上前面/左下背面、(d) 自動再採番が毎回走るか。
2. B-MANGA Line: 円柱＋カメラのシーンで、円柱のフチ内部線に中間頂点の線幅調整が効くか（`vertex_analysis` の角度判定と「カメラビュー上の端から端まで」の扱い）。
3. B-MANGA Line: キューブ×円錐で交差線が (a) 交差部分に出る、(b) 交差以外に出ない、(c) 1本になる、(d) 線幅・中間頂点グラフが効く。
4. Tokyo0004 で「レンダリング範囲内を選択」→アウトライン+内部線+交差線オン→「ラインを適用」の所要時間（2026-07-01 に30分超の回帰報告。1分前後が目標値）。
5. フキダシ: ウニフラ選択で新規作成した時に「始点乱れ」がオンになるか（`apply_balloon_line_style_defaults`）。
6. フキダシ: レイヤー「複製」（Ctrl+Shift+D 含む）でシード（`shape_seed`・`line_shape_seed`）が変わらないか。
7. 魚眼モードのグレー帯（レンダリング領域外マスク）が領域の位置・ズームに連動するか。※該当コードが静的に見つかっていないため未実装の可能性が高い。
8. ページファイル: 編集中ページの用紙ガイド線幅が他ページと同じか。作品ファイルでページ選択して真っ白にならないか。ハンドルが対象より一回り大きいか。
9. ツールプリセット: 「なめらか自由形状」で作成→確定→プリセットを「標準」等へ切替できるか。パネル上のカーソル残留がないか。
10. B-MANGA Line: `BMANGA_LINE_OT_refresh_linked` / `BMANGA_LINE_OT_apply_active_to_linked` をコマファイル相当（リンク読み込み）で実行し、一括上書き/補正が機能するか。

### Phase 2: ユーザー判断が必要な項目（着手前に選択肢を提示）

- **交差線「作成方式」ドロップダウン**（`intersection_method` EnumProperty と panels の該当 prop 行）: 推奨は「UIから削除し SHELL（ライン素材・高速）固定。BOOLEAN/SDF 実装は当面コード内に残すが露出しない」。ユーザーが (b) 残す を選んだ場合は既定 SHELL のまま何もしない。
- **アウトライン初期値**（`outline_enabled` default）: 現行オンのまま（推奨）か、オフへ変更か。
- **板ポリ除外の初期値**（`exclude_sheet_meshes` default）: 現行オフのまま（最新指示準拠・推奨）か、オンへ変更か。
- **B-MANGA Render タブの常時表示+案内表示**: 現行挙動で確定してよいか。

### Phase 3: コード修正（Phase 1 で要修正と確定したもの＋確定済み未達）

1. **シード一元化の完成**: `line_shape_seed`（乱れシード）を `shape_seed` に統合するか、`shape_seed` から導出する形に変更。複製時（layer_stack_duplicate 系オペレータ）にシード値を保持するよう明示コピー。既存ファイルの `line_shape_seed` 値は読み込み時に一度だけ移行。
2. **ウニフラ始点乱れ**: Phase 1-5 で新規作成時にオフだった場合、ウニフラ既定値の適用漏れを修正（プロパティ default 変更ではなく、ウニフラ選択時の defaults 適用経路を確実化。効果線側の既定と食い違わないこと）。
3. **円柱フチ内部線の中間頂点調整**: Phase 1-2 で効かなかった場合、鋭角の無い連続内部線を「カメラビュー上の端から端まで」を一本として中間頂点を割り当てる処理を `vertex_analysis` / `inner_lines` に追加。
4. **魚眼グレー帯の連動**: Phase 1-7 で非連動なら、帯の描画をレンダリング領域（カメラフレーム）座標から算出する方式へ修正。
5. **枠線カットの並び順**: Phase 1-1 で数値順に並べ替わってしまう場合、カット時の `align_coma_order` を「分割元の位置に新コマを挿入する」挙動へ変更（他操作の呼び出し元に影響しないようカット専用の引数で）。
6. **交差線方式UIの削除**（Phase 2 でユーザーが削除を選んだ場合のみ）。
7. **AOV版ラインのみ表示コードの整理**（低優先・任意）: `viewport_aov.py` を無効化ユーティリティのみに縮小するか、削除して set_line_only 側の後方互換処理に置き換え。
8. 修正のたびに「徹底チェック」（変更起点の関数チェーン追跡）を行い、`test/` の関連チェックスクリプトを更新。CHANGELOG 更新。コミットはユーザー指示に従う。

### Phase 4: 実機再確認と記録

- Phase 3 の修正項目を Blender 実機で再確認し、結果を `_verify/` と本計画書末尾に記録。
- 完了後、AGENT_INBOX.md の該当項目を「完了」へ移動し、照合レポートの B/C 群のステータスを追記（照合レポート本文は上書きせず、本計画書に結果を書く）。
