# 効果線・フキダシ Geometry Nodes 化計画

作成日: 2026-05-21
対象 Blender: 5.1.1

## 背景

効果線、フキダシ、ウニフラッシュは、B-Name パネルから設定を変えられる実体表示として維持しつつ、Geometry Nodes を通した生成状態へ移す必要がある。

2026-05-21 時点の既存実装は、効果線は Grease Pencil ストローク、フキダシとウニフラッシュは Mesh Object として B-Name 側で形状を同期している。いきなり Geometry Nodes のみへ置き換えると、選択、アウトライナー階層、マスク、書き出し、既存ファイル再読込に影響する。

## Phase 1: ノード管理とパネル値同期

- 既存の効果線・フキダシ・ウニフラッシュ実体へ B-Name 用 Geometry Nodes を付与する。
- B-Name パネルの値をノード入力へ同期する。
- 既存の見た目、選択、書き出し、保存データは壊さない。
- 実機テストで、効果線・フキダシ・ウニフラッシュそれぞれにノードが付き、パネル相当の値変更がノード入力へ反映されることを確認する。

## Phase 2: ノード内部生成への移行

- 効果線の代表形状から順に、ノード内部で線群を生成する。
- フキダシは矩形・楕円など単純形状から移行し、雲、トゲ、ウニフラッシュは段階的に移す。
- 旧実体は当面、編集・互換・書き出しのための参照形状として残し、表示結果をノード出力へ切り替える。
- 保存済み作品を開いたとき、旧実体からノード入力を再構築できる修復処理を用意する。

2026-05-21 v0.6.040 時点:

- 効果線とウニフラッシュは、入力形状を使わず Geometry Nodes 内部で線群を生成する。
- 楕円フキダシは Geometry Nodes 内部で塗りと輪郭を生成する。
- 雲、トゲ、その他の複雑フキダシ形状は、編集・互換・書き出しを壊さないため、当面は既存形状を参照する。
- 古い空に近いノードグループは、アドオン登録時または対象作成時に現在仕様の生成ノードへ作り直す。

## Phase 3: 旧生成経路の縮小

- 選択、マスク、書き出しがノード出力を正しく扱えることを確認した後、B-Name 側の直接形状生成を編集用データ生成に限定する。
- 不要になった旧描画経路を削除する。

## 検証方針

- ノード付与と入力同期: `test/blender_geometry_nodes_bridge_check.py`
- 既存フキダシ・ウニフラッシュ形状: `test/blender_balloon_uni_flash_check.py`
- 効果線表示・マスク回帰: `test/blender_effect_line_mask_visibility_check.py`
- 用紙・実体表示の安全性: `test/blender_real_object_safety_check.py`
