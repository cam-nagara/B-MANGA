# 効果線・フキダシ Geometry Nodes 化計画

作成日: 2026-05-21
対象 Blender: 5.1.1

## 背景

効果線、フキダシ、ウニフラッシュは、B-MANGA パネルから設定を変えられる実体表示として維持しつつ、Geometry Nodes を通した生成状態へ移す必要がある。

2026-05-21 時点の既存実装は、効果線は Grease Pencil ストローク、フキダシとウニフラッシュは Mesh Object として B-MANGA 側で形状を同期している。いきなり Geometry Nodes のみへ置き換えると、選択、アウトライナー階層、マスク、書き出し、既存ファイル再読込に影響する。

## Phase 1: ノード管理とパネル値同期

- 既存の効果線・フキダシ・ウニフラッシュ実体へ B-MANGA 用 Geometry Nodes を付与する。
- B-MANGA パネルの値をノード入力へ同期する。
- 既存の見た目、選択、書き出し、保存データは壊さない。
- 実機テストで、効果線・フキダシ・ウニフラッシュそれぞれにノードが付き、パネル相当の値変更がノード入力へ反映されることを確認する。

## Phase 2: ノード内部生成への移行

- 効果線の代表形状から順に、ノード内部で線群を生成する。
- フキダシは矩形・楕円など単純形状から移行し、雲、トゲ、ウニフラッシュは段階的に移す。
- 旧実体は当面、編集・互換・書き出しのための参照形状として残し、表示結果をノード出力へ切り替える。
- 保存済み作品を開いたとき、旧実体からノード入力を再構築できる修復処理を用意する。

2026-05-21 v0.6.040 時点:

- 効果線とウニフラッシュは、入力形状を使わず Geometry Nodes 内部で線群を生成する。
- 楕円フキダシは Geometry Nodes 内部で塗りと輪郭を生成する。
- 雲、トゲ、その他の複雑フキダシ形状は、編集・互換・書き出しを壊さないため、当面は既存形状を参照する。
- 古い空に近いノードグループは、アドオン登録時または対象作成時に現在仕様の生成ノードへ作り直す。

2026-05-21 v0.6.041 時点:

- フキダシの見える本体は空の実体になり、表示結果は Geometry Nodes の出力だけになった。
- Geometry Nodes を非表示にしたとき、B-MANGA 側で作った下敷き形状が画面に残る経路は削除した。
- 複雑フキダシ形状は互換性維持のため非表示の参照形状を持つが、画面に表示される形状は Geometry Nodes が参照形状を読み込んだ出力に限定する。
- ページ一覧のフキダシ画面描画は停止し、B-MANGA は Blender 実体と Geometry Nodes を操作する側に寄せた。
- フキダシと効果線の作成・更新後、コマ枠線が内包要素より前面になる表示順へ即時更新する。

2026-05-22 v0.6.055 時点:

- 雲、もやもや、トゲ系の形状係数を共通グループ化し、フキダシ本体、効果線の始点形状、効果線の終点形状が同じ計算を使う構成へ更新する。
- フキダシ本体、効果線の始点形状、効果線の終点形状に同じような波形ノードを重複配置しない。共通項目の追加・修正は共通グループに集約する。
- フキダシ・効果線のクリックだけでは実体を作成しない。ドラッグ中は B-MANGA の範囲表示だけを出し、ドロップ時にその範囲の実体を作成する。
- コマ外かつページ内で作成したフキダシ・効果線は、ページ上ですべてのコマより前面に置く。ページ外にも作成できる。
- 移植棚卸し用の実機テストを追加し、フキダシ・効果線のノード入力欠落、生成結果への接続、共通形状グループの利用数を確認する。
- B-MANGA 側に残す処理は、実体作成、UI 値の同期、素材・表示状態の同期、選択・階層・マスク管理に限定する。形状、線数、距離指定の密度補正、入り抜き範囲などの生成計算は Geometry Nodes 側へ寄せる。
- 色、不透明度、塗りマテリアル、グラデーションなど Blender 素材として管理する項目は、生成形状ノードとは別に素材同期として監査する。モディファイア上に表示する値は、表示に効くことを検証対象にする。

## Phase 3: 旧生成経路の縮小

- 選択、マスク、書き出しがノード出力を正しく扱えることを確認した後、B-MANGA 側の直接形状生成を編集用データ生成に限定する。
- 不要になった旧描画経路を削除する。
