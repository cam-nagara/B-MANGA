# B-MANGA Line 全形状・全線種の線幅検証計画（2026-07-06）

## 目的

平面、立方体、UV球、円錐、円柱、六角柱、トーラスが互いに一部重なる状態で、アウトライン、稜谷線、交差線、選択線の線幅が600dpiの指定線幅どおりになるかを、Blender実機で数値測定と画像目視の両方で確認する。

## 共通設定

- 線幅: アウトライン、稜谷線、交差線、選択線をすべて `0.3mm` に統一する。
- 期待線幅: `0.3mm * 600dpi / 25.4 = 7.09px`。
- オフセット: 4線種すべて `0.000`。
- 線幅基準距離: カメラから原点までの距離を全オブジェクトへ設定する。
- 面の厚みを均一に: オン。
- 線幅の均一化（頂点単位）: オン。
- 線幅の均一化（オブジェクト単位）: オン。ただし頂点単位の均一化が優先される実装なので、同時オン時の最終線幅を確認する。
- 作成範囲制限: 4線種すべてオフ。
- カメラ範囲外を非表示: 検証中は全対象が確実に見えるようオフ。
- 交差線の作成方式: ライン素材（高速）固定。
- レンダー: 600dpi換算が見える解像度で、白地に線種別カラーを出す。

## 実行手順

1. Blender 5.1実機で専用検証スクリプトを実行する。
2. スクリプトで21ペアをグリッド配置し、各ペア内の2形状を重ねる。
3. 各オブジェクトに4線種を有効化し、Freestyleマーク辺を最低1本以上作って選択線も出るようにする。
4. B-MANGA Lineを適用し、カメラ補正を更新する。
5. 線種ごとに線幅設定、補正後モディファイア値、頂点グループ有無、生成線モディファイア有無、交差線ターゲット有無、各組み合わせの画像切り出し範囲をJSONへ出す。
6. 4線種を色分けした全体画像と、線種別の確認画像を出す。
7. 画像解析で、各組み合わせ・各線種の切り出し範囲から線マスクを作り、水平・垂直・斜め2方向の断面幅を測定し、期待値 `7.09px` に最も近い断面を代表値にする。
8. AI目視で、各線種が同じ太さに見えるか、局所的な太り・細り・消失・破綻がないか確認する。
9. 数値測定で期待線幅 `7.09px` に対し、1px未満のズレを目標、1〜2pxを注意、3px以上を不合格として扱う。

## 合格条件

- 21組み合わせすべてで交差線が生成され、交差部分から大きく外れない。
- 4線種すべてが表示される。
- 4線種の設定線幅はすべて `0.3mm`、600dpi換算で `7.09px`。
- オフセットはすべて `0.000`。
- 面の厚みを均一に、線幅の均一化（頂点単位）、線幅の均一化（オブジェクト単位）がオン。
- 画像解析で代表ピクセル幅が丸め後 `5px` 〜 `9px` に収まる。`4px` 以下または `10px` 以上は3px以上のズレとして不合格。
- 目視画像で線種間の明らかな線幅差、急激な細り、消失、ガタつきがない。

## 2026-07-06 実行結果

- Blender 5.1.2実機で `test/blender_b_manga_line_width_all_shapes_visual_check.py` を実行し、21組み合わせの全体画像・線種別画像・測定用JSONを `_verify/2026-07-06_bml_width_all_shapes_strict/` へ出力した。
- `test/b_manga_line_width_pixel_analyze.py _verify/2026-07-06_bml_width_all_shapes_strict` で、21組み合わせ × 4線種 = 84項目をピクセル測定した。
- 期待線幅は `7.087px`、合格範囲は `5px` 〜 `9px`。
- 測定結果は `failures=0`。3px以上のズレは検出されなかった。

## 不合格時の対応

- 特定線種だけ太さが異なる場合は、線種別の補正経路を確認する。
- 特定形状だけ太さが異なる場合は、非均一スケール・評価後メッシュ・頂点グループ重みの扱いを確認する。
- 交差線だけ異なる場合は、ライン素材（高速）のターゲット線幅受け渡しと交差線チューブ半径を確認する。
- 修正後、同じ21組み合わせを再実行してJSONと画像を更新する。
