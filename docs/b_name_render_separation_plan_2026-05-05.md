# B-Name-Render 分離計画

作成日: 2026-05-05
対象 Blender: 5.1.1
セッション種別: 開発セッション

## 1. 確定した範囲

B-Name 本体は次の範囲に限定する。

- ページ一覧ファイルでの作画
- コマ用blendファイルでの 3D ビューポート上モデル配置

次の機能は B-Name 本体から分離し、別アドオン **B-Name-Render** に移す。

- 出力プリセット
- レンダーパス切替
- ファイル出力ノード制御
- Pencil+4 線画出力
- 魚眼レンダリング
- 出力解像度モード
- 出力用カメラ設定
- PSD / PDF / PNG / JPEG / TIFF などの完成画像書き出し

## 2. 参照元

分離先の B-Name-Render は次を参照元として扱う。

- `D:\Develop\Blender\暫定安定版_編集禁止\カメラ操作＆出力プリセット.py`
- `D:\Develop\Blender\暫定安定版_編集禁止\eeVR-master_ミウラ修正 (19).zip`

`eeVR-master_ミウラ修正 (19).zip` は GPL-3.0 の eeVR を含むため、B-Name-Render のライセンスは `GPL-3.0-or-later` とする。B-Name 本体は既存の `GPL-2.0-or-later` のまま維持する。

## 3. B-Name-Render の基本方針

B-Name-Render はリポジトリ内の独立アドオンとして開発する。

配置:

```text
addons/
  b_name_render/
    blender_manifest.toml
    __init__.py
    core.py
    preset_library.py
    command_runner.py
    eevr_bridge.py
    operators.py
    panels.py
```

表示名は **B-Name-Render**。Python パッケージ名と Extension ID は `b_name_render`。

## 4. 出力プリセットの新設計

従来の「キャラ」「背景」などのボタンは、すべて **上から順に実行するカード型コマンドリスト** として表現する。

カードの代表コマンド:

- 出力状態を退避して初期化
- 出力状態を復元
- ビューレイヤーを有効化
- コレクションを除外 / 除外解除
- ノードをミュート / ミュート解除
- ファイル出力ノードをフレーム名で切替
- AOV 入力を設定
- 出力ファイル名を設定
- 出力フォルダを設定
- レンダーを実行
- eeVR を設定
- eeVR で魚眼レンダリング
- eeVR の方向画像だけをレンダリング
- eeVR の方向画像を合成
- 画像ノードを再読み込み

UI:

- カードをクリックすると選択
- 同じカードを短時間に 2 回クリックすると設定ダイアログを開く
- リスト右の `+` ボタンでカードを追加
- 上下移動、削除、プリセット初期化、プリセット実行を提供する

## 6. B-Name 本体から外すもの

初期分離では、B-Name 本体の「書き出し」パネルと書き出しオペレータの登録を停止する。コードファイルはすぐには削除しない。B-Name-Render 側の実装が十分に安定した段階で、移植済みコードを削除する。

残すもの:

- ページ一覧での作画
- コマ用blendファイルへの移動
- コマ用blendファイルの 3D 配置
- 作画のために必要な下絵表示やカメラビュー補助
- コマサムネイル / 高品質プレビュー生成

移すもの:

- 完成画像の書き出し
- PSD / PDF 出力
- 出力プリセット
- 出力用魚眼レンダリング
- Pencil+4 出力連動

## 7. 実装段階

### Phase R1: 土台

- B-Name-Render の独立アドオン土台を作る
- カード型プリセット UI を作る
- 参照元ボタン名を初期プリセットとして登録する
- B-Name 本体の書き出し UI / オペレータ登録を止める

### Phase R2: 出力プリセット再現

- 参照元の各ボタンをカード列として精密化する
- ノード、ビューレイヤー、コレクション、ファイル出力ノードの操作を実機で確認する
- 実行前後の状態復元漏れを潰す

### Phase R3: eeVR 統合

- eeVR の設定項目を B-Name-Render 側に持つ
- 魚眼レンダリング、方向画像レンダリング、パノラマ合成を B-Name-Render から実行する
- `カメラ操作＆出力プリセット.py` 側の魚眼時処理との関係を保つ

### Phase R4: B-Name 書き出し系の移植完了

- B-Name の PSD / PDF / ページ画像書き出しを B-Name-Render へ移す
- B-Name 本体から未使用の出力コードを削除する
- B-Name 本体の仕様正本から書き出し要求を完全に B-Name-Render へ移管する

## 8. テスト方針

- B-Name 本体: register 時に「書き出し」パネルと書き出しオペレータが登録されないことを確認する
- B-Name-Render: register、初期プリセット作成、カード追加、カード編集、プリセット実行の空実行を確認する
- 参照元再現: 各プリセットを実行し、ビューレイヤー / ノード / 出力ファイル名 / 魚眼時の切替が参照元と一致するか実機で確認する

## 9. 注意点

- 参照元の単一スクリプトには 3000 行を超える処理があるため、B-Name-Render では巨大ファイル化させず、カード定義・実行エンジン・eeVR 連携・UI に分ける
- 参照元はグローバル変数で状態を持つが、B-Name-Render では実行ごとにセッション状態を作り、終了時に復元する
- CSV を生成する場合は BOM 付き UTF-8 を使う
- ユーザー向け表示は Blender UI 上の表記値を基準にする
