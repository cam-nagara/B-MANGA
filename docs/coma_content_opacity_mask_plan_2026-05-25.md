# コマ内容の不透明度マスク方式 計画書

作成日: 2026-05-25  
対象: B-MANGA / B-MANGA Render
状態: 実装前計画

## 1. 目的

コマ内のフキダシ、効果線、グリースペンシル、ラスター、画像、テキストなどを、コマ形状で物理的に切断せず、コマごとの不透明度マスクで表示・レンダリングする方式へ切り替える。

これにより次を解決する。

- 輪郭ぼかしのコマ内で、フキダシや効果線がコマ境界で不自然に切れる問題
- フキダシのフチや線がコマ枠との接点で細くなる、消える、はみ出す問題
- 効果線やフキダシの実体形状を壊してしまい、あとから編集しにくくなる問題
- PSD 出力時に、コマ内レイヤーを個別に破壊的マスクする必要がある問題

## 2. 基本方針

### 2.1 切らずに透明化する

- コマ内の作画要素は、元のカーブ、メッシュ、グリースペンシル、画像を壊さない。
- コマ外にはみ出した部分は、コマ単位の不透明度マスクで透明にする。
- 輪郭ぼかしがある場合は、コマ境界のぼかし幅に合わせて、線、塗り、フチ、効果線をすべて同じマスクでフェードさせる。
- マスクは Blender ファイル内に保存し、B-MANGA が無効でもレンダリング結果が保たれる状態を目標にする。

### 2.2 コマ単位のマスクを正本にする

- マスクはコマごとに 1 つ作成する。
- マスク名はページとコマの両方を含める。
- 表示ページ番号が変わるページ入れ替えにも対応するため、見える名前には現在のページ番号を含め、内部参照にはページ ID とコマ ID を保持する。
- 例: `コマ内容マスク_p0001_0001_c04_600dpi`
  - `p0001`: ページ ID
  - `0001`: 現在のページ番号
  - `c04`: コマ ID
  - `600dpi`: マスク解像度

### 2.3 PSD はコマフォルダのレイヤーマスクにする

- PSD 出力では、コマごとのレイヤーフォルダにコマ内容マスクをレイヤーマスクとして付ける。
- フキダシ、効果線、グリースペンシル、ラスターなど個別レイヤーには、原則としてコマ用レイヤーマスクを付けない。
- コマ外またはページ直下の要素は、コマフォルダ外の通常レイヤーとして扱い、コマ内容マスクを付けない。
- 輪郭ぼかしのコマは、PSD のコマフォルダマスクも同じぼかしを持つ。

## 3. 対象要素

### 3.1 マスク対象

コマ配下に所属している次の要素は、同じコマ内容マスクを共有する。

- フキダシ
- 効果線
- グリースペンシル
- ラスター
- 画像
- テキスト
- その他、コマ内レイヤーとして扱われる表示要素

### 3.2 マスク対象外

- ページ直下の作画要素
- ページ外の作画要素
- 用紙
- 用紙ガイド線
- セーフライン外塗り
- コマ枠線
- コマプレビュー

コマ枠線は常にコマ内要素より前面に表示される。コマ内容マスクでコマ枠線を透明化しない。

## 4. 親子関係と移動への対応

### 4.1 所属の決定

各作画要素のマスクは、現在の所属先から決める。

- コマ内: そのコマのコマ内容マスクを適用
- ページ直下: コマ内容マスクを解除
- ページ外: コマ内容マスクを解除

座標だけで所属を推測しない。レイヤーリストやアウトライナー上の所属を正とする。

### 4.2 別コマへの移動

フキダシ、効果線、グリースペンシル、ラスター、画像、テキストなどを別コマへ移動した場合:

- 旧コマのマスク参照を外す。
- 新コマのマスク参照を付ける。
- オブジェクトの形状や制御点は変更しない。
- 表示順は移動先コマのレイヤー順に従う。

### 4.3 別ページへの移動

別ページのコマへ移動した場合:

- 新しいページ ID とコマ ID を参照するマスクへ差し替える。
- 元ページのマスク画像を誤って使い続けない。
- ページ番号が同じに見える場合でも、内部のページ ID を基準に区別する。

### 4.4 ページ外への移動

ページ外へ移動した場合:

- コマ内容マスクを完全に解除する。
- コマの輪郭ぼかしやコマ外クリップの影響を受けない。
- ページ外で作成したフキダシや効果線も通常どおり選択・移動できる。

### 4.5 ページ入れ替え

ページ順を入れ替えた場合:

- マスクの内部参照はページ ID とコマ ID で保持するため壊れない。
- 表示名や保存名に含まれるページ番号は、次回マスク再生成時に現在のページ番号へ揃える。
- 旧ページ番号名のマスク画像が残った場合は、孤立マスクとして削除対象にする。

### 4.6 コマ形状変更

コマの頂点、輪郭ぼかし、白フチ、枠線、魚眼などを変更した場合:

- コマ内容マスクを再生成する。
- コマ配下の作画要素は再生成しない。
- 作画要素側のカーブやグリースペンシルの制御点を壊さない。

## 5. マスク画像の生成

### 5.1 解像度

- 既定は作品の解像度と DPI を基準にする。
- 600dpi での線幅やぼかし幅と整合するよう、mm から px へ変換する。
- 低解像度プレビュー用に縮小版を持つ場合でも、レンダリング・PSD 用の正本マスクとは分ける。

### 5.2 アルファ値

- コマ内は不透明。
- コマ外は透明。
- 輪郭ぼかしがある場合、境界から指定幅ぶん滑らかに透明へ移行する。
- 輪郭ぼかしのカーブを持つ場合、そのカーブをマスク画像のアルファへ反映する。

### 5.3 保存場所

ページ一覧ファイル内で使うマスク画像は、可能な限り Blender 画像データとして保持する。

外部ファイルとして保存する場合は、作品フォルダ内の管理領域に置き、ファイル名にはページ ID、表示ページ番号、コマ ID、解像度を含める。

## 6. Blender 表示・レンダリングでの適用

### 6.1 カーブ・メッシュ

- フキダシ、効果線、画像平面、テキスト平面など、マテリアルでアルファを扱える要素は、マテリアル側でコマ内容マスクのアルファを掛ける。
- 既存の線色、塗り色、不透明度、グラデーション、フチ、多重線の結果に、最後にコマ内容マスクを掛ける。
- マスクのために作画形状を切断しない。

### 6.2 グリースペンシル

- Blender 標準のグリースペンシルで画像アルファマスクを直接掛けられない場合は、表示・レンダリング用の補助実体を用意する。
- 元のグリースペンシルは編集用の正本として残す。
- PSD 出力ではコマフォルダのマスクで最終的に切られるため、個別のグリースペンシルレイヤーに同じマスクを重複して付けない。

### 6.3 ラスター

- ラスターは画像のアルファとコマ内容マスクを掛け合わせる。
- 編集用画像を破壊しない。
- コマ移動時はマスク参照だけを差し替える。

### 6.4 用紙ガイド線とセーフライン外塗り

- 用紙ガイド線とセーフライン外塗りはコマ内容マスクの対象外。
- これらは常に前面表示の設計を維持する。

## 7. PSD 出力

### 7.1 コマフォルダ構成

- コマごとにレイヤーフォルダを作る。
- そのフォルダに、コマ内容マスクをレイヤーマスクとして付ける。
- フォルダ内には、フキダシ、効果線、グリースペンシル、ラスター、画像、テキストを通常レイヤーとして入れる。

### 7.2 ページ外・ページ直下

- ページ外やページ直下の要素は、コマフォルダの外に出す。
- コマ内容マスクを付けない。

### 7.3 移動後の出力

- 別コマ、別ページへ移動した要素は、現在所属しているコマフォルダへ出力する。
- ページ入れ替え後も、現在のページ番号とページ ID に対応したマスクを使う。

## 8. 実装手順

### Phase 1: マスク正本と命名

- コマ内容マスクの命名規則を追加する。
- ページ ID、表示ページ番号、コマ ID、解像度を保持する。
- コマ形状からアルファ画像を生成する処理を追加する。
- 旧来の破壊的クリップと併存できるよう、適用経路を分ける。

### Phase 2: カーブ・メッシュへの適用

- フキダシ、効果線、画像平面、テキスト平面へコマ内容マスクを掛ける。
- 線、塗り、フチ、多重線、グラデーション、不透明度の最終結果にマスクを掛ける。
- コマ内からコマ外へ見切れるフキダシが、線やフチを壊さずフェードすることを確認する。

### Phase 3: 所属変更への追従

- レイヤーリストで別コマへ移動した時のマスク差し替え。
- アウトライナーで別コマへ移動した時のマスク差し替え。
- ページ外へ移動した時のマスク解除。
- ページ入れ替え時のマスク再命名・孤立マスク削除。

### Phase 4: グリースペンシルとラスター

- ラスターの表示・レンダリングへコマ内容マスクを掛ける。
- グリースペンシルは、直接マスクできる場合は直接適用し、できない場合は表示・レンダリング用補助実体を追加する。
- 元のグリースペンシルは編集用の正本として残す。

### Phase 5: PSD 出力

- コマフォルダにコマ内容マスクをレイヤーマスクとして付ける。
- フキダシや効果線など個別レイヤーには同じマスクを付けない。
- ページ外・ページ直下要素の出力先を確認する。

### Phase 6: 旧方式の縮小

- コマ内フキダシ・効果線に対する破壊的クリップを停止する。
- 既存ファイルの旧クリップ設定は、修復操作または再同期時に新方式へ置き換える。
- 旧マスクや孤立補助オブジェクトを削除できる修復操作を用意する。

## 9. 検証項目

### 9.1 実機検証

- 輪郭ぼかしなしコマで、フキダシ・効果線がコマ境界で硬く消える。
- 輪郭ぼかしありコマで、フキダシ・効果線・ラスターが境界で滑らかにフェードする。
- フキダシの線、塗り、外側フチ、内側フチ、多重線が同じ境界で消える。
- コマ枠線はマスクされず、常に前面に残る。
- 用紙ガイド線とセーフライン外塗りはマスクされない。
- 別コマ、別ページ、ページ外へ移動してもマスクが正しく切り替わる。
- ページ入れ替え後、マスク名と適用先が現在のページ番号と一致する。

### 9.2 AI 目視

- コマ境界で線が急に細くならない。
- フキダシのフチがコマ外へはみ出さない。
- 輪郭ぼかしのあるコマ内で、フキダシや効果線が同じぼかしで消える。
- 別コマの輪郭ぼかしが、他コマの効果線やフキダシの色を変えない。

### 9.3 PSD 検証

- PSD のコマフォルダにマスクが付く。
- フォルダ内のフキダシ・効果線・グリースペンシル・ラスター個別レイヤーに重複マスクが付かない。
- ページ外・ページ直下要素はコマフォルダ外に出る。

## 10. リスクと対策

- グリースペンシルへ画像マスクを直接適用できない場合がある。対策として編集用正本と表示・レンダリング用補助実体を分ける。
- マテリアルを共有したままマスクを入れると、別コマの色や不透明度に影響する。対策としてコマ内容マスクを持つ表示用マテリアルは必要に応じて要素ごとに分ける。
- ページ番号だけでマスク名を作ると、ページ入れ替えで参照が壊れる。対策としてページ ID とコマ ID を内部参照の正本にする。
- 古い破壊的クリップと新マスクが二重に効くと表示が欠ける。対策として新方式適用時に旧クリップを解除する。

## 11. 計画書チェック結果

2026-05-25 の作成時点で、次の観点を照合済み。

- フキダシ、効果線、グリースペンシル、ラスター、画像、テキストを対象に含めている。
- 用紙、用紙ガイド線、セーフライン外塗り、コマ枠線、コマプレビューを対象外にしている。
- 別コマ、別ページ、ページ外、ページ入れ替えの全ケースを含めている。
- マスク名にページ ID、表示ページ番号、コマ ID、解像度を含める方針になっている。
- PSD はコマフォルダのレイヤーマスクを正とし、個別レイヤーマスクを付けない方針になっている。
- 作画形状を切断せず、マスクで透明化する方針になっている。
