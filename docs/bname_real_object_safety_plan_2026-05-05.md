# B-Name 作品要素の実体化計画 2026-05-05

## 背景

B-Name の表示要素の一部は、3D ビューポート上のオーバーレイ描画だけで見えている。アドオンを無効化した場合、オーバーレイ描画は停止するため、作品中にあるはずの枠線やテキストが消えたように見える。これはユーザーにとって「データが消えたのではないか」と感じる危険な状態である。

## 基本方針

- 枠線、テキスト、フキダシ、効果線、ラスター、グリースペンシル、画像など、作品の内容そのものは Blender の実オブジェクトとして残す。
- オーバーレイは、選択枠、ハンドル、カーソル、作成中プレビュー、IME 入力中プレビューなどの編集補助に限定する。
- アドオンが有効な時の操作性を崩さず、アドオンが無効な時も作品要素が見える状態を保つ。
- 書き出しの正本は従来どおり B-Name のデータと書き出し処理とする。実オブジェクトは、ビューポート表示と安全性を補強するための同期対象として扱う。

## 第一段階

1. テキストを、空オブジェクトではなく透明画像付きの平面として同期する。
   - 既存の縦書き、横書き、文字色、白フチ、部分スタイルを、書き出し用組版と同じ経路で画像化する。
   - テキストコレクションへの集約は維持する。
   - IME 入力中や選択範囲表示は、従来どおり編集補助オーバーレイで表示する。
2. コマ枠線を、コマごとのカーブオブジェクトとして同期する。
   - コマ背景用の平面とは別に、枠線だけの実体を持つ。
   - コマ形状変更やページ配置変更に追従する。
   - 第一段階では通常の枠線を実体化し、辺ごとの破線・点線などの完全再現は後続段階で扱う。
3. フキダシは既存のカーブオブジェクト同期を維持し、尻尾や詳細形状の不足は後続段階で補強する。

## 後続段階

- フキダシの尻尾を実オブジェクト側にも反映する。
- コマ枠線の辺ごとの線種、白フチ、角処理を実オブジェクト側でも完全再現する。
- 用紙のガイド線群も、必要に応じて実オブジェクト化する。ただし用紙要素は常に最前面に見えるという既存仕様を維持する。
- 画像レイヤーの空オブジェクト表示を、実画像平面へ戻すか検討する。

## 検証基準

- 新規作品でテキスト、フキダシ、効果線、コマ枠線を作成し、アウトライナー上に実体が残ること。
- B-Name の同期後、テキストが空オブジェクトではなく画像平面として存在すること。
- コマ枠線がコマ背景とは別のカーブとして存在すること。
- アドオンを登録解除しても、テキスト、フキダシ、効果線、コマ枠線が Blender データ内に残ること。
- 既存のレイヤー一覧、選択、詳細設定、ページ配置変更が破綻しないこと。
