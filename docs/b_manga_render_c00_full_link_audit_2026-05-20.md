現状は「完全連動」とは言えない。

## 発見事項

### 1. AOV入力が実マテリアルのソケット型に合っていない

`c00.blend` には `マテリアル方舟` というメッシュオブジェクトが存在し、`無効` コレクション内にある。このオブジェクトには同名のマテリアルが貼られており、そのマテリアル内にAOV出力用のシェーダーノードグループが入っている。

確認結果:

- オブジェクト: `マテリアル方舟`
- 種別: メッシュ
- 所属: `無効`
- マテリアル: `マテリアル方舟`
- AOV系入力: `落ち影切替` / `透過切替`
- 検出されたAOV関連マテリアル: `マテリアル方舟` / `マテリアル方舟.001`

このマテリアルを検証用メッシュへ貼り、キャラ・背景・効果の各コレクションへ配置して B-MANGA Render の全プリセットを実行した。

結果:

- `レイアウト`、`効果`、`背景パース`、`背景エフェクト`、`背景フォグ`、`背景雲`、`背景空`、旧互換のコマ出力系は画像を生成した。
- `キャラ`、`キャラパス`、`キャラ陰影`、`キャラアルファ`、`キャラ透過`、`キャラ_低速`、`キャラpen`、`背景`、`背景_低速`、`背景D`、`背景G`、`背景AO`、`背景線画用`、`背景ベース`、`背景植物`、`背景グラデ` などはAOV入力で失敗した。

失敗内容:

```text
NodeSocketInt.default_value expected an int type, not float
```

影響:

- AOV切替を含む主要なキャラ・背景プリセットが、レンダー前に止まる。
- `c00.blend` 側には必要なAOV用マテリアルがあるのに、B-MANGA Render が値を正しく入れられていない。

対応:

- B-MANGA Render のAOV入力カードは、対象ソケットの型を見て値を代入する必要がある。
- 整数/真偽値系の切替には整数値を入れる。
- 小数ソケットには小数値を入れる。
- 実行前監査で「対象AOV入力が何個見つかり、値を何個変更できたか」を表示する。

### 2. 「ページ」「全コマ統合」ノード参照は旧仕様

B-MANGA Render の「すべて」「ページ」プリセットは、合成側で「ページ」「全コマ統合」を切り替える前提になっている。

指定の `c00.blend` には「コマ」はあるが、「ページ」「全コマ統合」は見つからなかった。

これは最新の B-MANGA では `c00.blend` 側の不足ではなく、B-MANGA Render 側に旧仕様が残っている状態と判断する。現行 B-MANGA では、コマ用blendファイル側でページ全体や全コマ統合を完結させる前提は不要。

影響:

- 「ページ」プリセットはカード上は完走するが、ページ全体の合成切り替えが実体に当たらない。
- 「すべて」プリセットも同じく、ページ側の仕上げ工程が空振りする。

対応:

- B-MANGA Render 側で「ページ」「全コマ統合」参照を削除または現行のコマ用blendファイル工程へ置き換える。
- `c00.blend` 側に「ページ」「全コマ統合」相当の合成ノードを復元する必要はない。
- どうしても旧ファイル互換を残す場合だけ、互換プリセットとして分離する。

### 4. Blender 5.1 のファイル出力ノード API に未対応

指定の `c00.blend` のファイル出力ノードは、Blender 5.1 の形式で保存先とファイル名を持っている。

現在の B-MANGA Render には、旧形式の保存先・スロット名だけを触る処理が残っている。そのため、カードの「出力フォルダ」「出力画像名」指定が、`c00.blend` のファイル出力ノードへ正しく届かない可能性が高い。

影響:

- 検証用フォルダへ出したい場面でも、一部の出力が `c00\passes` へ出た。
- 魚眼系出力も既定の `passes` に固定されやすい。

対応:

- Blender 5.1 のファイル出力ノード形式に対応する。
- 旧形式と新形式の両方を扱う。
- 通常パス出力と魚眼出力の保存先を、B-MANGA のコマ用blendファイルの出力先へ統一する。

### 8. 出力範囲が B-MANGA の現在コマに完全連動していない

追加で、指定 `c00.blend` をコマ用blendファイルとしてコピーし、横長でも縦長でもない非対称なコマを作って、魚眼モード・縮小モード・B-MANGA Render の魚眼系プリセット・ページ一覧へ戻る・再オープンを実機で往復した。

証跡:

- `.codex/ai_audit/20260520_output_range/bmanga_render_output_range_roundtrip.json`

結果:

- `c00.blend` 由来の古いカメラ下絵に、`コマ枠.png`、`コマ01`、`コマ02` など「コマ」を含む画像スロットが多数残っている。
- B-MANGA が生成した現在コマ用の下絵も追加されるが、古い「コマ」下絵より後ろに追加される。
- 現在のレンダー範囲計算は、最初に見つかった「コマ」を含む下絵を使うため、B-MANGA が生成した現在コマ用の下絵ではなく、`c00.blend` 側の古い下絵でレンダー範囲が決まる。
- そのため、B-MANGA Render の出力範囲は現時点では最新 B-MANGA の現在コマに完全連動していない。

実機往復で確認した代表値:

```text
コマを開いた直後       6071x8598 / 魚眼ON / 縮小OFF / レンダー範囲 0.105,0.895,0.221093,0.778907
魚眼モードON           8598x8598 / 魚眼ON / 縮小OFF / レンダー範囲 0.105,0.895,0.105,0.895
魚眼ON+縮小50%         4299x4299 / 魚眼ON / 縮小ON  / レンダー範囲 0.105,0.895,0.105,0.895
B-MANGA Render実行時    4299x4299 / 魚眼ON / 縮小ON  / キャラpen方向・背景pen方向は同じ範囲を使用
再オープン直後         6071x8598 / 魚眼ON / 縮小ON  / レンダー範囲 0.105,0.895,0.221093,0.778907
```

影響:

- 魚眼モード中でも、コマ用blendファイルをページ一覧へ戻ってから開き直すと、魚眼・縮小フラグは残るのに、解像度が正方形/縮小後ではなく通常原稿サイズへ戻る。
- これは「魚眼の形状がページの形状にフィットした状態になる」症状と一致する。
- B-MANGA Render の魚眼系プリセットは、開き直し直後に実行すると、B-MANGA が意図した魚眼・縮小レイアウトではなく、再オープン時に戻された出力範囲で動く可能性が高い。
- 古い `c00.blend` 下絵が残っている限り、数値上のレンダー範囲が合って見えても、B-MANGA の現在コマではなく旧下絵に従っている可能性がある。

対応:

- B-MANGA 側のレンダー範囲計算は、B-MANGA が生成した現在コマ用の下絵を最優先する。
- `c00.blend` 由来の未管理下絵は、レンダー範囲計算から除外する。
- コマ用blendファイルを開いた直後、現在の魚眼モード・縮小モード・縮小率を必ず再適用し、最後にレンダー範囲を再計算する。
- B-MANGA Render のプリセット実行直前にも、B-MANGA の現在設定から出力解像度・魚眼設定・縮小設定・レンダー範囲を再同期する。
- `c00.blend` 側では、テンプレートとして不要な古いカメラ下絵を削除するか、少なくとも B-MANGA の現在コマ下絵と混同されない名前へ整理する。

## 完全連動テスト計画

この計画での「完全連動」は、単にプリセットがエラーなく終わることではない。最新 B-MANGA に必要な B-MANGA Render の全プリセットについて、次の流れが実データで確認できた状態を指す。

1. `c00.blend` 内の実AOV用マテリアルを貼ったメッシュをレンダー対象にする。
2. B-MANGA Render のプリセットで、必要なAOV切替とビューレイヤー切替を行う。
3. `出力_○○` 系ノードグループから、期待される全パス画像を実ファイルとして出す。

## 追加必須条件: 全画像実体検証

ユーザー指摘により、本計画の完了条件を強化する。Workbench の低負荷レンダーや、レンダー呼び出しの差し替えだけでは「レンダリング結果が正しい」ことの完了条件にしない。

### 禁止事項

- Workbench 固定の目視シートを、魚眼・陰影・透過・合成・Pencil+4 の正否判定として扱わない。
- 代表プリセットだけで「全プリセット確認済み」と言わない。
- パス画像の存在確認だけで、合成画像の確認を省略しない。
- 合成画像だけを見て、元パス画像の欠落や中身違いを省略しない。
- レンダー呼び出しをダミーPNG生成に差し替えたテストを、実レンダー検証として扱わない。
- 魚眼モードOFFで通った結果を、魚眼モードONの結果として扱わない。
- 旧 `c00.blend` 由来の下絵や画像ノードに引っ張られた結果を、B-MANGA 現在コマ連動として扱わない。

### テスト用blendファイルの条件

開発セッションでは、指定 `c00.blend` を直接保存せず、検証用コピーとして `b_manga_render_full_fixture.blend` を作成する。この fixture は次を満たす。

- 全プリセットが参照するビューレイヤーを持つ。
- 全プリセットが参照するコレクションを持つ。
- `キャラ`、`キャラアルファ`、`背景`、`背景MH`、`効果`、`効果アルファ`、`レイアウト`、`アタリ`、`空`、`植物`、`エフェクト`、`フォグ`、`雲`、`グラデ_白`、`グラデ_黒`、`Zパース`、`Xパース`、`Yパース`、`コマ枠` それぞれに、目視で区別できる実メッシュを置く。
- `マテリアル方舟` 相当のAOV入力を持つマテリアルを、キャラ系・背景系の検証メッシュへ貼る。

### プリセット別マニフェスト

テストは実行前に、組み込み全プリセットから次のマニフェストを作る。

- プリセット名
- 実行カード列
- 使用ビューレイヤー
- 使用コレクション
- AOV入力名
- 有効化されるファイル出力ノード

- 画像ノード再読み込み後に参照される入力画像一覧

- 魚眼ON時に追加で期待される魚眼画像・方向画像・合成画像一覧

このマニフェストに載っている画像は、すべて実ファイルとして生成・開封・AI目視対象にする。マニフェストに載っていない画像が出た場合も、出力漏れまたは命名ずれの可能性として報告する。

### 実レンダー検証の条件

全プリセットについて、次を実行する。

- 通常モードで実行する。
- 魚眼系カードを含むプリセットは、魚眼モードONでも実行する。
- 縮小モードの影響を受ける出力は、縮小モードONでも実行する。
- 魚眼系カードを含むプリセットは、魚眼ON + 縮小ONでも実行する。
- 実行時のレンダーエンジンは、プリセット指定を使う。検証時間短縮のために解像度・サンプル数は下げてもよいが、Workbench固定へ置き換えてはいけない。
- 画像出力は検証用フォルダへ集約し、元の `c00.blend` 配下へ漏れないことを確認する。
- 各画像について、ファイル存在、開封可否、解像度、RGBA、アルファ分布、空画像でないこと、プリセット名・出力名との対応を記録する。

### パス画像からコンポジット済み画像までの流れ

完了には、次の流れを全対象で確認する必要がある。

1. パス出力プリセットで、マニフェスト上の全パス画像が出る。
2. 出たパス画像を、合成側画像ノードが相対パスで読み込む。
3. 画像ノード再読み込み後、必須画像ノードの未読込が0になる。
4. 統合系プリセットを実行し、マニフェスト上の全コンポジット済み画像が出る。
5. コンポジット済み画像に、該当パス画像の内容が反映されている。
6. 入力パスが欠けた場合は、合成結果の異常または入力欠けとして検出できる。

### AI目視の条件

AI目視は、1枚の巨大シートだけで済ませない。次の分割シートと判定JSONを作る。

- パス画像シート: プリセット別に、全パス画像を出力名つきで並べる。
- コンポジット画像シート: 統合・線画・AOV・効果など、全コンポジット済み画像を並べる。
- 魚眼シート: 通常、魚眼ON、縮小ON、魚眼ON+縮小ON、再オープン後を横並びで比較する。
- 出力範囲シート: ページ一覧の現在コマ、カメラ下絵、レンダー結果、レンダー範囲を比較する。
- 失敗/要確認シート: 欠落、空画像、名前不一致、想定外の真っ黒/真っ白、旧下絵参照をまとめる。

各画像の判定は `OK` / `NG` / `未完了` のいずれかにする。`要人間確認` は最終合格に含めない。AI目視で判定不能な画像が残る場合、その検証は未完了とする。

### c00 / B-MANGA 出力範囲完全連動の条件

出力範囲は、数値確認だけでなく、実画像とAI目視で確認する。

- 通常モード、縮小ON、魚眼ON、魚眼ON+縮小ONを個別に確認する。
- B-MANGA が生成した現在コマ下絵と、旧 `c00.blend` 下絵が同時に存在する状態で確認する。
- レンダー範囲計算に使った下絵名を記録する。
- ページ一覧へ戻る前、戻った後、再オープン直後、再オープン後のB-MANGA Render実行後を比較する。
- 魚眼ON + 縮小ONで再オープンしても、解像度、魚眼投影、縮小率、レンダー範囲、見た目が戻らないことを確認する。
- 魚眼の円形/投影結果が、縦長ページや旧下絵の形へ引き伸ばされて見えた場合は失敗とする。

### 最終合格条件

最終合格は、次がすべて満たされた場合だけとする。

- 全プリセットの通常モード実レンダーが完了。
- 魚眼系全プリセットの魚眼ON実レンダーが完了。
- 縮小対象の全ケースで縮小ON実レンダーが完了。
- 全パス画像が存在し、AI目視で出力名どおり。
- 全コンポジット済み画像が存在し、AI目視で入力パス反映済み。
- 画像ノード再読み込み後、生成対象画像の未読込が0。
- 出力先漏れが0。
- B-MANGA現在コマ下絵以外を出力範囲計算に使うケースが0。
- 魚眼/縮小/再オープン後の見た目劣化が0。
- 判定不能または未完了画像が0。
