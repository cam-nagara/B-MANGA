# B-MANGA Line（線画生成アドオン） 設計意図

> 本書は B-MANGA Line における「意図の正本」。ユーザーの指示・希望を粒度を丸めずに記録する。
> 運用ルールは `docs/仕様定点観測_設計意図.md` に従う。B-MANGA 本体の意図は `docs/B-MANGA_設計意図.md` を参照。
> 記載の区別: **確定** = ユーザー明言・最終判断 / **暫定** = 実装や文脈からの読み取りで正解未確認 / **要確認** = 矛盾・不足あり。

## 1. 用途と大前提

- B-MANGA のコマファイルでは様々なオブジェクトをリンク読み込みしてレンダリングする。そのため**リンク元ファイルで B-MANGA Line の線画設定を行い、それをコマファイルへ読み込む**のが基本の使い方。（確定）
- カメラ基準は「カメラビューのカメラ」とする。リンク読み込みしたオブジェクトごとに基準カメラを設定させる仕様は不可。（確定 2026-06-28）
- **線幅は印刷上の mm に一致させるのが必須。常に。**「線幅(mm)を Blender 内部長へ 1m=1000mm で変換する」ような、印刷 mm と一致しない仕様は不可。コマ枠線・フキダシ・効果線の線幅と同じく、解像度に従い mm 指定どおりに出る。（確定 2026-06-29、ユーザー最重要事項）
- ただし「印刷サイズ上で全オブジェクトの線幅を均一にする」自動補正は不要（勝手にオブジェクトごとのモディファイア幅を別々の値にしない）。カメラから離れるほど細くなるのは自然な挙動として許容し、代わりに**「線幅基準距離」**（この距離にあるオブジェクトが指定 mm どおりの線幅になる基準。ワールド内距離 m 単位で数値入力、初期値 2m）を設ける。（確定 2026-07-01）
- 線幅の初期値は 0.3mm（印刷サイズ基準）。（確定 2026-07-01）
- 大規模アセット（例: `Japanese_Streetscape_Tokyo_0004.blend`）の全オブジェクトへ一括適用できる軽さが実用ライン。全メッシュ選択状態で、パネルの**すべてのチェックボックスのオン・オフそれぞれが実用時間内（当初の依頼では20分以内、以後の文脈では数分〜即時を期待）に完了**しなければならない。30分以上かかる・止まる・「ある組み合わせの時だけ極端に重い」は不可（もぐらたたき禁止）。（確定 2026-06-29〜07-02）
- 個別の修正が別の組み合わせを重くする「もぐらたたき」状態は不可。オン・オフの全組み合わせで無駄なく最速であること。（確定 2026-07-02）

## 2. ライン種別と共通仕様

対象は アウトライン / 内部線 / 交差線 の3種。

- 3種それぞれにオン・オフのチェックボックスを持つ（アウトラインにも実装する）。（確定 2026-07-01）
- 3種それぞれで線の色を設定できる。（確定 2026-07-02）
- 3種それぞれでソリッドモディファイアのオフセットをパネルの各ラインのセクション内で調節できる。複数選択時は一括適用。（確定 2026-07-01）
- 「ラインのみを表示」「ラインを表示」「ラインを非表示」は、アウトラインだけでなく内部線・交差線もまとめて対象にする。（確定 2026-06-29）
- 複数オブジェクトを選択した状態でライン設定を変えたら、選択中のすべてのオブジェクトに設定が上書き反映される。（確定 2026-06-29）
- 設定変更（線幅・チェックボックス・スライダー）は「選択中かつ生成済みのライン」だけを対象にして軽くする。生成後に線幅を変えるだけで重くなるのは不可。（確定 2026-07-01）
- **初期値ではすべてのチェックボックスをオフにする。**（確定 2026-06-30 14:09。これより前の「内部線と交差線の初期値はオン」(06-29) を上書きする最終判断）。ただしこの指示より後に追加・変更されたものは対象外: 「作成範囲を制限」は初期値オン（確定 2026-07-01 14:31）、アウトラインのオン・オフチェックボックスは 2026-07-01 に追加依頼され初期値の明示なし（現行実装はオン = 暫定観測）
- 「カメラからの距離で非表示」（m 単位の数値付き）をアウトライン・内部線・交差線それぞれに持つ。目的はカメラから指定距離以上離れたラインを非表示にして動作・レンダリングを軽くすること。（確定 2026-06-28）
- 板ポリ（平面ポリゴン）は内部線・交差線の対象から除外できる（「おすすめ通りに実装」指示）。初期値は 2026-06-30 13:55 に「除外オン」と指示されたが、14分後の「初期値ではすべてのチェックボックスをオフ」（同日 14:09）が最終判断のため、除外チェックの初期値もオフとする。（確定 — 最新判断優先。ただし「板ポリ除外だけは別枠でオンのつもりだった」可能性は要確認）
- 全ライン設定を一つのプリセットにまとめ、プリセット管理機能を持ち、選択中の全オブジェクトに適用できる。ラインの非表示と削除もそれぞれできる。（確定 2026-06-27）
- パネルの項目配置は影響範囲で分類する: アウトラインのみに影響 → アウトライン設定 / 内部線のみ → 内部線設定 / 交差線のみ → 交差線設定 / 全体に影響 → カメラ設定。どれがどれに影響するか判別できること。（確定 2026-06-30）
- ラインを設定できないオブジェクトがあってはならない（Blender 追加のプリミティブ以外の、アセット由来オブジェクトにも設定できる）。（確定 2026-06-28）
- blend ファイル保存時に、オブジェクトの Smooth by Angle（自動スムーズ）設定を壊してはいけない（保存の瞬間にモディファイアが消滅して赤アイコン化し、スムーズシェードにされる事象の再発禁止）。（確定 2026-07-02）
- 透明・半透明メッシュにラインを設定した場合、裏面が見えてもライン色で塗りつぶされず、透明の面のまま維持される。（確定 2026-06-28）

## 3. 線幅の均一化（2種）と名称

- 機能は2種類あり、名称と配置を以下に固定する（確定 2026-06-30）:
  - 「線幅の均一化（オブジェクト単位）」（旧称: カメラ距離で線幅を補正）— 上に配置
  - 「線幅の均一化（頂点単位）」（旧称: 線幅の均一化）— その下に配置
- 「線幅の均一化」はオン・オフを切り替えて使う機能であり、常時適用へ勝手に変えてはいけない。（確定 2026-06-29。dskjal.com の「線幅の均一化」等を参考に実装）
- オンの時は、mm で設定したとおりの線幅が解像度に従って出ることを、コマ枠線・フキダシ・効果線と同水準で守る。（確定 2026-06-28）
- 「線幅の均一化（頂点単位）」の初期値はオフ。（確定 2026-06-30。複数回念押しあり）
- オフに切り替える操作が固まる・何時間もかかるのは不可。（確定 2026-06-29〜30）

## 4. アウトライン

- 面としてのソリッド化した線（ソリッドモディファイア方式）で生成する。（暫定 — 実装方式として定着、ユーザーは方式自体は否定していない）
- 開いた立体・アセット由来オブジェクトでもアウトラインが生成されること。「Cube だけ正常」のような状態は不可。（確定 2026-07-01）
- アウトラインにも中間頂点の線幅調整を適用する。対象は内部線と同様、検出角度で検出された角と角の間のライン。（確定 2026-06-29）
- 「ラインのみ表示」でもソリッドモディファイアのオフセットは 1.0 を維持し、表示切替でオフセットを変えない。（確定 2026-07-01）

## 5. 内部線

- 検出角度で検出された角と角の間を線にする。検出角度の初期値は 60 度。（確定 2026-07-01）
- 内部線をオンにしてもテクスチャが内部線メッシュで覆い隠されてはいけない。（確定 2026-06-28）
- 「指定済みの辺だけ線にする」オプションを持つ（マークした辺のみ）。初期値はオフ。（確定 2026-06-29）
- 円柱のフチのような（鋭角の無い）内部線にも中間頂点の線幅調整が効くこと。鋭角が無い場合は「カメラビュー上の内部線の端から端まで」を一本の範囲として扱ってよい。（確定 2026-07-02）

### 5.1 中間頂点（角と角の間の線幅変化）

- 角と角をつなぐライン上に中間頂点を置き、そこへ向けて線幅を変化させる。中間頂点の細さの初期値はゼロ。（確定 2026-06-27）
- 中間頂点の線幅がゼロでも、中央の前後の区間が「なだらかに細く見える」のが正解。一定の極細線幅が長く伸びるのは失敗。（確定 2026-06-27）
- 「乱れ」スライダー: 中間頂点位置をランダムに決める。数値は「角から角までの距離のうち、真ん中から前後何%の範囲でランダムにするか」。（確定 2026-06-27）
- 角と角の間のラインごとに、中間頂点位置は別々のランダム値になる。（確定 2026-06-27）
- 中間頂点へ向かう線幅変化の仕方をグラフで設定できる（急に下がってなだらかに、なだらかに変化して急に上がる、等）。全内部線一括設定。数値項目の羅列ではなく、ちゃんとしたグラフ UI であること。ポイント追加やカーブ調節が確実に反映されること。（確定 2026-06-27〜07-02。「全く機能していません」と複数回指摘あり — 機能することが仕様）
- 中間頂点の線幅調整は、アウトライン・内部線・交差線それぞれに**別々に**設定できる。（確定 2026-06-29）
- 各ライン種の中間頂点の線幅調整それぞれに検出角度スライダーを持つ。（確定 2026-07-02）

## 6. 交差線

- **生成方式は「ライン素材（高速）」方式のみが正。** 交差しているオブジェクトを検出する処理は不要。ラインを持つオブジェクト同士が交差すれば、ライン用ソリッドモディファイアの厚み分の交差部分が自然に生じるので、その部分がラインの色で表示されればよい。（確定 2026-07-02。「実際の交差部分に線を作る方式へ戻す」「方式を選択制にする」は依頼しておらず不可 — 高速方式で交差線をしっかり出し、中間頂点にも対応させるのが指示）
- 交差線の交差対象は、交差しているオブジェクトすべてを自動で対象にする。対象の設定欄は不要。すべてのオブジェクトにラインが設定されている前提で問題ない設計にする。（確定 2026-06-29）
- 交差線をオンにした時、交差対象ラインの太さ部分が元メッシュとソリッドメッシュで二重線になる場合は、その内側を線の色で塗りつぶす。（確定 2026-06-27）
- 出るべき交差部分に交差線が出ること。交差していない部分に線が生じてはいけない。円錐とキューブの交差線が二本になるのは不可。（確定 2026-07-01〜02）
- 交差線にも、線幅・中間頂点の線幅調整・中間頂点への変化グラフが機能する。（確定 2026-07-02）
- 交差専用の大量メッシュを生成して大規模シーンで重くなる方式は不可。（確定 2026-07-02）

## 7. 作成範囲の制限（パフォーマンス機能）

- 内部線・交差線の「作成範囲を制限」: 表示範囲ではなく**作成**範囲を、カメラから一定距離以内に限定する。距離の初期値は 10m。チェックの初期値はオン。（確定 2026-06-29 / 2026-07-01）
- すべてのオブジェクトを選択した状態でも、カメラに写っている範囲のオブジェクトのみを作成対象にできる。魚眼（コマファイルの魚眼モード）にも対応する。（確定 2026-07-01）
- 「レンダリング範囲内のメッシュオブジェクトを全選択」するボタンを持つ。（確定 2026-07-01）

## 8. 表示系

- 「ラインのみを表示」ボタン: ライン用マテリアル以外をすべてシンプルな光沢100%・白のノードに差し替え、背景も真っ白にする**一時マテリアル差し替え方式**。Cycles レンダーモードで確認できること。（確定 2026-07-01）
- AOV を使った「ラインのみ表示」は重いため実装しない。（確定 2026-07-01。06-29 の「AOV を利用できないか」という打診を自身の最終判断で撤回）
- 「ラインのみ表示」→「通常表示に戻す」でソリッドモディファイアのオフセット等の設定値を変えない（オフセットは 1.0 のまま）。（確定 2026-07-01）
- AOV 出力自体は常にオンとし、ライン用マテリアルにデフォルトで AOV を入れる。コンポジット出力セクションの UI は不要。AOV が欠けたマテリアルは自動補修する。（確定 2026-06-28〜29）
- パネルにすべての項目が表示されること（項目欠けの再発禁止）。（確定 2026-06-29）
- アドオンが有効化できない（プリファレンスでオンにするとエラー）状態の再発禁止。（確定 2026-06-28〜29）

## 9. リンク運用・一括操作

- コマファイルで、リンク読み込みしたすべてのオブジェクトのラインに対して、一括で設定を上書き（もしくは補正）できるようにしたい。（確定 2026-06-28）
- 2026-07-03 実装状況: 「リンク素材のラインを補正」「リンク素材へ選択設定を上書き」オペレータあり。**ライブラリオーバーライドへの上書きは保存・開き直し後も保持される**（v0.3.73 で修正）。オーバーライドを作らない直接リンクへの上書きは Blender の仕様上セッション中のみ有効で保存されない — このUX（自動オーバーライド化するか等）は要確認（AGENT_INBOX に選択肢を記録済み）。
