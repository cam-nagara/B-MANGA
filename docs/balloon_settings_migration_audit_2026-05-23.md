# フキダシ設定移行監査 (2026-05-23)

## 結論

2026-05-23 時点の軽量カーブ方式では、フキダシの設定項目はまだ完全移植ではない。
今回の監査で、塗りが実表示で黒または未塗りに見える不具合を確認し、塗り面と輪郭線を表示補助側で別々に作るよう修正した。

完全移植として残っている項目は、線種、塗り輪郭ぼかし、塗りぼかしのディザ化、外側白フチ、内側白フチ、合成モード「比較 (明)」である。

## 設計意図

- フキダシの正本は、Blender標準機能で制御点編集できるカーブ実体にする。
- B-MANGAは作成、詳細設定、再生成、整合性確認を担当し、確定済みフキダシ本体はB-MANGAの画面描画だけに依存しない。
- フキダシ用のジオメトリノードは、塗り、線、不透明度、グラデーションなどの軽い補助に限定する。
- 雲、もやもや、トゲなどの外形そのものは、常時ノード生成せず、明示再生成時にカーブ制御点として作る。
- 使わない重い設定は、ジオメトリノードモディファイアの設定欄に常時出さない。

## 監査範囲

- 詳細設定ダイアログのフキダシ項目
- レイヤー一覧からの表示/非表示
- カーブ実体として残る表示状態
- コマ内マスクとページ直下フキダシ
- 書き出し用ラスタ描画との設定差分
- 実機レンダー画像による白塗り/黒輪郭のAI目視確認

## 実機確認

- 白塗り・黒輪郭の楕円フキダシを作成し、実機レンダーで中央が白、輪郭が黒になることをピクセル検査した。
- 生成画像 `D:/Develop/Blender/B-MANGA/.codex/visual/balloon_curve_render/balloon_curve_white_fill_black_line.png` をAI目視し、白塗りと黒輪郭が分離していることを確認した。
- しっぽ追加、しっぽ設定、塗り不透明度、塗りグラデーション、塗り輪郭ぼかし設定の保存、外側白フチ/内側白フチ設定の保存、選択カーブ登録、手編集維持は既存実機テストで確認した。

## 今回追加した回帰対策

- フキダシの塗りと輪郭線を、表示補助側で別々に作成して別素材を割り当てる。
- 元カーブ本体の塗り/ベベル表示は切り、下敷きの黒塗りが残らないようにする。
- 線素材と塗り素材は発光＋透明合成にし、ライトの有無で白塗りが暗くならないようにする。
- 実機レンダーで、白塗りが白、輪郭線が黒として出ることをピクセル検査する。

## 残作業

完全移植と呼ぶには、線種、塗り輪郭ぼかし、外側白フチ、内側白フチ、比較(明)を軽量カーブ方式へ追加実装する必要がある。
これらは単にソケットをつなぐだけではなく、カーブ正本を壊さず、B-MANGA無効時にも表示が残り、移動中に重くならない方式で実装する。
