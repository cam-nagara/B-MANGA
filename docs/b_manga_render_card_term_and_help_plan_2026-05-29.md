# B-MANGA Render コマンド用語改善 + 選択コマンド設定の処理説明表示 計画書

作成日: 2026-05-29
種別: 計画書（調査セッションで作成 / 実装は開発セッションへ引き継ぎ）
対象アドオン: `addons/b_manga_render`

## 1. 目的

B-MANGA Render のコマンドについて、

1. 実態と合っていない用語を、動作が分かる名前へ変更する。
2. 「選択コマンド設定」セクションに、そのコマンドが「何を・どう対象にするか」を一文で表示する。

これにより、コマンドの表示だけでは処理内容（特に対象ノードの選び方）が分からない問題を解消する。

## 2. 背景・課題

- 「キャラ陰影」プリセットで、`出力_キャラ` グループ内の3つの陰影ファイル出力ノード（実ノード名 `出力` / `出力.001` / `出力.002`、ラベル「キャラベタ陰影」「キャラグレー陰影」「キャラホワイト陰影」）の出し分けは、実際には **レンダー系コマンド（現「フレーム出力レンダー」）が「出力」の部分一致** で行っている。
- しかしユーザーは「ノードミュート：出力_キャラ」コマンドがそれを行っていると誤解した。実機検証では、ノードミュートコマンド単体実行では3陰影ノードのミュートは変化しない（変わるのは `出力_キャラ` グループ入れ物ノード1個のみ）ことを確認済み。
- 原因は用語とUIにある:
  - コマンド種類名「フレーム出力レンダー」が、何をするコマンドか伝えていない。
  - 設定項目「フレーム名」が、実際は「ノードの名前・ラベルに部分一致するキーワード」なのに、その意味が伝わらない。
  - 同じ「ミュート」でも、種類によって **完全一致（ノードミュート）／部分一致（ファイル出力切替・レンダー系）** とロジックが異なるのに、UI上で区別できない。

### 確認済みの実装挙動（行番号非依存）

- `command_runner.py` の `_set_node_mute(scene, node_name, mute)`: 全ノードを走査し、**`node.name` または `node.label` が `node_name` と完全一致**するノードだけ `mute` を設定する。部分一致しない。
- `command_runner.py` の `_set_output_group(group_name, label, mute)`: `bpy.data.node_groups` の該当グループ内を再帰走査し、`type == "OUTPUT_FILE"` かつ `_node_matches_label(node, label)`（**部分一致**: name/label/親name/親label/入力ソケット名/ファイル出力項目の name・path・file_name のいずれかに `label` を含む）のノードの `mute` を設定する。
- `command_runner.py` の `_render_layer(scene, group_name, label, engine, sample_count)`: まず `_set_output_group(group_name, "", True)`（グループ内の全ファイル出力をミュート）→ `_set_output_group(group_name, label, False)`（`label` 部分一致のものだけミュート解除）→ `_render(...)` の順に実行する。

## 3. 決定事項（ユーザー合意済みの名称）

| 対象 | 現在 | 変更後 |
| --- | --- | --- |
| コマンド種類 `RENDER_LAYER` の表示名 | フレーム出力レンダー | **レンダー：ワード検出** |
| プロパティ `label_contains` の表示名 | フレーム名 | **検出ワード** |

補足: `label_contains` は `RENDER_LAYER` 以外に `SET_OUTPUT_GROUP`（ファイル出力切替）と `FISHEYE_*` 系コマンドでも共通利用される。表示名「検出ワード」はこれら全種類に反映される（部分一致キーワードという意味で統一されるため、望ましい）。

## 4. 変更詳細（行番号非依存）

### 4.1 種類名の変更
- ファイル: `addons/b_manga_render/core.py`
- 対象: `COMMAND_TYPE_ITEMS` 内の識別子 `"RENDER_LAYER"` のラベル文字列 `"フレーム出力レンダー"` を `"レンダー：ワード検出"` に変更する。
- 識別子 `RENDER_LAYER` そのものは変更しない（保存済みプリセット・preset_library との互換のため）。

### 4.2 項目名の変更
- ファイル: `addons/b_manga_render/core.py`
- 対象: `StringProperty(name="フレーム名", ...)` で定義されている `label_contains` の `name` を `"検出ワード"` に変更する。
- プロパティ識別子 `label_contains` は変更しない。

### 4.3 選択コマンド設定への処理説明表示の追加
- ファイル: `addons/b_manga_render/command_ui.py`
- 方針:
  - 新規ヘルパー関数 `command_help(command) -> str` を追加し、コマンド種類と現在の設定値から「このコマンドが何を・どう対象にするか」を表す一文を返す。
  - `draw_command(layout, command, context)` の末尾（各 `layout.prop(...)` の後）に、`layout.label(text=command_help(command), icon="INFO")` で説明文を表示する。長文は `layout` の複数行ラベルに分割するか、要点を1〜2行に収める。
- 安全性（`docs/B-MANGA_設計意図.md` の「描画中の不正なデータ書き込みでパネル描画が中断してはいけない」に準拠）:
  - `command_help` は値の取得を全て `getattr(command, ..., 既定値)` で行い、例外を投げない。プロパティ書き込みは一切しない。
  - 想定外の種類でも空文字を返すフォールバックを持つ。

### 4.4 マニュアル更新
- ファイル: `docs/B-MANGA Render_マニュアル.md`
- 「コマンドの種類」表の「フレーム出力レンダー」行を「レンダー：ワード検出」に更新し、説明を「グループ内で検出ワードに部分一致する出力ノードだけ有効化し、他を無効化してレンダーします」に修正。
- 「種類ごとの設定項目」表の「ノード名 / ミュート」「対象 / フレーム名 / ミュート」を、新名称「検出ワード」に更新。
- 「ノードミュート＝完全一致」「ファイル出力切替・レンダー系＝検出ワードの部分一致」という判定基準の違いを明記する一節を追加する。

## 5. 各コマンド種類の説明文テンプレート（4.3 で表示する一文）

`{}` は現在の設定値を差し込む。完全一致／部分一致の別を必ず明示する。

| 種類 | 説明文（案） |
| --- | --- |
| STATE_BEGIN | 現在のレンダー・出力状態を退避し、プリセット用に初期化します。 |
| STATE_END | 退避した状態を元に戻します。 |
| SET_VIEW_LAYER | ビューレイヤー「{view_layer_name}」を{有効/無効}にします。 |
| SET_COLLECTION_EXCLUDE | コレクション「{collection_name}」を{表示/除外}します。 |
| SET_NODE_MUTE | 名前またはラベルが「{node_name}」と**完全一致**するノードを{ミュート/ミュート解除}します（部分一致しません）。 |
| SET_OUTPUT_GROUP | グループ「{node_group_name}」内で、名前・ラベルに「{label_contains}」を**含む**ファイル出力ノードを{ミュート/ミュート解除}します（部分一致）。 |
| RENDER_LAYER（レンダー：ワード検出） | グループ「{node_group_name}」内で、名前・ラベルに「{label_contains}」を**含む**出力ノードだけ有効化し、他のファイル出力を無効化してからレンダーします（部分一致 / {engine}・{sample_count}サンプル）。 |
| SET_AOV_INPUT | グループ「{node_group_name}」の入力「{input_name}」を {float_value} にします。 |
| SET_OUTPUT_NAME | 出力画像名を「{text_value}」にします。 |
| SET_OUTPUT_FOLDER | 出力フォルダを「{folder_path}」にします。 |
| RELOAD_IMAGES | すべての画像ノードを再読み込みします。 |
| RENDER | 現在の設定でレンダーします（{engine}・{sample_count}サンプル）。 |
| FISHEYE_RENDER_IMAGE_OR_LAYER 他 FISHEYE_* | 魚眼モード時は eeVR、通常モード時はワード検出レンダーに分岐します（対象グループ「{node_group_name}」/ 検出ワード「{label_contains}」）。 |
| EEVR_* | eeVR の対応処理を呼び出します（魚眼モード時のみ）。 |
| OPERATOR | オペレーター「{operator_idname}」を実行します。 |

注: 「{有効/無効}」「{ミュート/ミュート解除}」は対応する真偽値（`view_layer_enabled`、`exclude_collection`、`mute`）から決定する。

## 6. 影響範囲・注意点

- `command_ui.py` の `auto_command_name` / `display_name` は種類ラベルを `command_type_label` 経由で取得するため、4.1 の種類名変更でコマンド一覧の自動生成名も自動的に「レンダー：ワード検出: …」へ追従する（追加対応不要）。
- 処理ロジック（`command_runner.py`）は**変更しない**。今回は表示名と説明表示のみ。既存プリセットの実行結果は変わらない。
- `label_contains` の表示名変更は `SET_OUTPUT_GROUP` / `FISHEYE_*` にも及ぶ。マニュアルの該当箇所も合わせて更新する。
- 説明文に差し込む値は未設定（空文字）の場合もあるため、空のときに不自然にならない文面・フォールバックにする。
- バージョンは PATCH を +0.0.1 のみ（B-MANGA のバージョン規約に従う）。

## 7. 検証項目（開発セッションで実機確認）

- コマンド種類セレクタに「レンダー：ワード検出」が表示され、該当コマンドの設定欄に項目「検出ワード」が出る。
- 「ファイル出力切替」コマンドの設定欄も項目名が「検出ワード」になっている。
- 選択コマンド設定に、種類ごとの説明文が表示される。特に:
  - ノードミュート → 「完全一致」と明記されている。
  - レンダー：ワード検出 / ファイル出力切替 → 「含む（部分一致）」と明記されている。
- 設定値が空・未設定でも、パネル描画が中断しない（説明生成で例外が出ない）。
- 既存「キャラ陰影」等のプリセットを実行し、出力結果（3陰影ノードの出し分け）が従来どおりであることを確認。

## 8. 引き継ぎメモ

- 本計画は表示・用語・ヘルプ表示のみの変更で、コマンド実行ロジックには手を入れない。
- 実装後、CHANGELOG に用語変更（フレーム出力レンダー→レンダー：ワード検出 / フレーム名→検出ワード）と説明表示追加を記載する。
