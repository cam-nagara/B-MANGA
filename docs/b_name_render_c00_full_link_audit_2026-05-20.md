## 発見事項

### 1. AOV入力が実マテリアルのソケット型に合っていない

`c00.blend` には `マテリアル方舟` というメッシュオブジェクトが存在し、`無効` コレクション内にある。このオブジェクトには同名のマテリアルが貼られており、そのマテリアル内にAOV出力用のシェーダーノードグループが入っている。

確認結果:

- オブジェクト: `マテリアル方舟`
- 種別: メッシュ
- 所属: `無効`
- マテリアル: `マテリアル方舟`
- AOV系入力: `落ち影切替` / `透過切替`
- 検出されたAOV関連マテリアル: `マテリアル方舟` / `マテリアル方舟.001`

このマテリアルを検証用メッシュへ貼り、キャラ・背景・効果の各コレクションへ配置して B-Name-Render の全プリセットを実行した。

結果:

- `レイアウト`、`効果`、`背景パース`、`背景エフェクト`、`背景フォグ`、`背景雲`、`背景空`、旧互換のコマ出力系は画像を生成した。
- `キャラ`、`キャラパス`、`キャラ陰影`、`キャラアルファ`、`キャラ透過`、`キャラ_低速`、`キャラpen`、`背景`、`背景_低速`、`背景D`、`背景G`、`背景AO`、`背景線画用`、`背景ベース`、`背景植物`、`背景グラデ` などはAOV入力で失敗した。

失敗内容:

```text
NodeSocketInt.default_value expected an int type, not float
```

影響:

- AOV切替を含む主要なキャラ・背景プリセットが、レンダー前に止まる。
- `c00.blend` 側には必要なAOV用マテリアルがあるのに、B-Name-Render が値を正しく入れられていない。

対応:

- B-Name-Render のAOV入力カードは、対象ソケットの型を見て値を代入する必要がある。
- 整数/真偽値系の切替には整数値を入れる。
- 小数ソケットには小数値を入れる。
- 実行前監査で「対象AOV入力が何個見つかり、値を何個変更できたか」を表示する。

これは最新の B-Name では `c00.blend` 側の不足ではなく、B-Name-Render 側に旧仕様が残っている状態と判断する。現行 B-Name では、コマ用blendファイル側でページ全体や全コマ統合を完結させる前提は不要。

影響:

- 「すべて」プリセットも同じく、ページ側の仕上げ工程が空振りする。

### 4. Blender 5.1 のファイル出力ノード API に未対応

- 魚眼系出力も既定の `passes` に固定されやすい。

- Blender 5.1 のファイル出力ノード形式に対応する。

- 通常パス出力と魚眼出力の保存先を、B-Name のコマ用blendファイルの出力先へ統一する。

### 8. 出力範囲が B-Name の現在コマに完全連動していない

追加で、指定 `c00.blend` をコマ用blendファイルとしてコピーし、横長でも縦長でもない非対称なコマを作って、魚眼モード・縮小モード・B-Name-Render の魚眼系プリセット・ページ一覧へ戻る・再オープンを実機で往復した。

証跡:

- `.codex/ai_audit/20260520_output_range/bname_render_output_range_roundtrip.json`

結果:

- `c00.blend` 由来の古いカメラ下絵に、`コマ枠.png`、`コマ01`、`コマ02` など「コマ」を含む画像スロットが多数残っている。
- B-Name が生成した現在コマ用の下絵も追加されるが、古い「コマ」下絵より後ろに追加される。
- 現在のレンダー範囲計算は、最初に見つかった「コマ」を含む下絵を使うため、B-Name が生成した現在コマ用の下絵ではなく、`c00.blend` 側の古い下絵でレンダー範囲が決まる。
- そのため、B-Name-Render の出力範囲は現時点では最新 B-Name の現在コマに完全連動していない。

実機往復で確認した代表値:

```text
コマを開いた直後       6071x8598 / 魚眼ON / 縮小OFF / レンダー範囲 0.105,0.895,0.221093,0.778907
魚眼モードON           8598x8598 / 魚眼ON / 縮小OFF / レンダー範囲 0.105,0.895,0.105,0.895
魚眼ON+縮小50%         4299x4299 / 魚眼ON / 縮小ON  / レンダー範囲 0.105,0.895,0.105,0.895

```

影響:

- 魚眼モード中でも、コマ用blendファイルをページ一覧へ戻ってから開き直すと、魚眼・縮小フラグは残るのに、解像度が正方形/縮小後ではなく通常原稿サイズへ戻る。
- これは「魚眼の形状がページの形状にフィットした状態になる」症状と一致する。
- B-Name-Render の魚眼系プリセットは、開き直し直後に実行すると、B-Name が意図した魚眼・縮小レイアウトではなく、再オープン時に戻された出力範囲で動く可能性が高い。
- 古い `c00.blend` 下絵が残っている限り、数値上のレンダー範囲が合って見えても、B-Name の現在コマではなく旧下絵に従っている可能性がある。

対応:

- B-Name 側のレンダー範囲計算は、B-Name が生成した現在コマ用の下絵を最優先する。
- `c00.blend` 由来の未管理下絵は、レンダー範囲計算から除外する。
- コマ用blendファイルを開いた直後、現在の魚眼モード・縮小モード・縮小率を必ず再適用し、最後にレンダー範囲を再計算する。
- B-Name-Render のプリセット実行直前にも、B-Name の現在設定から出力解像度・魚眼設定・縮小設定・レンダー範囲を再同期する。
- `c00.blend` 側では、テンプレートとして不要な古いカメラ下絵を削除するか、少なくとも B-Name の現在コマ下絵と混同されない名前へ整理する。

## 完全連動テスト計画

この計画での「完全連動」は、単にプリセットがエラーなく終わることではない。最新 B-Name に必要な B-Name-Render の全プリセットについて、次の流れが実データで確認できた状態を指す。

1. `c00.blend` 内の実AOV用マテリアルを貼ったメッシュをレンダー対象にする。
2. B-Name-Render のプリセットで、必要なAOV切替とビューレイヤー切替を行う。
3. `出力_○○` 系ノードグループから、期待される全パス画像を実ファイルとして出す。
4. 出力されたパス画像を、合成側の画像ノードが相対パスで読み込む。
5. `C_○○` 系ノードグループを使う統合プリセットで、パス画像群から仕上げ画像を出す。

### AI目視の合格条件

- 全プリセットのパス画像と合成結果を、プリセット名・出力名つきの一覧画像へまとめる。
- 一覧画像には、エラー、欠落、空画像、未読込画像数を表示する。
- AI目視用プロンプトには「どのプリセットのどの画像を確認すべきか」を列挙する。
- 目視対象は代理色キューブだけでなく、`マテリアル方舟` 由来マテリアルを貼った実メッシュ出力を含む。
- すべてのパス画像について、「対象コレクションのメッシュが出ているか」「別コレクションが混ざっていないか」「アルファ/深度/線画/陰影/トーン/パース/効果/空/雲/フォグなど、出力名に対応した見え方になっているか」を確認する。
- すべてのコンポジット済み画像について、「必要なパス画像が反映されているか」「合成順・マスク・透明部分・線画・陰影・階調が破綻していないか」を確認する。
- 画像が存在しても、出力名と中身が一致しない場合は失敗にする。
- 画像が存在しても、検証用メッシュが消えている、対象外メッシュだけが見えている、全体が均一色で判定不能、想定しない黒つぶれ/白飛びがある場合は失敗にする。
- 魚眼モード・縮小モード・再オープン後の画像も目視対象に含め、魚眼形状がページ形状へ引き伸ばされていないことを確認する。
- B-Name-Render の出力範囲が、ページ一覧上の現在コマ位置と一致していることを確認する。
- AI目視シートは、パス画像用とコンポジット済み画像用を分け、必要ならプリセットカテゴリ別にも分割する。1枚に詰め込みすぎて判定不能にしない。
- AI目視結果はJSON/Markdownに残し、各画像を `OK` / `NG` / `要人間確認` に分類する。
