# B-MANGA Liner ライン局所細分化・輪郭平滑化 計画

日付: 2026-07-10
状態: 完了

## 目的

「中間頂点用サブディビジョンを自動設定」が元メッシュへ追加していた
サブディビジョンサーフェス、三角面の四角面化、クリース書込みを廃止する。
元メッシュとその素材表示を変更せず、B-MANGA Liner が生成するライン形状だけへ
補間点を追加する。

## 確定方針

- 元メッシュの頂点・辺・面・法線・クリース属性を変更しない。
- 元オブジェクトへサブディビジョンサーフェスを新規追加しない。
- ユーザーが作成した既存のサブディビジョンサーフェスは、名前が旧自動生成名と
  一致していても、所有情報が確認できない限り変更・削除しない。
- アウトラインは、元形状をそのまま出力する経路と、基準カメラから見える輪郭線を
  生成する経路を分離する。全面を複製したライン殻は生成しない。
- 輪郭線は浅い連続角だけを滑らかな曲線へ変換し、鋭い角、端点、分岐は維持する。
- 生成ラインは元メッシュ成分と結合せず、別インスタンスとして表示する。オブジェクト
  モードのワイヤー表示や後段のメッシュ評価へ、生成ラインの分割面を混入させない。
- 稜谷線・交差線・選択線は、検出後のカーブだけを細分化する。
- 板ポリ、開いたメッシュ、三角面主体、非多様体を含め、元メッシュを変更する
  旧方式へのフォールバックは設けない。形状ごとに旧方式と新方式が混在する
  状態を作らない。
- カメラからオブジェクト原点までの距離では細分化密度を決めない。巨大・広域
  オブジェクトでも同一のライン補間密度を使う。
- 「ビューポートのレベル数をレンダーに合わせる」は廃止し、B-MANGA Liner が
  ユーザーのサブディビジョンサーフェス段数を変更する経路をなくす。

## 実装

1. 基準カメラから見える面の境界だけを輪郭カーブへ変換するGeometry Nodesを
   独立モジュールへ実装する。元形状は無加工で通し、輪郭カーブだけを角保持付きで
   平滑化、線幅補間、法線方向移動、チューブ化して別インスタンスとして出力する。
2. 「中間頂点用サブディビジョンを自動設定」を
   「中間頂点用ライン細分化」に変更する。オンでは専用ライン殻と各線種カーブの
   補間を有効化し、オフでは補間を外す。
3. 旧自動生成モディファイアは、旧処理が残した所有情報を持つものだけを、
   明示的な反映・削除操作時に撤去する。読込だけでは元形状を変更しない。
4. 旧四角面化は不可逆なため、自動復元しない。既存ファイルの現在の面構成を
   そのまま保持する。
5. シーン全体のサブディビジョンサーフェス段数同期UI、プロパティ、更新処理、
   読込・レンダー時ハンドラーを削除する。

## 回帰条件

- 元メッシュの頂点・辺・面数、座標、素材割当、UV、法線が反映前後で一致する。
- ユーザー作成のサブディビジョンサーフェスの方式、段数、レンダー段数、表示、
  順序が反映・削除・保存再読込・レンダー前後で一致する。
- アウトライン、稜谷線、交差線、選択線を同時表示しても、線幅・色・素材表示・
  テクスチャ表示が変わらない。
- 三角面、開いた形状、非多様体、重複面を含むテストメッシュで処理が停止せず、
  元メッシュへ書込みを行わない。
- 低密度メッシュで中間頂点の線幅変化が生成ライン側の補間点へ現れる。
- 旧自動生成物を含むファイルで、安全に所有確認できる生成物だけが反映操作時に
  撤去される。

## 検証

- Python構文検査。
- B-MANGA Linerの関連実機回帰テスト。
- 新規のライン局所細分化実機テスト。
- Cube、Cylinder、Cone、Plane、三角面主体、非多様体の画面レンダー比較。
- 元メッシュ成分の頂点・辺・面数、元メッシュ指紋、ユーザー作成モディファイアの
  完全一致と、生成ラインを実体化した場合の面数を別々に機械検査する。

## 完了結果

- 元メッシュへサブディビジョンサーフェス、四角面化、クリース書込みを行う経路を
  廃止した。初回実装の全面ライン殻も、元メッシュが細分化されたように見えることと、
  線形分割では輪郭が滑らかにならないことを確認したため廃止した。
- 最終方式は、基準カメラから見える輪郭線だけを抽出し、40度未満の浅い曲がりを
  滑らかに補間する。角柱の直角、端点、分岐は直線のまま維持する。
- 生成ラインは別インスタンスとして保持し、元メッシュ成分の頂点・辺・面数と
  テクスチャ表示を変えない。近距離でも元面を覆う全面殻を生成しない。
- 「ビューポートのレベル数をレンダーに合わせる」を削除し、ユーザー作成の
  サブディビジョンサーフェスを反映・削除・保存再読込で変更しないことを確認した。
- 12,800三角面の元メッシュ成分は12,800面のまま保持し、輪郭線を実体化した場合だけ
  61,440面を追加する。評価時間は0.006〜0.012秒で、全面ライン殻の約0.023秒より短い。
- 開いた三角面、非多様体、重複面、同名のユーザー作成モディファイアを含む実機
  テストに合格した。
- Blender 5.1.2で、低密度円形メッシュ、角柱、板ポリ、開いた三角面、非多様体、
  重複面、負のスケール、透視投影、平行投影を検証した。
- 実データのマンホールで元メッシュ49頂点・96辺・48面が評価後も完全一致し、
  テクスチャを保ったまま輪郭が滑らかに密着することをAI目視で確認した。
- アウトライン・稜谷線・交差線の同時表示、透明面の奥を隠す設定、ラインのみ表示で、
  飛び出し、黒ベタ、元面の隠れ、線幅・線色の異常がないことを確認した。
- B-MANGA Linerの実機回帰69本を、Blenderの終了コードだけでなくPython例外文字列も
  失敗扱いにして全件合格した。
- 実データではレンダリング範囲644オブジェクトの選択が0.043秒、作成範囲内137件の
  アウトライン反映が2.10秒、作成済み22件の直接更新が0.12秒だった。
