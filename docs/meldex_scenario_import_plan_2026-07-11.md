# Meldexシナリオ取込計画（2026-07-11）

状態: 完了（B-MANGA v0.6.481 / Meldex v0.6.175、AI目視で長いルビの四辺余白を追補）

## 目的

Meldexの現行「シナリオ」からB-MANGAへ、ページ構成、タイプ、本文、改行、ルビを安全に送信し、既存作品へフキダシとテキストを生成する。

## 確定仕様

- コマは生成・変更しない。
- シナリオのページ数がB-MANGAの既存ページ数を超える場合だけ、不足ページを末尾へ追加する。既存ページと余剰ページは保持する。
- シナリオ各行の「タイプ」文字列とプリセット名は、空白・大文字小文字・Unicode正規化を一切変えず完全一致で比較する。
- 同名のフキダシプリセットがあればフキダシへ適用し、無ければ標準の楕円フキダシを使う。
- 同名のテキストプリセットがあれば子テキストへ併用し、無ければ標準テキストを使う。両プリセットが同名なら両方を適用する。
- すべての本文行はフキダシ＋子テキストとして生成する。区切り行など本文が空の行は生成しない。
- テキストはMeldexの明示改行を保持する。明示ルビ、自動ルビ、エスケープ文字、手動リンクの表示文字列をMeldex画面と同じ可視本文＋構造化ルビへ変換する。
- テキストプリセット適用後に自然サイズを計算し、ルビを含む文字領域へ余白を加えてフキダシを自動調整する。
- 同じシナリオを再送した場合は取込元文書ID＋行IDで既存のMeldex由来要素を更新し、重複生成しない。手作業要素は削除・変更しない。

## 送信契約

- Meldexデスクトップ版／シナリオ単独アプリのローカル処理を経由し、B-MANGAへ送る。Cloud版からは送信しない。
- endpointは `POST /api/scenario`、契約名 `meldex-bmanga-scenario`、契約版1。
- payloadは送信元文書情報と、ページごとの `rowId` / `type` / `body` / `rubies` を持つ。B-MANGA側で作品名・話数を推測して上書きしない。

## 安全設計

- B-MANGAの受信機能は既定オフ。ユーザーがプリファレンスで明示的に有効化する。
- 32バイト相当の接続トークンを生成し、ヘッダーを定数時間比較する。トークンはログ、応答、設定書き出しZIPへ含めない。
- Originヘッダー付き通信を値に関係なく拒否し、MeldexのPython中継だけを許可する。
- 127.0.0.1固定、指定ポートでfail-closed、JSON限定、本文サイズ上限、キュー上限、契約検証を行う。
- Meldex中継は送信先を127.0.0.1＋検証済みポートへ固定し、任意URLを受け取らない。

## B-MANGA実装

- 受信HTTPとシナリオ取込ロジックを分離する。
- 作品一覧ファイルでページ詳細を一時読込し、既存内容へ追記してpage.jsonを保存後、元が未読込だったページは再び解放する。
- 不足ページ追加にはページ登録とディレクトリ作成だけを使い、基本枠コマ作成処理を呼ばない。
- 一括取込中はフキダシ／テキスト実体の自動生成を停止し、現在開いているページだけ必要時に再同期する。
- 取込元文書ID・行IDをフキダシ／テキストの非表示保存項目として保持する。

## Meldex実装

- シナリオの「エクスポート」へ「B-MANGAへ送信...」を追加する。
- 保存直前と同じ同期を行ってから、区切りタイプでページ分割し、明示／自動ルビを可視本文上の範囲へ解決する。
- ブラウザ画面からB-MANGAへ直接送らず、ローカルAPI→Python中継→B-MANGAの順に送る。
- ダイアログでポート、接続トークン、送信ページ数・本文数を確認できる。トークンは永続保存しない。

## 検証

- 純粋テスト: ページ分割、完全一致4組合せ、明示／自動ルビ、改行、エスケープ、リンク表示、再送時の冪等性、契約不正。
- HTTPテスト: 既定オフ、Origin拒否、認証失敗、正常受信、JSON以外、過大本文、キュー満杯、停止後接続不可、秘密情報非出力。
- Blender 5.1.2実機: 既存2ページ＋3ページ送信で1ページだけ追加、全コマ不変、既存レイヤー保持、フキダシ／テキストプリセット、標準フキダシ、改行・ルビ表示、自動サイズ、保存再読込、再送重複なし。
- Meldex実ブラウザ: デスクトップ版とシナリオ単独アプリで送信ダイアログ、成功／失敗案内、Cloud版非表示を確認する。

## 追記 (2026-07-12)

2026-07-12 ユーザー指示: 取込で追加した不足ページには通常のページ追加 (BMANGA_OT_page_add) と同じ基本枠コマを作成する仕様へ変更した（既存ページは従来どおり変更しない）。上記「B-MANGA実装」の「不足ページ追加にはページ登録とディレクトリ作成だけを使い、基本枠コマ作成処理を呼ばない」は本変更で上書きされる。

あわせて、自動生成するフキダシ／テキストのレイヤー順を「紐付けられた各テキストレイヤーの直下 (前面側) にフキダシレイヤーを配置する」仕様へ変更した。`utils/layer_stack.py` の `_collect_page_layer_targets` と `_find_insert_index_for_target` にペア対応の並び替えを追加し、取込経路 (`io/meldex_scenario_import.py` の `import_document`) では同期直後に `layer_stack.normalize_paired_layer_order` で今回新規作成したペアだけを最終保証する。手動で並び替えた既存ペアの順序は変更しない。詳細は `_verify/2026-07-12_meldex_pair_order/` の検証スクリプトを参照。
